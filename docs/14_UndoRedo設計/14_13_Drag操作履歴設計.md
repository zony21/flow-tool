# 14_13_Drag操作履歴設計

## 1. 目的

本書は、AI Flow DesignerのDrag操作におけるUndo / Redo履歴粒度を定義する。

Drag操作はmousemoveが大量に発生するため、すべてをCommand化すると履歴が壊れ、性能も低下する。

## 2. 基本方針

- Drag中の中間移動は履歴化しない。
- Drag開始時の状態をbeforeとして保持する。
- Drag完了時の状態をafterとして保持する。
- 1回のDragを1Commandにする。
- 複数選択DragはCompositeCommandまたはMoveSelectionCommandにする。

## 3. 対象操作

- Node移動
- Comment移動
- Nodeリサイズ
- Lane高さ変更
- Stage幅変更
- 複数Node移動

## 4. Drag開始

保持するもの:

- targetId
- before position
- before size
- before laneId
- before stageId

## 5. Drag中

Drag中は表示のみ更新する。

禁止:

- undoStackへpush
- SaveRequest生成
- Backend保存
- Validation全体実行

## 6. Drag完了

Drag完了時にCommandを生成する。

移動がない場合はCommandを作らない。

## 7. テスト観点

- 1回のDragで1Commandだけ生成される。
- 移動なしDragではCommand生成されない。
- 複数Node Dragが1履歴になる。

## 8. 完了条件

- Drag中とDrag完了の責務が分離されている。
- Drag履歴粒度が定義されている。
