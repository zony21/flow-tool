# 14_17_履歴表示設計

## 1. 目的

本書は、AI Flow DesignerのUndo / Redo履歴表示設計を定義する。

履歴表示は、ユーザーが直前の操作やUndo / Redo可能な操作を把握するための補助UIである。初期実装ではToolbarの活性制御とTooltipを中心にし、詳細なHistory Panelは将来対応とする。

## 2. 基本方針

- 初期はToolbarでUndo / Redo可否を表示する。
- Undo / Redoボタンに次に戻せる操作名をTooltip表示する。
- 詳細History Panelは将来機能とする。
- 履歴表示はFrontend一時状態であり、Backend保存しない。

## 3. Toolbar表示

Undoボタン:

- undoStackが空なら非活性。
- undoStack末尾のlabelをTooltip表示。

Redoボタン:

- redoStackが空なら非活性。
- redoStack末尾のlabelをTooltip表示。

## 4. History Panel 将来方針

将来、以下を表示する。

- Command一覧
- commandType
- label
- createdAt
- targetIds
- 保存済み位置

## 5. 保存済み位置表示

将来、lastSavedCommandIdを利用して、どこまで保存済みかを表示できるようにする。

## 6. 禁止事項

- 履歴表示をBackendへ保存する。
- Command内部の巨大PayloadをそのままUI表示する。
- 削除済要素の詳細を壊れた状態で表示する。

## 7. テスト観点

- undoStackが空のときUndoボタンが非活性になる。
- redoStackが空のときRedoボタンが非活性になる。
- TooltipにCommand labelが表示される。

## 8. 完了条件

- Toolbarでの履歴表示方針が定義されている。
- 将来History Panelへの拡張方針が定義されている。
