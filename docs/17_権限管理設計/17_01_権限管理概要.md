# 17_01_権限管理概要

## 1. 目的

AI Flow Designerの認証、利用者、役割、操作制御の基本設計を定義する。

## 2. 基本方針

- 認証と権限判定を分離する。
- Project単位で利用権限を管理する。
- Frontend制御だけに依存しない。
- Backend API側で必ず確認する。

## 3. 管理対象

- User
- ProjectMember
- Role
- Permission

## 4. Role

- Owner
- Admin
- Editor
- Viewer

## 5. 制御対象

- Project
- Flow
- Template
- Image
- Export
- Setting

## 6. 完了条件

利用者、役割、操作制御の基本方針が定義されている。
