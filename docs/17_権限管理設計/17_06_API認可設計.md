# 17_06_API認可設計

## 1. 目的

Backend API実行時の操作確認方針を定義する。

## 2. 基本方針

- Frontend制御だけを信用しない。
- API処理前に権限確認を行う。
- Project情報からRoleを取得する。

## 3. 処理流れ

1. User確認
2. Project確認
3. Role取得
4. Permission確認
5. 処理実行

## 4. 対象API

- Project API
- Flow API
- Template API
- Image API
- Export API

## 5. 完了条件

Backend側の操作確認責務が定義されている。
