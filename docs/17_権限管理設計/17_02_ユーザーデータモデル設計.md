# 17_02_ユーザーデータモデル設計

## 1. 目的

利用者情報を管理するためのデータ構造を定義する。

## 2. User

保持項目:

- userId
- loginProvider
- providerUserId
- displayName
- email
- createdAt
- updatedAt

## 3. ProjectMember

保持項目:

- projectId
- userId
- role

## 4. 基本方針

- UserとProject権限は分離する。
- 同じUserでもProjectごとにRoleを変更できる。

## 5. 完了条件

利用者情報とProject参加情報の責務が分離されている。
