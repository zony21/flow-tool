# 16_20_画像管理設計まとめ

## 1. 目的

本書は、16_画像管理設計の総まとめである。

画像管理は、画像本体をStorageへ置き、DBには管理情報を保持する方式とする。Image Node、Export、Version Snapshotと連携し、設計情報として画像を安全に扱う。

## 2. 責務

- 画像登録
- 画像表示
- 画像差替
- 画像削除
- 画像Metadata管理
- Image Node連携
- Export連携
- 権限確認

## 3. 対象外

- Undo / Redo履歴への画像本体保存
- DBへの画像本体保存
- Storage内部パスの公開

## 4. 実装順序

1. 画像データモデル
2. Storage Service
3. Upload API
4. 表示 API
5. Image Node連携
6. Frontend UI
7. Error / Validation
8. 確認観点整備

## 5. 完了条件

16_画像管理設計は、画像登録、Storage、Image Node連携、表示、差替、削除、権限、API、Frontend、Error方針が定義されている状態を完了とする。
