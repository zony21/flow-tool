# 16_08_画像差替設計

## 1. 目的

登録済み画像の差替仕様を定義する。

## 2. 基本方針

- Image Nodeは維持する。
- 差替時も参照関係を壊さない。
- Metadataを更新する。
- 必要に応じて旧画像を保持する。

## 3. 差替フロー

1. 新画像選択
2. Validation
3. Storage保存
4. Metadata更新
5. Thumbnail再生成
6. 表示更新

## 4. Version連携

Version Snapshotが参照する画像は保持可能にする。

## 5. 禁止事項

- Node削除による差替
- 参照切れ発生

## 6. 完了条件

安全に画像差替できる仕様が定義されている。
