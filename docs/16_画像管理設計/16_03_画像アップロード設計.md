# 16_03_画像アップロード設計

## 1. 目的

画像アップロード処理の仕様を定義する。

## 2. 基本方針

- Frontendで事前Validationする。
- Backendで必ず再検証する。
- 保存前にファイル名を内部ID化する。
- MetadataをDBへ保存する。
- 実体はStorageへ保存する。

## 3. 対応形式

- PNG
- JPG
- JPEG
- SVG
- WebP

## 4. Upload Flow

1. ファイル選択
2. 拡張子確認
3. サイズ確認
4. Backend送信
5. MIME確認
6. Storage保存
7. Metadata登録

## 5. Validation

- ファイルサイズ上限
- MIME
- 拡張子
- SVG安全確認

## 6. 完了条件

安全に画像アップロードできる処理が定義されている。
