# 16_画像管理設計

## 1. 本書の位置付け

本フォルダはAI Flow Designerの画像管理機能設計を管理する。

画像は単なる装飾ではなく、Flow理解を補助する設計情報として扱う。

## 2. 基本方針

- DBには画像バイナリを直接保存しない。
- Storage + Metadata方式を採用する。
- 初期はローカルStorage、将来クラウドStorageへ移行可能にする。
- Image Nodeと連携する。
- Version Snapshot、Export、権限を考慮する。
- Storage内部パスをFrontendへ直接公開しない。
