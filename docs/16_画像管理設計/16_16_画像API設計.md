# 16_16_画像API設計

## 1. 目的

画像管理で利用するAPIの責務を定義する。

## 2. API一覧

- 画像登録
- 画像一覧取得
- 画像詳細取得
- 画像表示
- 画像更新
- 画像削除

## 3. 共通方針

- Project権限を確認する。
- Metadataを返却する。
- 内部保存先は返却しない。
- ErrorResponse形式を統一する。

## 4. 登録API

入力:

- projectId
- file
- description

出力:

- imageId
- originalFileName
- mimeType
- fileSize
- width
- height

## 5. 表示API

画像表示はAPI経由で行う。
権限確認後、適切なContent-Typeで返却する。

## 6. テスト観点

- 権限ありユーザーが画像登録できる。
- 権限なしユーザーは画像取得できない。
- 内部保存先がResponseに含まれない。

## 7. 完了条件

画像管理APIの責務と返却方針が定義されている。
