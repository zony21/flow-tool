# 16_02_画像データモデル

## 1. 目的

画像ファイル管理用データモデルを定義する。

## 2. 基本方針

画像本体はStorage、管理情報はDBで保持する。

## 3. IMAGE_FILE

保持項目:

- IMAGE_ID
- PROJECT_ID
- FILE_NAME
- ORIGINAL_FILE_NAME
- MIME_TYPE
- FILE_SIZE
- STORAGE_PATH
- HASH
- WIDTH
- HEIGHT
- IS_DELETED
- SYSTEM_CREATE_DATETIME
- SYSTEM_UPDATE_DATETIME

## 4. 関連

Image NodeはIMAGE_IDを参照する。

## 5. 禁止事項

- DBへの画像バイナリ保存
- 絶対パスの直接公開
- 未使用画像の永久保持

## 6. 完了条件

画像MetadataとStorage責務が分離されている。
