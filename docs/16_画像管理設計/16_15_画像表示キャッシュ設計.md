# 16_15_画像表示キャッシュ設計

## 1. 目的

画像表示を高速化しつつ、差替後に古い表示が残らないようにする方針を定義する。

## 2. 基本方針

- Browser Cacheを活用する。
- 画像更新時は識別値を変更する。
- Frontendは内部保存先を直接参照しない。

## 3. 対象

- Thumbnail
- Canvas表示画像
- Preview画像

## 4. 更新時の扱い

画像差替時はupdatedAtまたはversionをURL生成に反映する。

## 5. テスト観点

- 再表示時にCacheを利用できる。
- 差替後は新しい画像が表示される。
- 権限なし画像は表示できない。

## 6. 完了条件

画像表示速度と更新反映を両立できる設計になっている。
