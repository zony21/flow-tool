# 15_19_テンプレートテスト設計

## 1. 目的

本書は、AI Flow DesignerのTemplate機能に関するテスト設計を定義する。

TemplateはFlow構造の再利用機能であり、ID再採番、参照整合性、Undo / Redo、Import / Export、権限を重点的に検証する。

## 2. テスト分類

- Unit Test
- Service Test
- API Test
- UI Test
- E2E Test

## 3. Unit Test

対象:

- TemplateBuilder
- TemplateValidator
- TemplateApplyService
- ID再採番処理
- TemplateJson Serializer

観点:

- TemplateJsonを生成できる。
- Link参照を検証できる。
- Comment参照を検証できる。
- SchemaVersionを検証できる。

## 4. Service Test

- Template作成
- Template更新
- Template削除
- Template複製
- Template適用
- Import
- Export

## 5. API Test

- Template一覧取得
- Template詳細取得
- Template保存
- Template削除
- Template Import
- Template Export

## 6. UI Test

- Template Dialog表示
- 検索
- Preview
- Append適用
- Replace確認Dialog
- 保存Dialog

## 7. E2E Test

代表シナリオ:

1. FlowからTemplateを保存する。
2. Template一覧で検索する。
3. TemplateをAppend適用する。
4. Undoで戻す。
5. Redoで再適用する。
6. TemplateをExportする。
7. ExportしたTemplateをImportする。

## 8. 権限テスト

- ViewerはTemplateを適用できない。
- EditorはTemplateを作成できる。
- Project外ユーザーはProject Templateを参照できない。

## 9. 完了条件

- Template保存、読込、適用、ID再採番、権限、Import / Exportのテスト観点が定義されている。
