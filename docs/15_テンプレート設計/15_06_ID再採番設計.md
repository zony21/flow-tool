# 15_06_ID再採番設計

## 1. 目的

本書は、TemplateをFlowへ適用する時のID再採番方針を定義する。

Templateは再利用される部品であるため、Template内のIDをそのまま現在Flowへ入れてはならない。適用時に新しいIDへ置き換え、要素同士のつながりも新しいID同士で成立するようにする。

## 2. 基本方針

- Template適用時に新しいIDを発行する。
- Lane、Stage、Node、Link、Commentを対象にする。
- 旧IDと新IDの対応を一時的に保持する。
- 置き換え後に構造Validationを行う。
- 置き換えに失敗したTemplateは適用しない。

## 3. 対象

- Lane ID
- Stage ID
- Node ID
- Link ID
- Comment ID

## 4. 処理順序

1. Laneの新IDを作成する。
2. Stageの新IDを作成する。
3. Nodeの新IDを作成する。
4. Linkの新IDを作成する。
5. Commentの新IDを作成する。
6. 各要素の参照先を新IDへ合わせる。
7. Validationを実行する。

## 5. 参照更新

Linkは新しいNode ID同士を参照するように更新する。

Commentは対象種別に応じて、新しいLane、Stage、Node、LinkのIDへ更新する。

参照先がTemplate内に存在しない場合は、Validation ErrorまたはWarningとして扱う。

## 6. 禁止事項

- Template内IDをそのままFlowへ入れる。
- 参照更新に失敗した状態で適用する。
- ID再採番前の情報を保存済みFlowとして扱う。

## 7. テスト観点

- 適用時に各要素のIDが変わる。
- Linkが新しいNodeを参照する。
- Commentが新しい対象を参照する。
- 参照先がない場合にErrorになる。

## 8. 完了条件

- ID再採番対象と処理順序が定義されている。
- Template適用時の参照更新方針が定義されている。
