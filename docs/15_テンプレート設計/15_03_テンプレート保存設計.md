# 15_03_テンプレート保存設計

## 1. 目的

本書は、AI Flow DesignerのTemplate保存設計を定義する。

Template保存は、現在のFlow構造または選択範囲を再利用可能なTemplateJsonとして保存する処理である。

## 2. 保存元

保存元は以下とする。

- 現在Flow全体
- 選択中のLane / Stage / Node / Link / Comment
- 既存Templateの複製
- Importファイル

## 3. 保存対象

- Lane
- Stage
- Node
- Link
- Comment
- AI専用メモ
- NodeType固有Property
- Layout情報
- Style情報

## 4. 保存対象外

- Undo / Redo履歴
- Selection状態
- Viewport
- Zoom / Pan
- 編集ロック
- Version Snapshot

## 5. 保存フロー

```text
TemplateSaveDialog
  ↓
TemplateService.create
  ↓
TemplateBuilder
  ↓
TemplateValidator
  ↓
TemplateRepository
  ↓
保存完了
```

## 6. TemplateBuilder

TemplateBuilderはFlow構造からTemplateJsonを生成する。

処理:

1. 保存対象要素を収集する。
2. 対象Node間のLinkを抽出する。
3. 対象Commentを抽出する。
4. Template内IDを設定する。
5. Metadataを付与する。
6. TemplateJsonを生成する。

## 7. 選択範囲保存

選択NodeをTemplate化する場合、選択Node間のLinkのみ保存する。

選択外Nodeへ接続するLinkは保存対象外、または外部接続情報としてWarning扱いにする。

## 8. Validation

保存前に以下を検証する。

- Template名必須
- Template名重複
- 保存対象要素が1件以上
- Linkのsource / targetがTemplate内に存在する
- image NodeのimageFileIdが存在する

## 9. テスト観点

- Flow全体をTemplate保存できる。
- 選択NodeだけをTemplate保存できる。
- 選択外NodeへのLinkを除外できる。
- Template名重複を検出できる。

## 10. 完了条件

- Template保存元、保存対象、保存フローが定義されている。
- 実装者がTemplate保存処理を実装できる。
