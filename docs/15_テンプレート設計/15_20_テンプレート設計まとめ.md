# 15_20_テンプレート設計まとめ

## 1. 目的

本書は、15_テンプレート設計の総まとめである。

Templateは、AI Flow DesignerにおけるFlow構造の再利用機能である。Lane、Stage、Node、Link、Comment、AIメモを構造化データとして保存し、別Flowへ安全に適用する。

## 2. Templateの責務

- Flow構造の再利用
- 標準Template提供
- Project Template管理
- Template保存
- Template適用
- Template複製
- Import / Export
- 検索・分類

## 3. Templateの対象外

- Undo / Redo履歴
- Version Snapshot
- ProjectMember
- 編集ロック
- Canvas Viewport
- Zoom / Pan
- Selection状態

## 4. 実装順序

1. Templateデータモデル
2. TemplateRepository
3. TemplateService
4. Template保存
5. Template読込
6. Template適用
7. ID再採番
8. Template UI
9. Import / Export
10. Test

## 5. 実装チェックリスト

- [ ] TemplateJsonにschemaVersionを持たせる。
- [ ] Template一覧でTemplateJson全体を返さない。
- [ ] 適用時にIDを再採番する。
- [ ] Link参照を更新する。
- [ ] Comment参照を更新する。
- [ ] Replace時は確認Dialogを表示する。
- [ ] Template適用をUndo可能にする。
- [ ] Standard Templateを直接編集させない。
- [ ] Import JSONを検証する。

## 6. 完了条件

15_テンプレート設計は、以下を満たすことで完了とする。

- Template概要、データモデル、保存、読込、適用が定義されている。
- Lane / Stage / Node / Link / Comment / AIメモの扱いが定義されている。
- Standard Template、Project Template、複製、検索、Import / Exportが定義されている。
- Version互換、権限、UI、Error、Testが定義されている。
- 実装者が本章を読んでTemplate機能を実装できる。
