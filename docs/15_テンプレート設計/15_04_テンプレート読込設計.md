# 15_04_テンプレート読込設計

## 1. 目的

本書は、AI Flow DesignerのTemplate読込設計を定義する。

Template読込は、一覧表示、詳細表示、Preview、適用前確認のためにTemplate情報を取得する処理である。

## 2. 基本方針

- 一覧APIではTemplateJson全体を返さない。
- 詳細APIでTemplateJsonを取得する。
- Preview用に軽量なSummaryを用意する。
- Project権限を確認する。
- Standard Templateは対象Projectに依存せず参照可能とする。

## 3. 一覧項目

- templateId
- templateName
- description
- templateType
- category
- tags
- updatedAt
- nodeCount
- linkCount

## 4. 詳細項目

- templateId
- templateName
- description
- templateType
- category
- tags
- templateJson
- schemaVersion
- metadata

## 5. Preview

Previewでは以下を表示する。

- Lane数
- Stage数
- Node数
- Link数
- Comment数
- 代表Node名

Canvas Previewは将来機能とする。

## 6. 読込フロー

```text
TemplateListPage
  ↓
TemplateApi.search
  ↓
TemplateService.search
  ↓
TemplateRepository.search
```

詳細:

```text
TemplateDetailDialog
  ↓
TemplateApi.getDetail
  ↓
TemplateService.getDetail
  ↓
TemplateRepository.findById
```

## 7. 権限

Project TemplateはProject権限が必要。
Standard Templateはログイン済みユーザーが参照可能。

## 8. テスト観点

- Template一覧を取得できる。
- 一覧でTemplateJsonを返さない。
- 詳細でTemplateJsonを取得できる。
- 権限なしProject Templateを取得できない。

## 9. 完了条件

- Template一覧、詳細、Preview、権限方針が定義されている。
- 実装者がTemplate読込処理を実装できる。
