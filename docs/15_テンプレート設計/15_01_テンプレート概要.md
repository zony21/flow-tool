# 15_01_テンプレート概要

## 1. 目的

本書は、AI Flow DesignerのTemplate機能全体方針を定義する。

Templateは、繰り返し利用するFlow構造を再利用可能な単位として保存し、新規Flow作成や設計標準化を支援する機能である。

Templateは見た目だけの複製ではない。Lane / Stage / Node / Link / Comment / AIメモを含むSSOT構造を再利用する。

## 2. 基本方針

- Templateは構造化データとして保持する。
- 適用時はIDを必ず再採番する。
- Link参照、Comment参照も再採番後のIDへ更新する。
- 標準TemplateとProject Templateを分ける。
- Template適用はUndo / Redo可能な複合Commandとして扱う。
- TemplateはVersion Snapshotではない。
- Template Import / Exportへ将来対応できる構造にする。

## 3. Template種別

| 種別 | 内容 |
| --- | --- |
| Standard Template | システム標準Template |
| Project Template | Project内で管理するTemplate |
| Duplicated Project Template | 既存ProjectやFlowから生成するTemplate |
| Imported Template | 外部ファイルから取り込むTemplate |

## 4. 保存対象

Templateに含められるもの:

- Lane
- Stage
- Node
- Link
- Comment
- AI専用メモ
- Layout情報
- Style情報
- NodeType固有Property

保存しないもの:

- ProjectMember
- Version Snapshot
- 編集ロック
- Undo / Redo履歴
- Canvas Viewport
- Zoom / Pan
- Selection状態

## 5. 適用モード

Template適用には以下を用意する。

- Append: 現在Flowへ追加
- Replace: 現在Flowを置換
- New Flow: Templateから新規Flow作成

## 6. SSOTとの関係

TemplateはSSOTの一部構造を保存したものだが、適用前はFlowの正ではない。

適用時にID再採番と構造Validationを行い、Flow構造へ変換してから編集対象になる。

## 7. Undo / Redoとの関係

Template適用は複数要素をまとめて追加・置換するため、CompositeCommandとして扱う。

Undoで適用前状態へ戻し、Redoで再適用できる。

## 8. Version Snapshotとの違い

| 項目 | Template | Version Snapshot |
| --- | --- | --- |
| 目的 | 再利用 | 正式履歴 |
| 適用先 | 任意Flow | 元Flow中心 |
| ID | 再採番 | 当時の状態保持 |
| Export対象 | 将来対応 | 対応 |
| Undo対象 | 適用操作は対象 | 作成自体は対象外 |

## 9. 管理機能

- 作成
- 更新
- 削除
- 複製
- 検索
- 分類
- Import
- Export
- 適用

## 10. テスト観点

- TemplateからFlow要素を生成できる。
- 適用時にID再採番される。
- Link参照が再採番後IDへ更新される。
- Append適用をUndoできる。
- Replace適用をUndoできる。
- Standard TemplateとProject Templateを区別できる。

## 11. 完了条件

- Templateの目的、種別、保存対象、適用モードが定義されている。
- SSOT、Undo / Redo、Snapshotとの責務分離が明確である。
- 実装者がTemplate機能全体を理解できる。
