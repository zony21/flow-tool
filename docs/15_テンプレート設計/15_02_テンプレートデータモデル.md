# 15_02_テンプレートデータモデル

## 1. 目的

本書は、AI Flow DesignerのTemplateデータモデルを定義する。

Templateは、Flow構造の一部または全体を再利用可能な単位として保持するため、検索用メタデータと構造データを分けて管理する。

## 2. 基本方針

- Template本体はDBで管理する。
- Template構造はTemplateJsonで保持する。
- TemplateJsonにはSchemaVersionを持たせる。
- 適用時はTemplateJsonをFlow構造へ変換する。
- 検索・分類に必要な項目はDB列として持つ。

## 3. TEMPLATE項目

- TEMPLATE_ID
- PROJECT_ID
- TEMPLATE_NAME
- DESCRIPTION
- TEMPLATE_TYPE
- CATEGORY
- TAGS
- TEMPLATE_JSON
- SCHEMA_VERSION
- IS_STANDARD
- IS_DELETED
- SYSTEM_CREATE_USER_ID
- SYSTEM_CREATE_DATETIME
- SYSTEM_UPDATE_USER_ID
- SYSTEM_UPDATE_DATETIME

## 4. TEMPLATE_TYPE

| 値 | 内容 |
| --- | --- |
| standard | システム標準 |
| project | Project内Template |
| imported | Import Template |
| duplicated | 既存Flow複製Template |

## 5. TemplateJson構造

```json
{
  "schemaVersion": "1.0",
  "name": "template name",
  "lanes": [],
  "stages": [],
  "nodes": [],
  "links": [],
  "comments": [],
  "metadata": {}
}
```

## 6. TemplateJson要素

- TemplateLane
- TemplateStage
- TemplateNode
- TemplateLink
- TemplateComment

各要素はTemplate内IDを保持する。Flowへ適用する時は新しいFlow用IDへ変換する。

## 7. Metadata

- createdFromProjectId
- createdFromFlowId
- sourceAppVersion
- recommendedUseCase

## 8. 禁止事項

- 検索項目をTemplateJson内だけに閉じ込める。
- Template適用後もTemplate内IDをそのままFlow IDとして使う。
- Undo / Redo履歴をTemplateJsonへ保存する。
- Version SnapshotをTemplateとしてそのまま扱う。

## 9. テスト観点

- TemplateJsonを読み書きできる。
- SchemaVersionを保持できる。
- Template一覧を検索用項目で取得できる。
- Template適用時にID再採番できる。

## 10. 完了条件

- TEMPLATE項目とTemplateJson構造が定義されている。
- 検索用メタデータと構造データの責務が分離されている。
