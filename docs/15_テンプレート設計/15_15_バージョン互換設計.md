# 15_15_バージョン互換設計

## 1. 目的

本書は、AI Flow DesignerのTemplateバージョン互換設計を定義する。

Templateは長期利用されるため、将来のSchema変更後も可能な限り読み込める必要がある。

## 2. 基本方針

- TemplateJsonにschemaVersionを持たせる。
- 読込時にschemaVersionを確認する。
- 未対応Versionは適用前に拒否する。
- 将来はVersion変換処理を用意する。

## 3. SchemaVersion

初期値:

```text
1.0
```

## 4. 互換方針

| 変更 | 方針 |
| --- | --- |
| 項目追加 | 既定値で補完 |
| 項目名変更 | Converterで変換 |
| 必須構造変更 | 適用前に警告または拒否 |
| NodeType廃止 | 代替NodeTypeへ変換または拒否 |

## 5. Converter

将来、以下のような変換器を用意する。

```text
TemplateSchemaV1ToV2Converter
```

## 6. Import時の扱い

ImportされたTemplateが未対応schemaVersionの場合、保存前に拒否する。

## 7. テスト観点

- schemaVersion 1.0を読み込める。
- 未対応Versionを拒否できる。
- 項目追加時に既定値補完できる。

## 8. 完了条件

- schemaVersion、互換方針、Converter方針が定義されている。
