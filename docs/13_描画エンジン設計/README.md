# 13_描画エンジン設計

## 1. 本書の位置付け

本フォルダは、AI Flow Designerの描画エンジン設計を管理する。

描画エンジンは、SSOTであるProject / Flow / Lane / Stage / Node / Link / Commentを視覚的に編集するための表示・操作基盤である。
ただし、描画状態そのものは正ではない。正となるのはBackendで検証・保存された構造化データである。

## 2. 設計思想

- Vue Flowを利用するが、Vue Flow内部ModelをSSOTにしない。
- 描画Modelと保存ModelをAdapterで分離する。
- Canvasは編集体験を担う。
- Lane / Stage / Node / Link / Commentを統合的に描画する。
- 大規模Flowでも操作性を維持できる設計にする。
- 将来、描画ライブラリ変更が起きても影響を局所化できるようにする。
