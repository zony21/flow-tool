# 13_01_描画エンジン概要

## 1. 目的

本書は、AI Flow Designerの描画エンジン全体方針を定義する。

描画エンジンは、Project / Flow / Lane / Stage / Node / Link / Commentで構成されるSSOTを、ユーザーが直感的に編集できるCanvasとして表示するための基盤である。

ただし、Canvas上の見た目やVue Flow内部状態はSSOTではない。正となるのはBackendで保存・検証された構造化データである。

## 2. 基本方針

- Vue Flowを利用する。
- Vue Flow内部ModelをSSOTにしない。
- SSOTから描画Modelを生成する。
- 描画操作はCommandへ変換してStoreへ反映する。
- 描画状態、選択状態、Drag中状態を保存Modelへ混ぜない。
- 大規模Flowへ対応できるよう再描画を抑制する。

## 3. 描画対象

- Canvas
- Lane
- Stage
- Node
- Link
- Comment
- Image Node
- Selection
- Drag Preview
- Grid
- Snap Guide
- MiniMap
- Validation Highlight

## 4. 全体構成

```text
Flow Structure
  ↓ Adapter
Drawing Model
  ↓ Vue Flow / Canvas Layer
User Operation
  ↓ Event Adapter
Editing Command
  ↓ Pinia Store
Save Request
  ↓ Backend
```

## 5. 描画Model

描画Modelは表示用であり、保存時はSSOTへ変換する。

- CanvasViewport
- DrawingLane
- DrawingStage
- DrawingNode
- DrawingLink
- DrawingComment
- SelectionState
- DragState

## 6. Vue Flow利用範囲

Vue Flowに任せるもの:

- Node表示
- Edge表示
- Zoom / Pan
- 基本Selection
- Node Drag
- Connection操作

独自実装するもの:

- Lane / Stage背景描画
- NodeTypeごとの見た目
- Link意味情報
- Snap / Grid補助
- Validation Highlight
- Property Panel連携
- SSOT Adapter

## 7. Command方針

描画操作はCommand化する。

- AddNodeCommand
- MoveNodeCommand
- ResizeNodeCommand
- AddLinkCommand
- UpdateLinkCommand
- DeleteElementCommand
- MoveCommentCommand

## 8. 禁止事項

- Vue Flow Nodeをそのまま保存する。
- Vue Flow EdgeをそのままLinkとして保存する。
- 選択状態をSSOTへ保存する。
- Drag中の一時座標を即時保存Requestへ入れる。
- Lane / Stageを単なる背景画像として扱う。

## 9. テスト観点

- SSOTから描画Modelを生成できる。
- 描画ModelからVue Flow Node / Edgeを生成できる。
- Node移動がMoveNodeCommandへ変換される。
- Link接続がAddLinkCommandへ変換される。
- SelectionがProperty Panelへ反映される。
- Vue Flow内部状態がSaveRequestへ混入しない。

## 10. 完了条件

- 描画エンジンの責務が定義されている。
- SSOTと描画Modelの境界が明確である。
- Vue Flow依存範囲と独自実装範囲が定義されている。
- 実装者が描画エンジン基盤を実装できる。
