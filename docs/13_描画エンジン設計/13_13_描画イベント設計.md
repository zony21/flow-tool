# 13_13_描画イベント設計

## 1. 目的

本書は、AI Flow Designerの描画イベント設計を定義する。

CanvasではMouse、Keyboard、Touch、Wheelなど多様なイベントが発生する。イベントをComponentへ散らすと保守が難しくなるため、イベント処理を責務ごとに整理する。

## 2. 基本方針

- Vue Flow eventを直接全体へ散らさない。
- Event AdapterでApplication Commandへ変換する。
- Keyboard Shortcutは集中管理する。
- Mouse操作とKeyboard操作の結果を同じCommandへ寄せる。
- 権限や編集ロック状態を考慮する。

## 3. Mouseイベント

対象:

- click
- double click
- pointer down
- pointer move
- pointer up
- wheel
- context menu

主な用途:

- 選択
- Drag
- Link接続
- Context Menu表示
- Pan / Zoom

## 4. Keyboardイベント

初期ショートカット:

| キー | 操作 |
| --- | --- |
| Ctrl + S | 保存 |
| Ctrl + Z | Undo |
| Ctrl + Shift + Z | Redo |
| Delete | 削除確認 |
| Esc | 選択解除 |
| Ctrl + A | 全選択 |
| Ctrl + C | Copy |
| Ctrl + V | Paste |
| Ctrl + D | Duplicate |

## 5. Wheelイベント

WheelはZoomまたはScroll / Panに利用する。

方針:

- modifierありWheelでZoom
- 通常WheelはCanvas ScrollまたはVue Flow標準挙動
- Zoom範囲を超えない

## 6. Context Menu

右クリックでContext Menuを表示する。

対象:

- Canvas背景
- Node
- Link
- Comment
- Lane
- Stage

表示項目は対象により切り替える。

## 7. Touch対応

初期ではPCブラウザを主対象とする。
ただし将来タブレット操作へ拡張できるよう、Pointer Eventを優先する。

## 8. Event Guard

以下の場合、編集系イベントを拒否する。

- Viewer権限
- 編集ロック未取得
- 保存中
- Dialog表示中
- Readonly Snapshot表示中

## 9. テスト観点

- Node clickで選択できる。
- Deleteで削除確認が出る。
- Ctrl + Sで保存が呼ばれる。
- Viewer権限でDragできない。
- Context Menuが対象別に表示される。

## 10. 完了条件

- Mouse / Keyboard / Wheel / Context Menuの方針が定義されている。
- EventからCommandへの流れが明確である。
- 実装者が描画イベント処理を実装できる。
