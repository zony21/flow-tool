# 13_02_座標系・レイアウト設計

## 1. 本書の目的

本書は、AI Flow Designer の座標系およびレイアウト設計を定義する。

本システムでは、図面そのものではなく構造化データを正とするため、座標情報も構造化データの一部として扱う。

座標は単なる表示位置ではなく、Lane、Stage、Node、Link、Comment、Export、PDF出力、Version比較に影響する重要情報である。

## 2. 座標系の種類

本システムでは以下の座標系を扱う。

|座標系|概要|
|---|---|
|Canvas座標|保存対象となる論理座標|
|Viewport座標|ズーム・パン後の表示座標|
|Screen座標|ブラウザ画面上の座標|
|DOM座標|HTML要素の座標|
|Export座標|PDF/SVG出力時の座標|

## 3. 正となる座標

正となる座標はCanvas座標である。

DBに保存する座標はCanvas座標とする。

Viewport座標、Screen座標、DOM座標は保存してはならない。

## 4. Canvas座標

Canvas座標は無限キャンバス上の論理座標である。

保持対象:

- Node.x
- Node.y
- Node.width
- Node.height
- Comment.x
- Comment.y
- Comment.width
- Comment.height
- Link.controlPoints
- Lane.order
- Stage.order

## 5. Node座標

Node座標は左上を基準とする。

```text
Node.x = 左上X
Node.y = 左上Y
Node.width = 幅
Node.height = 高さ
```

中心座標が必要な場合は派生計算する。

```text
centerX = x + width / 2
centerY = y + height / 2
```

## 6. Link座標

LinkはFROM NodeとTO Nodeを参照する。

DBには以下を保持する。

- fromNodeId
- toNodeId
- fromAnchor
- toAnchor
- controlPoints

開始点・終了点の絶対座標は原則保存しない。

理由:

- Node移動時に破綻する
- Nodeサイズ変更時に追従しない
- Anchorから再計算可能

## 7. Anchor

Nodeの接続点をAnchorと呼ぶ。

初期Anchor:

- Top
- Right
- Bottom
- Left
- Center

将来拡張:

- TopLeft
- TopRight
- BottomLeft
- BottomRight
- Custom

## 8. Anchor座標計算

例:

```text
Top:
  x = node.x + node.width / 2
  y = node.y

Right:
  x = node.x + node.width
  y = node.y + node.height / 2

Bottom:
  x = node.x + node.width / 2
  y = node.y + node.height

Left:
  x = node.x
  y = node.y + node.height / 2
```

## 9. Lane / Stage所属

Nodeは以下の2つを持つ。

- laneId
- stageId

座標から自動判定するだけでなく、所属情報を明示的に保持する。

理由:

- AIが担当と工程を確実に理解するため
- Export時に担当と工程を明確に出力するため
- レイアウト崩れ時にも意味が失われないため

## 10. Node移動時の所属更新

Node移動時、移動先座標が別Lane/Stage領域に入った場合、所属更新候補を表示する。

初期方針:

- ドロップ時に所属を自動更新
- ただしプロパティで手動変更可能
- 境界上の場合は元の所属を維持

## 11. Grid

キャンバスにはグリッドを表示する。

設定:

- gridSize
- showGrid
- snapToGrid

初期値:

```text
gridSize = 20
showGrid = true
snapToGrid = true
```

## 12. Snap

Snap対象:

- Node移動
- Nodeリサイズ
- Comment移動
- Commentリサイズ
- Link制御点移動

SnapはCanvas座標で行う。

## 13. Snap計算

```text
snappedX = round(x / gridSize) * gridSize
snappedY = round(y / gridSize) * gridSize
```

Altキー押下中は一時的にSnap無効とする案も将来検討する。

## 14. Zoom / Pan

ZoomとPanはViewport情報として保持する。

保存対象ではない。

Viewport:

- x
- y
- zoom

## 15. Fit View

Fit Viewは表示中のNode/Comment/Link範囲からViewportを計算する。

対象:

- Node
- Comment
- 必要に応じてLane/Stage範囲

## 16. Export座標

PDF出力ではCanvas座標をExport座標へ変換する。

変換時に以下を考慮する。

- 余白
- ページサイズ
- スケール
- 表示範囲
- Lane/Stageヘッダ
- 画像Node

## 17. Version比較と座標

Version比較では座標変更も差分として扱う。

差分例:

- Node移動
- Nodeリサイズ
- Link制御点変更
- Comment移動
- Lane順序変更
- Stage順序変更

## 18. 境界値

座標は負数を許可する。

理由:

- 無限キャンバスで左上方向に配置可能にするため
- インポートや貼付時に柔軟性を持たせるため

ただし、極端な値はValidation対象とする。

例:

```text
-1,000,000 ～ 1,000,000
```

## 19. サイズ制限

Nodeサイズ:

```text
minWidth = 40
minHeight = 30
maxWidth = 2000
maxHeight = 2000
```

Commentサイズ:

```text
minWidth = 80
minHeight = 40
```

## 20. 禁止事項

- DOM座標をDB保存する
- Zoom後座標をNode座標として保存する
- Lane/Stage所属を座標だけから判断する
- Link開始点・終了点の絶対座標だけを保存する
- PDF座標をFlow座標として逆流させる

## 21. テスト観点

- Node移動でCanvas座標が保存される
- Zoom中にNode移動してもCanvas座標が正しい
- Snap有効でgridSize単位になる
- Snap無効で任意座標になる
- Node移動でLane/Stage所属が更新される
- LinkがNode移動に追従する
- PDF出力で座標が崩れない
- Version比較で座標変更が検出される

## 22. 完了条件

- Canvas座標が保存の正として統一されている
- Viewport座標とDB座標が分離されている
- Node/Link/Commentの座標計算が定義されている
- Lane/Stage所属が明示管理されている
- Export時の座標変換方針が明確である
