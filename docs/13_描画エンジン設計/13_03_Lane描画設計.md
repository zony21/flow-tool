# 13_03_Lane描画設計

## 1. 目的

本書は、AI Flow DesignerのLane描画設計を定義する。

Laneは、工程を誰・どのシステム・どの設備・どの責務が担うかを表す重要要素である。単なる背景帯ではなく、AIが責務を理解するための構造情報である。

## 2. 基本方針

- LaneはFlow配下の構造要素として扱う。
- LaneはCanvas上で横または縦方向の領域として描画する。
- NodeはLaneIdを保持する。
- Node移動時に座標からLane候補を判定する。
- Lane責務はProperty Panelで編集する。
- Lane削除時は内包Nodeの扱いをユーザーに確認する。

## 3. Lane描画Model

```ts
interface DrawingLane {
  laneId: string;
  name: string;
  responsibility: string;
  orderNo: number;
  x: number;
  y: number;
  width: number;
  height: number;
  style?: LaneStyle;
  hasValidationError: boolean;
}
```

## 4. 表示項目

- Lane名
- 責務説明
- 背景色
- 境界線
- 折りたたみ状態
- エラーBadge

## 5. Lane配置

初期方針ではLaneは縦方向に積む。

```text
Lane A
--------------------------------
Lane B
--------------------------------
Lane C
--------------------------------
```

将来、横方向配置も検討できる設計にする。

## 6. Laneサイズ

Lane heightは以下で決定する。

- 最小高さ
- 内包Nodeの最大Y座標
- ユーザーリサイズ値

初期最小高さ:

```text
160px
```

## 7. Node所属判定

Node移動終了時に、Node中心点が含まれるLaneを候補にする。

処理:

1. Node drag stop。
2. Node centerを計算。
3. Lane boundsと比較。
4. laneIdを更新候補にする。
5. flowStoreへMoveNodeCommandを反映。

## 8. Lane並び替え

LaneはorderNoで並び順を持つ。

並び替え時:

- orderNoを再計算する。
- Lane boundsを再計算する。
- NodeのlaneIdは変更しない。
- Nodeの見た目上の位置は必要に応じて補正する。

## 9. Lane削除

Lane削除時はDialogで選択する。

選択肢:

- MoveNodes: 内包Nodeを別Laneへ移動
- DeleteWithNodes: 内包Nodeと関連Linkを削除
- Cancel

描画側では削除実行前に削除候補Laneと内包NodeをHighlightする。

## 10. Validation表示

Lane責務未入力、Lane名未入力などはLane HeaderにWarning表示する。

表示例:

- Warning Icon
- Tooltip
- Property Panelへの誘導

## 11. 禁止事項

- Laneを単なる背景画像として描画する。
- NodeのLane所属を座標だけで確定し、Propertyで変更できなくする。
- Lane削除時に内包Nodeを無確認で削除する。
- Lane責務を省略可能な飾り情報として扱う。

## 12. テスト観点

- LaneがorderNo順に描画される。
- Node移動でLane候補を判定できる。
- Lane削除時に内包Nodeが検出される。
- Lane責務未入力がWarning表示される。
- Lane並び替えでNodeのlaneIdが維持される。

## 13. 完了条件

- Lane描画Modelが定義されている。
- Laneの責務、配置、サイズ、削除方針が明確である。
- 実装者がLane描画を実装できる。
