# 13_03_Node描画設計

## 1. 本書の目的

本書は、AI Flow Designer のNode描画設計を定義する。

Nodeは、AIが理解可能なシステムフローを構成する最小単位であり、単なる図形ではなく意味を持つ構造化要素である。

Node描画では、表示形状、担当、工程、種別、説明、拡張JSON、接続点、選択状態、Validation状態を正しく表現する。

## 2. Nodeの基本構造

Nodeは以下の情報を持つ。

- nodeId
- flowId
- laneId
- stageId
- nodeTypeId
- displayName
- description
- x
- y
- width
- height
- shape
- extensionJson
- order
- isDeleted

## 3. NodeType

初期NodeType:

|種別|用途|
|---|---|
|開始|フロー開始|
|終了|フロー終了|
|処理|通常処理|
|判定|条件分岐|
|六角形|特殊処理・外部処理|
|吹き出し|補足・注釈的処理|
|画像|画像を含むNode|

NodeTypeは拡張可能である。

## 4. Nodeコンポーネント

Vueコンポーネント構成例:

```text
FlowNodeRenderer.vue
  ├─ StartNode.vue
  ├─ EndNode.vue
  ├─ ProcessNode.vue
  ├─ DecisionNode.vue
  ├─ HexagonNode.vue
  ├─ BalloonNode.vue
  └─ ImageNode.vue
```

共通処理はBaseNode.vueへ集約してもよい。

## 5. 共通表示要素

すべてのNodeに共通して表示する。

- 表示名
- Node形状
- 選択状態
- Hover状態
- Validation状態
- 接続Handle
- Resize Handle
- Lock状態
- コメント有無アイコン

## 6. Node表示名

displayNameはNode内中央または上部に表示する。

改行を許可する。

長文の場合:

- 最大行数を制限
- 省略表示
- Tooltipで全文表示

## 7. 接続Handle

各Nodeには接続用Handleを持つ。

初期Handle:

- Top
- Right
- Bottom
- Left

表示方針:

- 通常時は薄く表示または非表示
- Hover時に表示
- Link作成中は強調
- 接続不可時は無効表示

## 8. Node操作

Node操作はすべてCommand化する。

- AddNodeCommand
- MoveNodeCommand
- MoveNodesCommand
- ResizeNodeCommand
- UpdateNodePropertyCommand
- DeleteNodeCommand
- DuplicateNodeCommand

## 9. Node移動

Node移動はVue Flowのdragイベントを利用する。

移動中:

- 見た目を追従
- Snap計算
- Lane/Stage候補表示

移動完了:

- MoveNodeCommandを発行
- x/y更新
- laneId/stageId更新候補適用
- AutoSave対象に追加

## 10. 複数Node移動

複数選択中にNodeをドラッグした場合、選択中Node全体を移動する。

CommandはMoveNodesCommandとする。

保持:

- nodeId
- beforeX
- beforeY
- afterX
- afterY
- beforeLaneId
- afterLaneId
- beforeStageId
- afterStageId

## 11. Node追加

Node追加方法:

- 左パレットからドラッグ＆ドロップ
- キャンバスをダブルクリック
- ショートカット
- 貼付
- Template適用

Node追加時はAddNodeCommandを発行する。

## 12. Node削除

Deleteキーまたはメニューから削除する。

削除は論理削除扱いとする。

Node削除時に関連Linkも削除対象とする。

Node紐付けCommentは以下の方針で扱う。

- 初期方針: 同時削除
- 将来: 独立コメントへ変換選択可能

## 13. Nodeコピー

コピー対象:

- Node本体
- Nodeに紐付くComment
- 選択範囲内のLink

コピー時には内部Clipboard形式へ変換する。

貼付時には新しいGUIDを採番する。

## 14. Alt複製

Altを押しながらドラッグした場合、Node複製を行う。

複製対象:

- Node
- 選択範囲内Link
- 必要に応じてComment

複製後は新Nodeを選択状態にする。

## 15. 画像Node

画像NodeはimageFileIdをextensionJsonに保持する。

表示:

- 画像サムネイル
- 表示名
- 画像未設定時のプレースホルダ
- 読込失敗時のエラー表示

extensionJson例:

```json
{
  "imageFileId": "guid",
  "fitMode": "contain",
  "opacity": 1.0
}
```

## 16. 判定Node

判定Nodeは分岐を表す。

推奨表示:

- ひし形
- displayName
- 条件はLink側に表示

判定条件そのものはLink.conditionに保持する。

## 17. Node Validation表示

Validationエラーがある場合:

- 枠線をエラー表示
- エラーアイコンを表示
- Tooltipにエラー内容
- プロパティパネルで詳細表示

エラー例:

- displayName未入力
- laneId未設定
- stageId未設定
- nodeType不正
- 画像NodeなのにimageFileIdなし

## 18. Node ViewModel

Vue Flowへ渡すNode ViewModel例:

```ts
type FlowNodeViewModel = {
  id: string
  type: string
  position: { x: number; y: number }
  dimensions: { width: number; height: number }
  data: {
    nodeId: string
    displayName: string
    nodeType: string
    laneId: string
    stageId: string
    validationErrors: string[]
    readonly: boolean
  }
}
```

## 19. 性能注意点

1000Nodeを想定し、以下を避ける。

- 全Nodeのdeep watch
- Nodeごとの重いcomputed
- 不要な画像再読込
- HoverだけでStore全体更新
- 大量Tooltipの常時生成

## 20. 禁止事項

- displayNameだけを正とする
- NodeTypeをCSSクラス名だけで判断する
- Node移動中に毎回API保存する
- Node削除で物理削除する
- 画像ファイルをNode内にBase64保存する
- Vue Flow NodeをDB Entityとして扱う

## 21. テスト観点

- NodeTypeごとに正しい形状で表示される
- Node移動で座標更新される
- Nodeリサイズでサイズ更新される
- Node削除で関連Linkが削除対象になる
- 画像Nodeが画像を表示する
- Validationエラーが表示される
- 読み取り専用で編集できない
- 1000Nodeで操作できる

## 22. 完了条件

- 全初期NodeTypeが描画できる
- Node操作がCommand化されている
- Nodeの意味情報が保持される
- Lane/Stage所属が失われない
- 画像Nodeが画像管理と連携できる
