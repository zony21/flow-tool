# 13_20_描画エンジン設計まとめ

## 1. 目的

本書は、13_描画エンジン設計の総まとめである。

描画エンジンは、SSOTをCanvas上で編集可能にするFrontend中核機能である。Vue Flowを利用しつつ、SSOT、描画Model、Command、Store、Property Panelを明確に分離する。

## 2. 描画エンジンの責務

- Canvas表示
- Lane / Stage描画
- Node描画
- Link描画
- Comment描画
- Selection
- Drag & Drop
- Snap / Grid
- MiniMap
- Clipboard
- レイアウト補助
- 描画イベント処理
- 描画性能最適化

## 3. 正ではないもの

以下はSSOTではない。

- Vue Flow Node
- Vue Flow Edge
- selected状態
- hovered状態
- dragging状態
- viewport
- zoom
- pan
- grid表示状態

## 4. 実装順序

推奨実装順序:

1. Canvas基盤
2. SSOT to DrawingModel Adapter
3. DrawingModel to Vue Flow Adapter
4. Node描画
5. Link描画
6. Lane / Stage背景
7. Selection
8. Drag & Drop
9. Snap / Grid
10. Comment描画
11. Image Node
12. Clipboard
13. MiniMap
14. レンダリング最適化
15. テスト

## 5. 実装チェックリスト

- [ ] Vue Flow Nodeを保存しない。
- [ ] Vue Flow Edgeを保存しない。
- [ ] Node移動はCommand化する。
- [ ] Link接続はCommand化する。
- [ ] SelectionはSSOTへ入れない。
- [ ] Lane / Stageを構造要素として扱う。
- [ ] Snap / Gridを表示補助として扱う。
- [ ] Copy / Paste時にID再採番する。
- [ ] Drag中に保存Requestを作らない。
- [ ] 大規模Flowで再描画を抑制する。

## 7. 完了条件

13_描画エンジン設計は、以下を満たすことで完了とする。

- Canvas、Lane、Stage、Node、Link、Commentが定義されている。
- Selection、DragDrop、SnapGrid、ZIndexが定義されている。
- Vue Flow連携、イベント、Clipboard、MiniMapが定義されている。
- レイアウト補助、性能、テストが定義されている。
- 実装者が本章を読んで描画エンジンを実装できる。
