# 13_01_描画エンジン全体設計

## 1. 本書の目的

本書は、AI Flow Designer の描画エンジン全体設計を定義する。

AI Flow Designer は、ExcelやVisioのような単なる図形エディタではなく、構造化データをSingle Source of Truthとして保持し、その構造化データを視覚的に編集するWebアプリケーションである。

したがって、描画エンジンの目的は「きれいな図を描くこと」だけではない。

描画エンジンは、以下を同時に満たす必要がある。

- 構造化データを正確に画面へ反映する
- ユーザー操作を構造化データへ正しく変換する
- Node / Link / Lane / Stage / Comment の関係を破綻させない
- AIが理解可能な意味情報を保持したまま編集できる
- 大量Node / 大量Linkでも操作可能な性能を確保する
- Undo/Redo、AutoSave、Version、Exportと整合する

## 2. 基本方針

描画エンジンは Vue Flow を基盤として実装する。

ただし、Vue Flow の内部状態をそのままSingle Source of Truthにはしない。

正となるデータは、アプリケーション側のFlow構造データである。

Vue Flow は、構造化データを描画・操作するためのView Engineとして利用する。

## 3. レイヤ構成

```text
Structured Flow Data
  ↓
Editor State
  ↓
View Model Converter
  ↓
Vue Flow Nodes / Edges
  ↓
Canvas Rendering
  ↓
User Interaction
  ↓
Command
  ↓
Structured Flow Data
```

## 4. Single Source of Truth

正となるのは以下である。

- Project
- Flow
- Lane
- Stage
- Node
- Link
- Comment
- Layout
- Version Snapshot

Vue Flow の nodes / edges は派生データである。

## 5. 派生データの扱い

Vue Flow用のNode/Edgeは、構造化データから生成する。

生成例:

```text
FLOW_NODE -> Vue Flow Node
FLOW_LINK -> Vue Flow Edge
FLOW_COMMENT -> Comment Overlay
LANE/STAGE -> Background Layer
```

Vue Flow上で操作が発生した場合は、直接DB保存用DTOを編集するのではなく、Commandへ変換してからEditor Stateを更新する。

## 6. 描画対象

描画エンジンは以下を描画対象とする。

- Lane
- Stage
- Node
- Link
- Comment
- Selection
- Grid
- Snap Guide
- Resize Handle
- Link Control Point
- Hover Highlight
- Lock Overlay
- Validation Error Marker

## 7. Canvas構成

キャンバスは以下の視覚レイヤで構成する。

```text
最背面
  Grid Layer
  Stage Background Layer
  Lane Background Layer
  Link Layer
  Node Layer
  Comment Layer
  Selection Layer
  Interaction Handle Layer
  Overlay Layer
最前面
```

## 8. レイヤ責務

|レイヤ|責務|
|---|---|
|Grid Layer|グリッド表示|
|Stage Background Layer|工程背景・区切り|
|Lane Background Layer|担当レーン背景・区切り|
|Link Layer|接続線描画|
|Node Layer|ノード描画|
|Comment Layer|コメント描画|
|Selection Layer|選択枠・範囲選択|
|Interaction Handle Layer|リサイズ・接続点・制御点|
|Overlay Layer|ロック・エラー・読み込み中表示|

## 9. Vue Flow利用範囲

Vue Flowを利用するもの:

- Node配置
- Edge描画
- Zoom
- Pan
- Selection
- Drag
- Connection
- Fit View
- Viewport管理

アプリ側で独自管理するもの:

- Lane / Stage 背景
- Node種別の意味
- Linkの通信情報
- Commentの独立/紐付け
- Undo/Redo Command
- AutoSave差分
- Validation
- Export用構造

## 10. 座標系

座標系はキャンバス座標を正とする。

保持する座標:

- Node X
- Node Y
- Node Width
- Node Height
- Comment X
- Comment Y
- Link Control Points
- Lane表示順
- Stage表示順

画面座標、CSS座標、Viewport座標は派生値である。

## 11. 操作イベント

主要操作イベント:

- Node追加
- Node移動
- Nodeリサイズ
- Node削除
- Link作成
- Link再接続
- Linkラベル編集
- Comment追加
- Comment移動
- Lane追加
- Lane削除
- Stage追加
- Stage削除
- 複数選択
- コピー
- 貼付
- Alt複製

すべてCommand化する。

## 12. 状態管理

描画状態はPiniaで管理する。

主なStore:

- flowEditorStore
- selectionStore
- viewportStore
- commandHistoryStore
- editorLockStore
- validationStore

## 13. 読み取り専用モード

Viewerまたはロック未取得時は読み取り専用とする。

読み取り専用で可能:

- Pan
- Zoom
- Fit
- Node選択
- Link選択
- プロパティ閲覧
- Export

不可:

- Node移動
- Node追加
- Link作成
- 削除
- 編集
- AutoSave

## 14. 性能目標

|対象|目標|
|---|---:|
|1000ノード初期描画|3秒以内|
|10000リンク初期描画|5秒以内|
|Nodeドラッグ追従|体感遅延なし|
|Zoom/Pan|体感遅延なし|
|選択切替|200ms以内|
|プロパティ切替|200ms以内|

## 15. 禁止事項

- Vue Flowの内部状態をDB保存の正とする
- Node表示名だけで意味を判断する
- Link線だけで通信情報を表現する
- DOM座標をそのままDBへ保存する
- Canvas全体を画像として正とする
- Undo/RedoをVue Flow内部履歴に依存する
- Lane/Stage削除でNodeを暗黙削除する

## 16. 完了条件

- Vue Flowと構造化データの責務が分離されている
- Node/Link/Lane/Stage/Commentを描画できる
- 操作がCommand化される
- 描画状態がPiniaで管理される
- Export/Version/AutoSaveと整合する
