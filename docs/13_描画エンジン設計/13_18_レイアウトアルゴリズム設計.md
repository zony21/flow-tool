# 13_18_レイアウトアルゴリズム設計

## 1. 目的

本書は、AI Flow Designerのレイアウトアルゴリズム設計を定義する。

レイアウトアルゴリズムは、NodeやCommentの配置を補助し、見やすいFlowを作成するための機能である。初期実装では自動配置を最小限にし、整列補助を中心にする。

## 2. 基本方針

- 初期は完全自動レイアウトを必須にしない。
- 整列、等間隔、Lane / Stage内整列を優先する。
- 自動レイアウト結果はCommandとして適用する。
- ユーザー確認なしに配置を大幅変更しない。
- SSOTのNode位置を更新する場合はUndo対象にする。

## 3. 初期対応

- 左揃え
- 右揃え
- 上揃え
- 下揃え
- 水平中央揃え
- 垂直中央揃え
- 水平方向等間隔
- 垂直方向等間隔

## 4. Lane / Stage内整列

対象LaneまたはStage内のNodeを整列できるようにする。

例:

- Lane内で上から順に整列
- Stage内で左から順に整列
- Lane × Stageセル内で中央配置

## 5. Auto Layout将来方針

将来、以下を検討する。

- DAG風自動配置
- Mermaidからの配置補助
- Link交差を減らす配置
- Lane / Stage制約付き配置
- AI提案による配置改善

## 6. Command

レイアウト変更はApplyLayoutCommandとして扱う。

保持内容:

- 対象Node一覧
- 変更前位置
- 変更後位置
- layoutType

## 7. 禁止事項

- ユーザー確認なしに全Nodeを大幅移動する。
- Lane / Stage所属を無視して配置する。
- 自動配置結果をUndo不可にする。
- Linkの意味を無視して接続順を変更する。

## 8. テスト観点

- 複数Nodeを左揃えできる。
- 等間隔配置ができる。
- Lane内整列ができる。
- ApplyLayoutCommandでUndoできる。

## 9. 完了条件

- 初期整列機能と将来Auto Layout方針が定義されている。
- Command化とUndo対象化が定義されている。
- 実装者が整列補助を実装できる。
