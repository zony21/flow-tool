# 13_06_Link描画設計

## 1. 目的

本書は、AI Flow DesignerのLink描画設計を定義する。

Linkは単なる線ではなく、Node間の処理順序、条件、データ受け渡し、通信の意味を表す構造要素である。描画上はEdgeとして表示するが、正となるのはSSOTのFlowLinkである。

## 2. 基本方針

- 初期Link描画はBezierとする。
- 将来Straight / Step / SmoothStepへ切り替え可能にする。
- LinkはSourceNodeIdとTargetNodeIdを必ず持つ。
- Loopは許可する。
- LinkのLabel、Condition、DataName、CommunicationTypeを表示できるようにする。
- Vue Flow Edgeは表示用Modelとして扱う。

## 3. Link描画Model

```ts
interface DrawingLink {
  linkId: string;
  sourceNodeId: string;
  targetNodeId: string;
  label?: string;
  condition?: string;
  dataName?: string;
  communicationType?: string;
  linkType: 'bezier' | 'straight' | 'step' | 'smoothstep';
  hasValidationError: boolean;
  isSelected: boolean;
}
```

## 4. 表示項目

- Link線
- 矢印
- Label
- Condition
- Error Badge
- Selection Highlight

## 5. 接続Handle

Nodeには接続Handleを持たせる。

初期Handle:

- top
- right
- bottom
- left

Vue FlowのHandleを利用するが、接続可否はアプリ側で判定する。

## 6. 接続ルール

許可:

- process -> process
- process -> decision
- decision -> process
- decision -> end
- any -> any の基本接続
- loop接続

警告:

- 自己参照Link
- decisionからConditionなしLink
- startへ戻るLink

禁止:

- source未指定
- target未指定
- 存在しないNodeへの接続
- 削除済Nodeへの接続

## 7. Link追加フロー

```text
User connects handle
  ↓
Vue Flow connect event
  ↓
validate source / target
  ↓
AddLinkCommand
  ↓
flowStore.addLink
  ↓
selectionStore.selectLink
  ↓
PropertyPanel shows LinkPropertyForm
```

## 8. Link更新

Property Panelで以下を編集する。

- label
- condition
- dataName
- communicationType
- linkType

編集時はUpdateLinkCommandを生成する。

## 9. Link削除

Link削除はDeleteElementCommandで扱う。

削除はFrontend上ではeditingFlowから除外またはdeleted扱いにする。
保存時にBackendへ論理削除として送る。

## 10. Validation表示

Linkに構造エラーがある場合、線色、Badge、Tooltipで表示する。

例:

- sourceNodeが存在しない
- targetNodeが存在しない
- decision分岐条件なし

## 11. 禁止事項

- Vue Flow Edgeをそのまま保存する。
- LinkをLabelだけの線として扱う。
- decision Nodeの条件Linkを無視する。
- Loopを描画側で禁止する。

## 12. テスト観点

- Bezier Linkを描画できる。
- Link接続がAddLinkCommandへ変換される。
- decisionからのLinkにCondition不足Warningを表示できる。
- Loop Linkを描画できる。
- Edge選択でProperty PanelがLink編集になる。

## 13. 完了条件

- Link描画Model、接続ルール、追加・更新・削除フローが定義されている。
- Bezierを初期値としつつ将来形式変更できる。
- 実装者がLink描画を実装できる。
