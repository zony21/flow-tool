# 13_14_コメント描画設計

## 1. 目的

本書は、AI Flow DesignerのComment描画設計を定義する。

Commentは、通常の補足、レビューコメント、AI専用メモ、警告などをCanvas上に表示する要素である。Commentは単なる注釈ではなく、AI理解補助やレビュー記録として利用される。

## 2. 基本方針

- 独立CommentとNode紐付けCommentに対応する。
- CommentはFlowCommentとしてSSOTに保持する。
- Comment位置はCanvas Coordinateで保持する。
- AIへ出力するかどうかをaiVisibleで制御する。
- CommentTypeにより見た目を切り替える。

## 3. Comment描画Model

```ts
interface DrawingComment {
  commentId: string;
  targetType: 'flow' | 'lane' | 'stage' | 'node' | 'link' | 'none';
  targetId?: string;
  commentType: 'normal' | 'warning' | 'aiMemo' | 'reviewNote';
  body: string;
  x: number;
  y: number;
  aiVisible: boolean;
  isSelected: boolean;
}
```

## 4. 表示種類

| 種類 | 表示 |
| --- | --- |
| normal | 通常Comment |
| warning | Warning表示 |
| aiMemo | AI用メモ表示 |
| reviewNote | レビュー指摘表示 |

## 5. 独立Comment

Canvas上の任意位置に配置できる。

操作:

- 作成
- 移動
- 編集
- 削除

移動時はMoveCommentCommandを生成する。

## 6. Node紐付けComment

Nodeに紐付くCommentは対象Node近くに表示する。

方針:

- Node移動時に追従する。
- 必要に応じてOffsetを保持する。
- Property Panelから紐付け変更できる。

## 7. AI専用メモ

AI専用メモはaiVisibleでExport対象を制御する。

表示は控えめにするが、存在はBadgeで分かるようにする。

## 8. Validation表示

空本文、削除済Target参照などをWarning表示する。

## 9. 禁止事項

- Commentを単なる画面メモとしてSSOTから外す。
- AI専用メモと通常Commentを混同する。
- Node紐付けCommentをNode削除時に放置する。

## 10. テスト観点

- 独立Commentを描画できる。
- Node移動時に紐付けCommentが追従する。
- aiVisibleでExport対象を制御できる。
- Comment選択でProperty Panelが切り替わる。

## 11. 完了条件

- Comment描画Modelと表示種類が定義されている。
- 独立Comment、Node紐付けComment、AIメモの方針が明確である。
- 実装者がComment描画を実装できる。
