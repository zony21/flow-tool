# 13_06_選択・範囲選択設計

## 1. 本書の目的

本書は、AI Flow Designer の選択・範囲選択設計を定義する。

フローエディタでは、Node、Link、Lane、Stage、Commentを選択し、プロパティ編集、コピー、削除、移動、Undo/Redoの対象とする。

選択状態は画面操作の基盤であり、プロパティパネル、ショートカット、Command生成に影響する。

## 2. 選択対象

選択可能な対象:

- Node
- Link
- Lane
- Stage
- Comment
- 複数Node
- 複数Link
- Node + Link + Comment 混在選択

Lane/Stageは背景ヘッダクリックで選択する。

## 3. Selection State

選択状態はselectionStoreで管理する。

保持例:

```ts
type SelectionState = {
  selectedNodeIds: string[]
  selectedLinkIds: string[]
  selectedLaneIds: string[]
  selectedStageIds: string[]
  selectedCommentIds: string[]
  primarySelection?: SelectionTarget
}
```

## 4. Primary Selection

Primary Selectionはプロパティパネルに表示する主対象である。

決定ルール:

- 最後にクリックした対象
- 複数選択時も最後に選ばれたもの
- 範囲選択時は最初または最後の対象

## 5. 単一選択

クリックで単一選択する。

挙動:

- 既存選択を解除
- クリック対象を選択
- primarySelectionを更新
- プロパティパネルを切替

## 6. 複数選択

CtrlまたはShiftクリックで複数選択する。

挙動:

- 対象が未選択なら追加
- 対象が選択済なら解除
- primarySelectionを更新

## 7. 範囲選択

キャンバス空白からドラッグして矩形範囲を作る。

範囲内の対象を選択する。

初期対象:

- Node
- Comment

将来:

- Linkも範囲選択対象に含める

## 8. Link範囲選択

Linkは曲線であるため、矩形との交差判定が難しい。

初期実装では、Linkのラベルまたは中点が範囲内にある場合に選択対象とする。

将来、曲線交差判定を実装する。

## 9. 全選択

Ctrl+Aで全選択する。

対象:

- 表示中FlowのNode
- Link
- Comment

Lane/Stageは初期では全選択対象に含めない。

## 10. 選択解除

以下で選択解除する。

- キャンバス空白クリック
- Escキー
- 別Flow読込
- 削除後
- 読み取り専用切替時

## 11. 選択表示

選択中対象は視覚的に強調する。

Node:

- 枠線強調
- Resize Handle表示

Link:

- 線強調
- 制御点表示

Comment:

- 枠線強調

Lane/Stage:

- ヘッダ強調

## 12. 複数選択枠

複数Node/Comment選択時は全体のBounding Boxを表示する。

Bounding Boxは以下から計算する。

- selected nodes
- selected comments

Linkのみ選択時はBounding Boxを表示しない。

## 13. 複数移動

複数Node/Comment選択時、ドラッグでまとめて移動する。

Command:

```text
MoveSelectionCommand
```

保持:

- 対象ID
- before座標
- after座標

## 14. プロパティパネル連携

選択状態に応じて右プロパティを切り替える。

|選択状態|表示|
|---|---|
|Node単一|Nodeプロパティ|
|Link単一|Linkプロパティ|
|Lane単一|Laneプロパティ|
|Stage単一|Stageプロパティ|
|Comment単一|Commentプロパティ|
|複数|一括編集プロパティ|
|なし|Flowプロパティ|

## 15. 一括編集

複数選択時に可能な一括編集:

- Node色
- Nodeサイズ
- Lane変更
- Stage変更
- 削除
- コピー
- 移動

初期実装では削除・コピー・移動を優先する。

## 16. 削除

Deleteキーで選択対象を削除する。

削除順:

1. Link
2. Comment
3. Node
4. Lane/Stage

Node削除時は関連Linkも削除対象とする。

Lane/Stage削除は確認ダイアログを必須とする。

## 17. コピー

Ctrl+Cで選択対象をコピーする。

コピー対象:

- Node
- Link
- Comment

Lane/Stageは初期コピー対象外。

選択範囲内Node同士をつなぐLinkのみコピーする。

## 18. 貼付

Ctrl+Vで貼付する。

貼付時:

- 新GUID採番
- 座標を少しずらす
- Linkのfrom/toを新Nodeへ張り替える
- 選択状態を貼付後対象へ変更

## 19. Alt複製

Altドラッグで選択対象を複製する。

挙動はコピー+貼付+移動に近い。

## 20. F2編集

F2でprimarySelectionの表示名編集を開始する。

対象:

- Node
- Linkラベル
- Lane名
- Stage名
- Comment本文

## 21. 読み取り専用時

読み取り専用でも選択は可能。

可能:

- 選択
- プロパティ閲覧
- コピー
- Export

不可:

- 移動
- 編集
- 削除
- 貼付
- 複製

## 22. 選択とUndo/Redo

選択状態そのものはUndo/Redo対象にしない。

ただし、Command実行後に選択状態を更新することは許可する。

例:

- Node追加後、そのNodeを選択
- 貼付後、貼付Nodeを選択
- 削除後、選択解除

## 23. 選択とAutoSave

選択状態はAutoSave対象外である。

保存対象ではない。

## 24. 選択とVersion

選択状態はVersion Snapshotへ含めない。

## 25. 性能注意点

大量Nodeで範囲選択する場合、全NodeのDOM判定は重い。

対策:

- NodeのCanvas座標で矩形判定
- DOM getBoundingClientRectを多用しない
- Link範囲選択は簡易判定から開始

## 26. 禁止事項

- 選択状態をDB保存する
- DOM状態だけで選択を管理する
- 複数選択中にprimarySelectionが不明になる
- Viewerで削除可能にする
- Lane/Stage削除を確認なしで実行する

## 27. テスト観点

- 単一選択できる
- Ctrlクリックで複数選択できる
- 範囲選択できる
- Ctrl+Aで全選択できる
- Escで解除できる
- 複数移動できる
- Deleteで削除できる
- コピー/貼付でID再採番される
- 読み取り専用では編集できない
- プロパティが選択対象に応じて切り替わる

## 28. 完了条件

- 選択状態がStoreで一元管理されている
- プロパティパネルと連携できる
- 複数選択操作がCommand化される
- 保存対象と非保存対象が分離されている
