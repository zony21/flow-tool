# 13_05_Node描画設計

## 1. 目的

本書は、AI Flow DesignerのNode描画設計を定義する。

NodeはFlow上の処理、判定、開始、終了、補足、画像などを表す主要要素である。Nodeは単なる図形ではなく、AIが処理内容・責務・入出力・条件を理解するための構造情報を持つ。

## 2. 基本方針

- NodeはSSOTのFlowNodeから生成する。
- Vue Flow Nodeは表示用Modelとして扱う。
- NodeIdはSSOTとVue Flow表示Modelで一致させる。
- NodeTypeごとに描画Componentを分ける。
- Node移動・リサイズはCommand化する。
- Nodeの詳細情報はProperty Panelで編集する。

## 3. Node描画Model

```ts
interface DrawingNode {
  nodeId: string;
  nodeType: FlowNodeType;
  title: string;
  description?: string;
  laneId?: string;
  stageId?: string;
  x: number;
  y: number;
  width: number;
  height: number;
  hasValidationError: boolean;
  hasAiNote: boolean;
  isSelected: boolean;
}
```

## 4. 初期NodeType

| NodeType | 用途 | Component |
| --- | --- | --- |
| start | 開始 | StartNode.vue |
| end | 終了 | EndNode.vue |
| process | 通常処理 | ProcessNode.vue |
| decision | 条件分岐 | DecisionNode.vue |
| hexagon | 外部・特殊処理 | HexagonNode.vue |
| balloon | 補足 | BalloonNode.vue |
| image | 画像 | ImageNode.vue |

## 5. Node表示項目

- Title
- NodeType Icon
- Validation Badge
- AI Note Badge
- Selected Border
- Resize Handle
- Link Handle

## 6. Node位置

Node positionはCanvas Coordinateで保持する。

保存対象:

- x
- y
- width
- height

保存しないもの:

- selected
- hovered
- dragging
- temporary position

## 7. Node移動

Node移動はdrag中とdrag完了で分ける。

Drag中:

- Vue Flow内部状態で表示を動かす。
- flowStoreへ都度保存反映しない。

Drag完了:

- MoveNodeCommandを生成する。
- x / yを更新する。
- Lane / Stage候補を判定する。
- undoRedoStoreへ登録する。

## 8. Nodeリサイズ

NodeリサイズはResizeNodeCommandとして扱う。

制約:

- 最小幅
- 最小高さ
- NodeType別の推奨サイズ

初期値:

- process: 180 x 80
- decision: 160 x 120
- image: 240 x 160

## 9. Lane / Stage所属

NodeはLaneIdとStageIdを保持する。

所属更新方法:

- Drag完了時に座標から候補判定
- Property Panelで明示変更

明示変更を優先する。

## 10. Validation表示

NodeにValidation Errorがある場合、Node上にBadgeを表示する。

例:

- Node名未入力
- Lane未設定
- decision条件不足
- imageFileId不足

BadgeクリックでProperty Panelを対象Nodeへ切り替える。

## 11. NodeType拡張

NodeTypeは拡張可能とする。

拡張時に必要なもの:

- NodeType定義
- Palette定義
- Canvas Component
- Property Form
- Validation Rule
- Export変換

描画側ではNodeRegistryでComponentを解決する。

## 12. 禁止事項

- Node Component内でAPI通信する。
- Node Component内で保存処理を行う。
- Vue Flow Node dataをSSOTとして扱う。
- selectedやdraggingを保存Requestへ含める。
- NodeTypeをif文だけで増やし続ける。

## 13. テスト観点

- NodeTypeごとに正しいComponentで描画される。
- Node移動がMoveNodeCommandへ変換される。
- NodeリサイズがResizeNodeCommandへ変換される。
- selected状態が保存Requestへ混入しない。
- Validation Badgeが表示される。
- NodeRegistryで拡張Nodeを描画できる。

## 14. 完了条件

- Node描画Modelが定義されている。
- NodeType別描画Component方針が定義されている。
- 移動、リサイズ、所属、Validation表示が定義されている。
- 実装者がNode描画を実装できる。
