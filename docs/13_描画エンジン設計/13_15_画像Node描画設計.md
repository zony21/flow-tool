# 13_15_画像Node描画設計

## 1. 目的

本書は、AI Flow Designerの画像Node描画設計を定義する。

画像Nodeは、画面キャプチャ、設備画像、処理説明図、参考図などをFlow内に配置するためのNodeである。画像実体はBackendの画像管理機能で扱い、Canvasでは参照と表示を行う。

## 2. 基本方針

- image NodeはFlowNodeの一種として扱う。
- imageFileIdでBackend画像メタデータを参照する。
- 物理パスをFrontendへ持たせない。
- 画像表示URLはBackend API経由とする。
- Resize可能にする。
- 代替テキストを持たせる。

## 3. 描画Model

```ts
interface DrawingImageNode extends DrawingNode {
  imageFileId?: string;
  imageUrl?: string;
  altText?: string;
  objectFit: 'contain' | 'cover';
}
```

## 4. 表示内容

- 画像サムネイル
- Node title
- altText任意表示
- Error Badge
- Resize Handle

## 5. 画像読込

画像URLは以下を利用する。

```text
/api/images/{imageFileId}
```

認証・権限確認はBackendで行う。

## 6. Resize

画像NodeはResize可能とする。

制約:

- 最小幅 120
- 最小高さ 80
- アスペクト比維持はOption

Resize完了時にResizeNodeCommandを生成する。

## 7. Error表示

以下の場合はError表示する。

- imageFileId未設定
- 画像取得失敗
- 権限不足
- 削除済画像参照

## 8. Property Panel連携

編集項目:

- title
- imageFileId
- altText
- objectFit
- width
- height
- AI専用メモ

## 9. 禁止事項

- 画像物理パスをFrontendで保持する。
- Base64をFlow構造へ直接埋め込む。
- 画像NodeをComment扱いにする。
- 画像取得失敗を無表示にする。

## 10. テスト観点

- imageFileIdから画像を表示できる。
- ResizeでNode sizeが更新される。
- 画像取得失敗時にError表示される。
- 物理パスが表示されない。

## 11. 完了条件

- 画像Nodeの描画、参照、Resize、Error表示が定義されている。
- Backend画像管理との境界が明確である。
- 実装者が画像Nodeを実装できる。
