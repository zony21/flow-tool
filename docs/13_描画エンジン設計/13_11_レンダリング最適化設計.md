# 13_11_レンダリング最適化設計

## 1. 目的

本書は、AI Flow Designerのレンダリング最適化設計を定義する。

AI Flow Designerは、将来的に数百〜数千Node規模のFlowを扱う可能性がある。Canvas操作が重くなると設計ツールとして成立しないため、描画更新、Store更新、Adapter変換、Validation表示を最適化する。

## 2. 基本方針

- Flow全体の再描画を避ける。
- Node単位、Link単位の差分更新を優先する。
- Drag中はStore更新を抑制する。
- 大量Nodeでは表示範囲外の描画を軽量化する。
- Property Panel更新は選択中要素に限定する。
- Export処理をCanvas描画に依存させない。

## 3. 最適化対象

- Flow to Drawing Model変換
- Drawing Model to Vue Flow変換
- Node Component描画
- Link Component描画
- Validation Highlight
- AI Review Highlight
- Selection Highlight
- MiniMap

## 4. Drag中の最適化

Drag中はVue Flow内部表示を利用し、flowStoreへの確定更新はDrag完了時のみ行う。

禁止:

```text
mousemoveごとにflowStore全体更新
```

推奨:

```text
dragStopでMoveNodeCommandを1回生成
```

## 5. Adapter最適化

Flow全体変換は以下に限定する。

- 初回Flow読込
- 保存成功後のBackend Response反映
- Snapshot復元
- Template適用

通常編集では対象要素だけを更新する。

## 6. Component最適化

Node Componentは表示に集中する。

禁止:

- API通信
- 重い計算
- Flow全体props受け取り
- deep watch

推奨:

- 必要最小限のdataのみ渡す
- selected / error / labelなど状態を分離
- memo化可能な値はcomputedへ集約

## 7. 大規模Flow対応

将来対応:

- viewport外Nodeの軽量表示
- Link簡略表示モード
- Comment折りたたみ
- Lane / Stage折りたたみ
- MiniMap簡略化
- 検索結果中心表示

## 8. Validation表示最適化

Validation結果はキャッシュし、入力1文字ごとにFlow全体Validationを走らせない。

- 項目単位Validationは即時
- Flow全体Validationは保存前、Export前、AIレビュー前
- Canvas BadgeはValidation結果を参照

## 9. 性能目標

初期目標:

- 200Node規模で快適に操作できる
- 1000Node規模で表示・編集可能
- Node移動で大きな遅延がない
- Property Panel切替が即時に近い

## 10. テスト観点

- Node移動で全Nodeが不要に再描画されない。
- Drag中にSaveRequestが生成されない。
- Flow詳細読込後に描画Modelを生成できる。
- 1000Node規模で表示できる。
- Validation Badge更新が局所更新になる。

## 11. 完了条件

- 再描画抑制方針が定義されている。
- Drag中と確定更新の責務が分離されている。
- 大規模Flow対応方針が定義されている。
- 実装者が性能劣化を避けた描画実装を行える。
