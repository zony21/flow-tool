# 13_07_描画性能設計

## 1. 本書の目的

本書は、AI Flow Designer の描画性能設計を定義する。

本システムでは、性能試験対象として1000ノード、10000リンク、Undo1000回を想定する。描画エンジンは、大量要素を扱っても実用可能な操作感を維持する必要がある。

## 2. 性能目標

|処理|目標|
|---|---:|
|初期Flow表示|5秒以内|
|1000Node表示|3秒以内|
|10000Link表示|5秒以内|
|Nodeドラッグ|体感遅延なし|
|Zoom/Pan|体感遅延なし|
|選択切替|200ms以内|
|プロパティ表示|200ms以内|
|Undo/Redo反映|500ms以内|

## 3. 基本方針

- Vueのリアクティブ対象を増やしすぎない
- deep watchを避ける
- Node/Linkの全再生成を避ける
- Hoverなど一時状態でStore全体を更新しない
- DOM数を抑える
- Tooltipや詳細表示を必要時のみ生成する
- 画像Nodeは遅延読込する
- Linkラベル表示を簡略化できるようにする

## 4. Store設計

巨大なFlow全体を1つの巨大reactive objectにしない。

推奨:

- nodesById
- linksById
- lanesById
- stagesById
- commentsById
- selectedIds
- viewport

ID Map形式で管理する。

## 5. 配列生成

Vue Flowへ渡すnodes/edges配列は必要時に生成する。

ただし、毎回全要素を新しいObjectで作ると再描画が増える。

対策:

- 更新対象だけ差し替え
- computed依存を分割
- ViewModel Cacheを導入可能にする

## 6. ViewModel Cache

Node/Link ViewModelをキャッシュする。

キャッシュキー:

- id
- revision
- selectionVersion
- validationVersion
- readonlyVersion

Nodeの表示に関係ない変更では再生成しない。

## 7. Hover状態

Hover状態は局所stateで管理する。

Store全体にHoverを書き込むと大量再描画の原因になる。

初期方針:

- Nodeコンポーネント内部state
- 必要な場合のみselectionStoreへ反映

## 8. Linkラベル最適化

10000LinkすべてにラベルDOMを表示すると重い。

対策:

- Zoom率が低い場合はラベル非表示
- 選択中Linkのみ詳細ラベル表示
- Hover中LinkのみTooltip表示
- communicationTypeアイコンを簡略化

## 9. 画像Node最適化

画像Nodeは遅延読込する。

対策:

- サムネイル利用
- 表示範囲内のみ読込
- 画像サイズを制限
- Base64埋め込み禁止
- キャッシュ利用

## 10. Zoom Level別表示

Zoom率に応じて表示情報を切り替える。

例:

|Zoom|表示|
|---|---|
|低|Node名のみ、Linkラベル非表示|
|中|Node名、主要Linkラベル|
|高|詳細、アイコン、エラー、Handle|

## 11. Drag中最適化

Nodeドラッグ中は重い処理を行わない。

禁止:

- 毎mousemoveでAutoSave
- 毎mousemoveで全Validation
- 毎mousemoveで全Link再計算
- 毎mousemoveでAPI呼び出し

実施:

- 見た目更新
- Snap計算
- 関連Linkのみ更新
- ドロップ時にCommand確定

## 12. Validation最適化

Validationは操作確定時または保存前に行う。

リアルタイムValidationは軽量なものに限定する。

重いValidation:

- 全Link整合性
- 全Node所属確認
- 循環検出
- Export可能性確認

これらは保存前または明示実行時に行う。

## 13. Undo/Redo性能

Undo/RedoはCommand単位で差分反映する。

Flow全体を毎回丸ごと復元しない。

ただしVersion Snapshotは別機能として全体保存する。

## 14. AutoSave性能

AutoSaveは差分のみ送信する。

送信対象:

- dirtyNodes
- dirtyLinks
- dirtyComments
- dirtyLanes
- dirtyStages
- deletedIds

全Flow送信は手動保存または初期実装簡略時のみ許可する。

## 15. Debounce

以下はDebounceする。

- プロパティ入力
- AutoSave
- 検索
- Validation
- Viewport保存

例:

```text
AutoSave: 30秒
プロパティ入力反映: 300ms
検索: 300ms
```

## 16. Throttle

以下はThrottleする。

- Drag中補助表示
- Resize中補助表示
- Viewport変更
- Mouse move

## 17. DOM設計

大量DOMを避ける。

禁止:

- 全Nodeに常時Tooltip
- 全Linkに常時詳細ラベル
- 全要素に重い影
- 全要素に複雑なCSS filter
- 大量SVG filter

## 18. CSS注意点

重いCSSを避ける。

注意:

- box-shadow多用
- filter
- backdrop-filter
- blur
- 大量transition
- 複雑なclip-path

Node形状で必要な場合も最小限にする。

## 19. 計測

描画性能は計測する。

計測項目:

- Flow読み込み開始
- API応答完了
- ViewModel生成完了
- Vue Flow描画完了
- 初回操作可能時刻

ログまたは開発用メトリクスに出す。

## 20. Performance Marker

開発時は以下を利用可能。

```ts
performance.mark('flow-load-start')
performance.mark('flow-viewmodel-created')
performance.mark('flow-render-ready')
performance.measure(...)
```

## 21. テストデータ

性能テスト用Flow:

- 100Node / 500Link
- 1000Node / 10000Link
- Lane 20
- Stage 50
- Comment 500
- ImageNode 100

## 22. 描画モード

将来的に以下の表示モードを用意する。

- Full
- Simple
- Performance
- ReadonlyReview

PerformanceモードではLinkラベルや影を減らす。

## 23. メモリ管理

Flow切替時は不要データを破棄する。

対象:

- ViewModel Cache
- Selection
- Hover state
- Validation result
- Image object URL
- Temporary clipboard

## 24. 禁止事項

- 10000Linkすべてに重いDOMを常時生成する
- Hoverで全体Storeを書き換える
- Drag中にAPI保存する
- Flow全体deep watch
- JSON.stringifyで差分判定を多用する
- 大量Nodeの座標判定にDOM APIを多用する

## 25. テスト観点

- 1000Nodeを表示できる
- 10000Linkを表示できる
- Dragが極端に重くならない
- Zoom/Panが操作可能
- Linkラベル非表示モードが効く
- Flow切替後にメモリが増え続けない
- AutoSaveが差分送信される

## 26. 完了条件

- 大量要素で操作不能にならない
- 性能劣化要因が設計上抑制されている
- 計測ポイントが定義されている
- 将来の仮想化・簡略表示へ拡張可能
