# 13_19_描画テスト設計

## 1. 目的

本書は、AI Flow Designerの描画エンジンに関するテスト設計を定義する。

描画エンジンは、SSOTからCanvas表示を生成し、ユーザー操作をCommandへ変換する重要領域である。見た目だけでなく、座標変換、選択、Drag、Link接続、Snap、Clipboard、性能まで検証する。

## 2. テスト分類

| 分類 | 対象 |
| --- | --- |
| Unit Test | Adapter / 座標変換 / Snap計算 |
| Component Test | Node / Link / Comment表示 |
| Interaction Test | Drag / Selection / Connect |
| E2E Test | Flow編集一連操作 |
| Performance Test | 大量Node描画 |

## 3. Unit Test

対象:

- screenToCanvasCoordinate
- canvasToScreenCoordinate
- snapToGrid
- flowToDrawingModelAdapter
- drawingNodeToVueFlowNodeAdapter
- vueFlowToCommandAdapter

## 4. Component Test

対象:

- StartNode
- EndNode
- ProcessNode
- DecisionNode
- ImageNode
- CommentBox
- LinkLabel

観点:

- NodeTypeごとの表示
- Validation Badge
- Selected表示
- AI Note Badge

## 5. Interaction Test

観点:

- Nodeクリックで選択
- Linkクリックで選択
- DragでMoveNodeCommand生成
- ConnectでAddLinkCommand生成
- Deleteで削除確認
- Ctrl + C / VでCopy Paste

## 6. E2E Test

代表シナリオ:

1. FlowEditorPageを開く。
2. Node PaletteからNode追加。
3. Nodeを移動。
4. Linkを接続。
5. Property PanelでLink条件入力。
6. Commentを追加。
7. 保存する。

## 7. Performance Test

対象:

- 200Node表示
- 1000Node表示
- 10000Link簡易表示
- Node Drag応答性
- MiniMap更新

## 8. 禁止事項

- Vue Flow内部実装そのものを過剰にテストする。
- SSOTと表示Modelの分離テストを省略する。
- Drag中に保存Requestが作られないことを未検証にする。

## 9. 完了条件

- 描画、座標、操作、性能のテスト観点が定義されている。
- 実装者が描画エンジンのテストを実装できる。
