# 05_02_認証API詳細設計

## 概要

GitHub OAuthを用いてユーザー認証を行う。

ログイン後、JWTを発行し、フロントエンドは以降のAPI呼び出しでAuthorizationヘッダーに付与する。

## API一覧

| メソッド | URL | 用途 |
|---|---|---|
| GET | /api/auth/github/login | GitHub認証開始 |
| GET | /api/auth/github/callback | GitHub認証戻り |
| GET | /api/auth/me | ログインユーザー取得 |
| POST | /api/auth/logout | ログアウト |

## GET /api/auth/github/login

### 用途

GitHub OAuth認証画面へリダイレクトする。

### 処理

1. stateを生成
2. stateをセッションまたは一時ストアに保存
3. GitHub認証URLへリダイレクト

### レスポンス

リダイレクトのためJSONレスポンスは返さない。

## GET /api/auth/github/callback

### 用途

GitHubからの認証戻りを受け取り、ユーザー情報を取得する。

### クエリ

| 項目 | 必須 | 説明 |
|---|---|---|
| code | Yes | GitHub認可コード |
| state | Yes | CSRF対策用state |

### 処理

1. state検証
2. codeをaccess_tokenへ交換
3. GitHub User APIでユーザー情報取得
4. USER_ACCOUNTを作成または更新
5. JWT発行
6. フロントエンドへリダイレクト

## GET /api/auth/me

### 用途

現在ログイン中のユーザーを返す。

### Response

```json
{
  "userId": "guid",
  "loginName": "github-user",
  "displayName": "User Name",
  "email": "user@example.com",
  "avatarUrl": "https://...",
  "role": "Editor"
}
```

## POST /api/auth/logout

### 用途

ログアウトする。

JWT方式ではクライアント側破棄が基本だが、将来ブラックリスト方式にも対応できる構成にする。

## セキュリティ

- state検証必須
- HTTPS前提
- JWT有効期限を設定
- リフレッシュトークンは初期実装では不要
- GitHubメール未公開時はemail nullを許容
