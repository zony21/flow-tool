# 05_01_API設計方針

## 目的

AI Flow DesignerのAPIは、フロントエンドが保持する編集状態とバックエンドDBの整合性を保ち、構造化データを正として管理するための境界である。

APIは単なるCRUDではなく、以下を担う。

- Project管理
- Flow管理
- Lane/Stage/Node/Link/Comment編集
- Flow保存
- Flowバージョン管理
- Export処理
- Template処理
- 権限検証
- 編集ロック
- 監査ログ

## 基本方針

- REST APIを基本とする。
- リクエスト/レスポンスはJSON。
- 認証はGitHub OAuth + JWT。
- DB Entityを直接返却しない。
- DTOを定義して返却する。
- IDはGUID文字列。
- 日時はISO 8601。
- すべての更新APIで権限チェックを行う。
- Flow編集系APIは編集ロックを確認する。
- 重要な更新はトランザクション管理する。

## 共通URL

```text
/api
```

## API分類

| 分類 | 用途 |
|---|---|
| Auth | GitHub OAuth認証 |
| Project | プロジェクト管理 |
| Flow | フロー管理 |
| Editor | エディタ編集 |
| Version | バージョン管理 |
| Template | テンプレート |
| Export | Mermaid/PDF/JSON/AI DSL出力 |
| Setting | ユーザー設定 |
| Image | 画像管理 |

## HTTPステータス

| ステータス | 用途 |
|---|---|
| 200 | 正常取得・正常更新 |
| 201 | 作成成功 |
| 204 | 削除成功 |
| 400 | 入力不正 |
| 401 | 未認証 |
| 403 | 権限なし |
| 404 | 対象なし |
| 409 | 競合・編集ロック |
| 422 | 業務バリデーションエラー |
| 500 | サーバーエラー |

## 共通レスポンス

```json
{
  "success": true,
  "data": {},
  "errors": [],
  "traceId": "string"
}
```

## エラーレスポンス

```json
{
  "success": false,
  "data": null,
  "errors": [
    {
      "code": "FLOW_LOCKED",
      "message": "このFlowは他ユーザーが編集中です。",
      "field": null
    }
  ],
  "traceId": "string"
}
```

## バリデーション方針

- 必須チェック
- 最大長チェック
- GUID形式チェック
- 権限チェック
- 参照整合性チェック
- Flow跨ぎ禁止チェック
- 削除済みデータ参照禁止チェック
- レーン/工程削除時の移動先チェック

## DTO命名

| 種別 | 命名 |
|---|---|
| Request | XxxRequest |
| Response | XxxResponse |
| 一覧項目 | XxxListItemResponse |
| 詳細 | XxxDetailResponse |

## 実装方針

ASP.NET CoreのControllerは薄く保ち、業務処理はApplication Serviceへ委譲する。

```text
Controller
 ↓
Application Service
 ↓
Repository
 ↓
EF Core
 ↓
SQLite / SQL Server
```
