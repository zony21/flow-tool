# 05_01_API設計方針

## 1. 目的

AI Flow Designer のAPI設計方針を定義する。
APIはFrontendとBackendを接続し、Project、Flow、FlowVersion、Lane、Stage、Node、Link、Commentを取得・保存・出力するための境界である。

## 2. 基本方針

画面上の図ではなく構造化データを正とする。
APIはDTOを通して構造化データを受け渡す。
Entityをそのまま返却しない。

## 3. REST方針

取得はGET、作成はPOST、更新はPUTまたはPATCH、削除はDELETEを使用する。
出力生成、複製、テンプレート適用など処理を伴う操作はPOSTを使用する。

## 4. 保存方針

FlowVersion保存ではLane、Stage、Node、Link、Commentを一括保存する。
保存途中で失敗した場合に不整合が残らないようトランザクションを使用する。

## 5. 出力方針

Mermaid、PDF、JSON、将来AI DSLはFlowVersionの構造化データから生成する。

## 6. テスト観点

- DTOがEntityと分離されていること
- FlowVersion保存が一括処理されること
- Template適用時にIDが再採番されること
- 出力APIが構造化データを基準にすること

## 7. 完了条件

各API詳細設計、DTO設計、エラー設計、OpenAPI定義へ展開できる状態であること。
