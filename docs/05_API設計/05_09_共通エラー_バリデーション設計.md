# 05_09_共通エラー・バリデーション設計

## エラーコード一覧

| コード | HTTP | 説明 |
|---|---:|---|
| UNAUTHORIZED | 401 | 未認証 |
| FORBIDDEN | 403 | 権限なし |
| NOT_FOUND | 404 | 対象なし |
| VALIDATION_ERROR | 422 | 入力不正 |
| FLOW_LOCKED | 409 | 編集ロック中 |
| REVISION_CONFLICT | 409 | リビジョン競合 |
| NODE_NOT_FOUND | 422 | Node不正 |
| LINK_INVALID_FLOW | 422 | Flow跨ぎLink |
| DELETE_POLICY_REQUIRED | 422 | 削除方針未指定 |
| EXPORT_FAILED | 500 | 出力失敗 |

## 共通エラー形式

```json
{
  "success": false,
  "data": null,
  "errors": [
    {
      "code": "VALIDATION_ERROR",
      "message": "プロジェクト名は必須です。",
      "field": "projectName"
    }
  ],
  "traceId": "00-abc"
}
```

## バリデーション実装

- DataAnnotations
- FluentValidation
- Application Service内の業務チェック

## 業務バリデーション

| 対象 | 内容 |
|---|---|
| Project | 同名重複 |
| Flow | 編集ロック |
| Lane | 削除時の移動先 |
| Stage | 削除時の移動先 |
| Node | Lane/Stage存在 |
| Link | From/To存在、同一Flow |
| Comment | 紐付けNode存在 |
| Export | 出力形式妥当 |

## ログ

サーバーエラーはtraceIdを必ず返す。

フロントエンドはtraceIdを表示またはコピー可能にする。
