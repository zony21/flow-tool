# 05_03_DTO設計

## 1. 目的

APIで使用するRequest DTOとResponse DTOの設計方針を定義する。
Entityを直接公開せず、画面に必要な構造へ変換して返却する。

## 2. DTO分類

- Auth DTO
- Project DTO
- Flow DTO
- FlowVersion DTO
- Lane DTO
- Stage DTO
- Node DTO
- Link DTO
- Comment DTO
- Template DTO
- Export DTO
- Error DTO

## 3. Project DTO

### ProjectResponse

| 項目 | 型 | 説明 |
| --- | --- | --- |
| projectId | string | Project ID |
| projectName | string | Project名 |
| description | string | 説明 |
| flowCount | number | Flow数 |
| updatedAt | string | 更新日時 |

### ProjectSaveRequest

| 項目 | 型 | 必須 |
| --- | --- | --- |
| projectName | string | YES |
| description | string | NO |
| displayOrder | number | NO |

## 4. Flow DTO

FlowResponseはflowId、projectId、flowName、category、description、latestVersionNo、nodeCount、updatedAtを持つ。
FlowSaveRequestはflowName、category、description、displayOrderを持つ。

## 5. FlowVersion DTO

FlowVersionDetailResponseはFlowVersionの構造全体を返す。

- flowVersionId
- flowId
- versionNo
- isLatest
- lanes
- stages
- nodes
- links
- comments

## 6. Node DTO

NodeDtoは以下を持つ。

- nodeId
- laneId
- stageId
- nodeType
- displayName
- processType
- description
- aiMemo
- x
- y
- width
- height

## 7. Link DTO

LinkDtoは以下を持つ。

- linkId
- fromNodeId
- toNodeId
- dataName
- communicationType
- conditionText
- description
- lineType

## 8. Comment DTO

CommentDtoは以下を持つ。

- commentId
- nodeId
- commentText
- isAiOnly
- x
- y

## 9. Error DTO

ErrorResponseはcode、message、details、traceIdを持つ。
traceIdはログ照合用であり、内部例外は含めない。

## 10. テスト観点

- Entityを直接返さないこと
- FlowVersionDetailResponseで構造全体を復元できること
- 必須項目がRequest DTOで表現されていること
- ErrorResponseが統一されていること

## 11. 完了条件

画面、DB、APIの項目対応がDTOで明確になっていること。
