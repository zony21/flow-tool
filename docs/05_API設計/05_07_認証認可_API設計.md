# 05_07_認証認可_API設計

## 1. 目的

GitHub OAuthを利用した認証APIと、Project単位の権限制御APIを定義する。

## 2. GitHubログイン開始

GET /api/auth/github/login

GitHub OAuthの認可画面へリダイレクトする。
stateを発行し、CSRF対策として検証する。

## 3. GitHubコールバック

GET /api/auth/github/callback

GitHubからのcodeを受け取り、ユーザー情報を取得する。
USERが存在しない場合は作成する。

## 4. ログインユーザー取得

GET /api/auth/me

現在のログインユーザー情報を返す。
画面初期表示、Header、権限制御で使用する。

## 5. ログアウト

POST /api/auth/logout

セッションを破棄する。

## 6. Project権限取得

GET /api/projects/{projectId}/permissions

対象Projectに対する参照、編集、出力、管理の可否を返す。

## 7. 権限制御方針

初期実装ではログイン済みユーザーを利用者として扱う。
将来はPROJECT_MEMBERを用いてOwner、Editor、Viewerを判定する。

## 8. エラー

- 401: 未ログイン
- 403: 操作不可
- 404: 対象Projectなし
- 500: 認証処理失敗

## 9. テスト観点

- GitHubログインできること
- ログインユーザー情報を取得できること
- 未ログインで保護APIが401になること
- 権限不足時に403になること

## 10. 完了条件

ログイン状態とProject権限をAPI側で判定できること。
