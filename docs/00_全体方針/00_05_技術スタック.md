# 00_05_技術スタック

## 1. 本書の目的

本書は、AI Flow Designer の技術スタックを定義する。

技術スタックは、Frontend、Backend、DB、認証、Flow描画、状態管理、Exportの実装前提を揃えるための基準である。

## 2. 採用技術一覧

|領域|採用技術|用途|
|---|---|---|
|Frontend|Vue 3|SPA画面実装|
|Frontend言語|TypeScript|型安全な実装|
|UI|PrimeVue|共通UI部品|
|Flow描画|Vue Flow|Node / Link描画|
|状態管理|Pinia|画面・編集状態管理|
|Build|Vite|開発・ビルド|
|Backend|ASP.NET Core .NET 8|API実装|
|ORM|Entity Framework Core|DBアクセス|
|初期DB|SQLite|導入容易性|
|将来DB|SQL Server|本格運用|
|認証|GitHub OAuth|ログイン|
|Export|Mermaid / PDF / JSON|成果物生成|
|管理|GitHub|ソース管理|

## 3. Frontend方針

FrontendはVue 3とTypeScriptで実装する。

Vue 3はComposition APIにより、Flowエディタの複雑な状態や操作をComposableとして分離しやすい。

TypeScriptは、Node、Link、Lane、Stage、Comment、Template、Versionなどの構造化DTOを安全に扱うために必須とする。

## 4. UI方針

UIはPrimeVueを基本とする。

Project一覧、Flow一覧、設定画面、Version管理画面、Export画面では、DataTable、Dialog、Toast、InputText、Dropdownなどの部品を利用する。

独自Componentは、PrimeVueで表現しきれないFlowエディタ固有部分に限定する。

## 5. Flow描画方針

Flow描画にはVue Flowを利用する。

ただし、Vue Flowの内部状態を保存の正とはしない。

保存の正は、DBとAPI DTOで管理されるProject、Flow、Lane、Stage、Node、Link、Commentである。

## 6. Backend方針

BackendはASP.NET Core .NET 8で実装する。

API、認証、認可、Service、Repository、Validation、LoggingをBackend側で責務分離して実装する。

DBアクセスにはEntity Framework Coreを利用し、MigrationもEF Coreで管理する。

## 7. DB方針

初期DBはSQLiteとする。

SQLiteは開発環境構築が容易で、小規模利用や初期検証に向いている。

将来的に利用規模が大きくなった場合はSQL Serverへ移行する。

そのため、DB設計ではSQLite固有機能に依存しすぎず、SQL Server移行を前提に型、Index、制約を設計する。

## 8. 認証方針

初期認証はGitHub OAuthとする。

独自IDとパスワード管理は初期実装では持たない。

内部的にはUSER_ACCOUNTを正とし、GitHubアカウント情報は外部認証情報として扱う。

## 9. Export方針

初期Exportは以下を対象とする。

- Mermaid Flowchart
- Mermaid Sequence
- PDF
- JSON

将来的にAI DSL、API仕様、DB更新一覧、通信一覧、PLC一覧へ拡張する。

## 10. ライブラリ追加方針

新規ライブラリを追加する場合は、既存技術で代替できないか、保守状況、TypeScript対応、ライセンス、Bundleサイズを確認する。

## 11. 完了条件

- 採用技術と利用領域が定義されている
- Frontend、Backend、DB、認証、Exportの前提が統一されている
- SQLite初期運用とSQL Server移行方針が明確である
