# 18_01_設定管理概要

## 1. 目的

AI Flow Designerの設定管理全体の方針を定義する。

設定管理は、利用者個人、Project、Editor、AI Review、Exportの動作を調整するための機能である。

## 2. 前提

AI Flow Designerの正は構造化データである。

設定は、構造化データの表示、編集、出力、AI解析の動作を制御する補助情報であり、Flow本体の意味を直接変更しない。

## 3. 設計思想

設定は以下の原則で管理する。

- 設定分類を明確に分ける
- User個人設定とProject共通設定を混在させない
- Projectに影響する設定は権限制御する
- 初期値を必ず持つ
- 不正な設定値は保存しない
- 将来拡張できる構造にする

## 4. 設定分類

| 分類 | 目的 |
| --- | --- |
| User Setting | 利用者個人の表示・操作体験を管理する |
| Project Setting | Project全体の既定動作を管理する |
| Editor Setting | Canvas / Grid / Snap等のEditor動作を管理する |
| AI Setting | AI Review / AI DSL出力の動作を管理する |
| Export Setting | PDF / Mermaid / JSON / AI DSLの出力設定を管理する |

## 5. 保存単位

| 設定 | 保存単位 |
| --- | --- |
| User Setting | User単位 |
| Project Setting | Project単位 |
| Editor Setting | User + Project単位、またはProject既定値 |
| AI Setting | Project単位 |
| Export Setting | Project単位 |

## 6. 初期値方針

設定が未登録の場合でも画面・APIが動作するよう、Backendで初期値を返す。

初期値はSettingDefaultServiceで一元管理する。

Frontend側にも表示用の初期値を持ってよいが、保存・API実行時の正はBackendの初期値とする。

## 7. 権限制御

設定変更は17章のPermissionと連携する。

| 設定 | 更新Permission |
| --- | --- |
| User Setting | Setting.UserUpdate |
| Project Setting | Setting.ProjectUpdate |
| Editor Setting | Setting.EditorUpdate |
| AI Setting | Setting.AiUpdate |
| Export Setting | Setting.ExportUpdate |

初期実装では、Project共通設定はOwnerのみ更新可能とする。

User SettingはOwner / Editor / Viewerすべて更新可能とする。

## 8. Validation方針

設定値はFrontendとBackendの両方でValidationする。

Backendでは必ず保存前に以下を確認する。

- 必須値
- 型
- 選択肢
- 数値範囲
- JSON形式
- Project権限

## 9. Error方針

| 状態 | HTTP Status | 説明 |
| --- | --- | --- |
| 未ログイン | 401 | 認証されていない |
| 権限不足 | 403 | 対象設定を変更できない |
| 設定値不正 | 400 | Validation Error |
| Projectなし | 404 | 対象Projectが存在しない |

## 10. 完了条件

- 設定分類が明確である
- 保存単位が明確である
- 初期値方針が定義されている
- 権限制御と連携している
- DB/API/Frontend/Backendへ展開可能である
