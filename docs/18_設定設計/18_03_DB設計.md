# 18_03_DB設計

## 1. 目的

AI Flow Designerの設定情報を保存するDB設計を定義する。

設定はUser個人設定とProject共通設定に分かれるため、保存単位ごとにテーブルを分ける。

## 2. DB設計方針

- 設定分類ごとにテーブルを分ける
- JSON丸ごと保存だけに依存しない
- よく使う設定はカラムとして保持する
- 将来拡張用にSETTING_JSONを保持できる構造にする
- User単位設定とProject単位設定を混在させない
- 作成日時・更新日時を必ず持つ

## 3. テーブル一覧

| テーブル | 保存単位 | 目的 |
| --- | --- | --- |
| USER_SETTING | User | 個人設定 |
| PROJECT_SETTING | Project | Project共通設定 |
| EDITOR_SETTING | User + Project | Editor操作設定 |
| AI_SETTING | Project | AI Review / AI DSL設定 |
| EXPORT_SETTING | Project | Export設定 |

## 4. USER_SETTING

### 4.1 目的

User個人の表示・操作設定を保存する。

### 4.2 カラム

| カラム | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| USER_SETTING_ID | TEXT | ○ | UUID |
| USER_ID | TEXT | ○ | User ID |
| THEME | TEXT | ○ | light / dark / system |
| LANGUAGE | TEXT | ○ | ja / en |
| DEFAULT_ZOOM | REAL | ○ | 初期ズーム |
| DENSITY | TEXT | ○ | compact / normal / comfortable |
| SHOW_MINI_MAP | INTEGER | ○ | 1=表示 |
| SHOW_GRID | INTEGER | ○ | 1=表示 |
| AUTO_SAVE | INTEGER | ○ | 1=有効 |
| SETTING_JSON | TEXT |  | 拡張設定JSON |
| CREATED_AT | DATETIME | ○ | 作成日時 |
| UPDATED_AT | DATETIME | ○ | 更新日時 |

### 4.3 制約

- USER_IDは一意
- THEMEはlight / dark / systemのみ
- DEFAULT_ZOOMは0.2〜3.0

| CREATED_AT | DATETIME | ○ | 作成日時 |

### 5.3 制約

- PROJECT_IDは一意

## 9. 初期データ作成方針

設定レコードは以下のいずれかで作成する。

- User初回ログイン時にUSER_SETTINGを作成
- Project作成時にPROJECT_SETTING / AI_SETTING / EXPORT_SETTINGを作成
- Editor初回表示時にEDITOR_SETTINGを作成

ただし、レコードが存在しない場合でもAPIはDefault値を返す。

## 10. EF Core方針

Entity:

- UserSettingEntity
- ProjectSettingEntity
- EditorSettingEntity
- AiSettingEntity
- ExportSettingEntity

DbContext:

- UserSettings
- ProjectSettings
- EditorSettings
- AiSettings
- ExportSettings

## 11. 完了条件

- 設定分類ごとのテーブルが定義されている
- 保存単位がDB制約で表現できる
- 初期値と未登録時の扱いが明確である
- EF Core Entityへ展開できる
