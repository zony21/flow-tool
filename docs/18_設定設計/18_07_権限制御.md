# 18_07_権限制御

## 1. 目的

設定機能と17_権限管理設計の連携方針を定義する。

設定にはUser個人に閉じるものとProject全体へ影響するものがあるため、影響範囲に応じて権限制御を分ける。

## 2. 基本方針

- User個人設定は本人のみ更新可能
- Project共通設定はOwnerのみ更新可能
- Editor個人設定はProject参加者であれば更新可能
- AI SettingとExport SettingはProject出力品質へ影響するためOwnerのみ更新可能
- Frontend制御だけに依存せずBackendで必ず認可する

## 3. Permission対応

| 設定分類 | 参照Permission | 更新Permission | 初期更新可能Role |
| --- | --- | --- | --- |
| User Setting | ログイン済み | Setting.UserUpdate | Owner / Editor / Viewer |
| Project Setting | Project.Read | Setting.ProjectUpdate | Owner |
| Editor Setting | Flow.Read | Setting.UserUpdate | Owner / Editor / Viewer |
| AI Setting | Project.Read | Setting.AiUpdate | Owner |
| Export Setting | Project.Read | Setting.ExportUpdate | Owner |

## 4. Viewerの扱い

ViewerはFlowを閲覧しExport可能なRoleである。

ただしProject全体に影響する設定は変更できない。

Viewerが変更可能な設定:

- User Setting
- Editor Settingの個人表示項目

Viewerが変更不可の設定:

- Project Setting
- AI Setting
- Export Setting

## 5. Editorの扱い

EditorはFlow編集者である。

Editorが変更可能な設定:

- User Setting
- Editor Settingの個人表示・操作項目

Editorが変更不可の設定:

- Project Setting
- AI Setting
- Export Setting
- Project共通Editor Setting

## 6. Ownerの扱い

Ownerはすべての設定を変更可能とする。

ただし、User Settingは各User本人の個人設定であるため、Ownerであっても他UserのUser Settingは変更しない。

## 7. Backend認可

Backendでは以下を確認する。

### 7.1 User Setting更新

- ログイン済みUserであること
- 更新対象がCurrentUser本人であること
- Setting.UserUpdateを持つこと

### 7.2 Editor Setting更新

- ログイン済みUserであること
- 対象Projectに参加していること
- Flow.Readを持つこと
- Setting.UserUpdateを持つこと

### 7.3 Project / AI / Export Setting更新

- ログイン済みUserであること
- 対象Projectに参加していること
- 対象Permissionを持つこと

## 8. Frontend制御

FrontendではProject Permission Storeを利用する。

保存ボタン制御:

| Panel | 保存ボタン |
| --- | --- |
| User Setting | ログイン済みなら活性 |
| Project Setting | Setting.ProjectUpdateがなければ非活性 |
| Editor Setting | Setting.UserUpdateがなければ非活性 |
| AI Setting | Setting.AiUpdateがなければ非活性 |
| Export Setting | Setting.ExportUpdateがなければ非活性 |

権限不足の場合は入力欄をreadonlyまたはdisabledにする。

## 9. 403発生時

設定保存時に403が返った場合は、権限が変更された可能性がある。

処理:

1. Project Permission Storeを再取得する
2. 対象Panelをreadonlyへ切り替える
3. Toastで権限不足を通知する
4. 入力中の値は即破棄しない

## 10. Test観点

19_テスト仕様では以下を確認する。

- ViewerはUser Settingを更新できる
- ViewerはProject Settingを更新できない
- EditorはEditor Settingを更新できる
- EditorはAI Settingを更新できない
- OwnerはProject Settingを更新できる
- Ownerは他UserのUser Settingを更新できない
- 403時にFrontendがPermission再取得する

## 11. 完了条件

- 設定分類ごとの参照・更新権限が明確である
- Owner / Editor / Viewerの差分が明確である
- Backend認可条件が定義されている
- Frontend制御方針が定義されている
- 19_テスト仕様へ展開可能である
