# 18_02_設定分類設計

## 1. 目的

AI Flow Designerで扱う設定を分類し、それぞれの責務、保存単位、初期値、権限制御を定義する。

## 2. 分類方針

設定は以下の5分類とする。

- User Setting
- Project Setting
- Editor Setting
- AI Setting
- Export Setting

分類を分ける理由は、保存単位、権限、影響範囲が異なるためである。

## 3. User Setting

### 3.1 目的

利用者個人の表示・操作体験を管理する。

### 3.2 保存単位

User単位。

### 3.3 設定項目

| 項目 | 型 | 初期値 | 説明 |
| --- | --- | --- | --- |
| theme | string | system | light / dark / system |
| language | string | ja | ja / en |
| defaultZoom | number | 1.0 | Editor初期ズーム |
| density | string | normal | compact / normal / comfortable |
| showMiniMap | boolean | true | MiniMap表示 |
| showGrid | boolean | true | Grid表示 |
| autoSave | boolean | false | 自動保存 |

### 3.4 更新権限

ログイン済みUser本人のみ更新可能。

PermissionはSetting.UserUpdateを使用する。

## 4. Project Setting

### 4.1 目的

Project全体の共通動作を管理する。

### 4.2 保存単位

Project単位。

### 4.3 設定項目

| 項目 | 型 | 初期値 | 説明 |
| --- | --- | --- | --- |
| projectNameRule | string | free | Project命名規則 |
| flowNameRule | string | free | Flow命名規則 |
| defaultExportFormat | string | pdf | 既定Export形式 |
| allowViewerExport | boolean | true | ViewerのExport可否 |
| versionRequiredOnExport | boolean | false | Export時Version作成必須 |
| archiveLocked | boolean | true | Archive時編集禁止 |

### 4.4 更新権限

Ownerのみ更新可能。

PermissionはSetting.ProjectUpdateを使用する。

## 5. Editor Setting

### 5.1 目的

Flow EditorのCanvas、Grid、Snap、接続線、表示に関する動作を管理する。

### 5.2 保存単位

初期実装ではUser + Project単位とする。

理由は、Editor体験は個人差が大きく、Project共通にすると使い勝手が悪くなるためである。

ただし、Projectの標準Editor設定を将来追加できる構造にする。

### 5.3 設定項目

| 項目 | 型 | 初期値 | 説明 |
| --- | --- | --- | --- |
| gridEnabled | boolean | true | Grid有効 |
| snapEnabled | boolean | true | Snap有効 |
| gridSize | number | 16 | Grid間隔 |
| defaultLinkType | string | bezier | bezier / straight / step / smoothstep |
| showLaneHeader | boolean | true | Lane Header表示 |
| showStageHeader | boolean | true | Stage Header表示 |
| nodeLabelVisible | boolean | true | Node Label表示 |
| canvasBackground | string | default | 背景種別 |

### 5.4 更新権限

User個人のEditor SettingはSetting.UserUpdateで更新可能。

Project共通Editor Settingを導入する場合はSetting.EditorUpdateを使用する。

## 6. AI Setting

### 6.1 目的

AI Review、AI DSL出力、AI解析の挙動を管理する。

### 6.2 保存単位

Project単位。

### 6.3 設定項目

| 項目 | 型 | 初期値 | 説明 |
| --- | --- | --- | --- |
| aiReviewEnabled | boolean | true | AI Review有効 |
| reviewStrictness | string | normal | low / normal / high |
| detectMissingProcess | boolean | true | 処理抜け検出 |
| detectResponsibilityIssue | boolean | true | 責務違反検出 |
| detectDbImpact | boolean | true | DB影響解析 |
| detectApiImpact | boolean | true | API影響解析 |
| dslDetailLevel | string | standard | simple / standard / detailed |
| includeMetadataInDsl | boolean | true | DSLにMetadataを含める |

### 6.4 更新権限

Ownerのみ更新可能。

PermissionはSetting.AiUpdateを使用する。

## 7. Export Setting

### 7.1 目的

PDF、Mermaid、JSON、AI DSLなどの出力設定を管理する。

### 7.2 保存単位

Project単位。

### 7.3 設定項目

| 項目 | 型 | 初期値 | 説明 |
| --- | --- | --- | --- |
| pdfPageSize | string | A4 | A4 / A3 |
| pdfOrientation | string | landscape | portrait / landscape |
| includeComments | boolean | true | コメント含有 |
| includeImages | boolean | true | 画像含有 |
| includeMetadata | boolean | true | Metadata含有 |
| mermaidDirection | string | LR | LR / TB |
| jsonPrettyPrint | boolean | true | JSON整形 |
| aiDslFormatVersion | string | v1 | AI DSL形式Version |

### 7.4 更新権限

Ownerのみ更新可能。

PermissionはSetting.ExportUpdateを使用する。

## 8. 完了条件

- 設定分類が5分類に整理されている
- 各設定の保存単位が明確である
- 初期値が定義されている
- 権限制御が定義されている
- DB/API/Frontendへ展開できる
