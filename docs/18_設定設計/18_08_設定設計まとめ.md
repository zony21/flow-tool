# 18_08_設定設計まとめ

## 1. 目的

18章で定義した設定設計をまとめ、実装時の判断基準を整理する。

## 2. 設定設計の位置付け

設定は、AI Flow Designerの表示、編集、AI Review、Exportの動作を制御する補助情報である。

SSOTの正はFlow構造化データであり、設定は構造化データの意味そのものを変更しない。

## 3. 設定分類

設定は以下の5分類とする。

| 分類 | 保存単位 | 主な用途 |
| --- | --- | --- |
| User Setting | User | 個人の表示・操作体験 |
| Project Setting | Project | Project全体の共通動作 |
| Editor Setting | User + Project | Canvas / Grid / Snap等 |
| AI Setting | Project | AI Review / AI DSL設定 |
| Export Setting | Project | PDF / Mermaid / JSON / AI DSL出力設定 |

## 4. DBまとめ

使用テーブル:

- USER_SETTING
- PROJECT_SETTING
- EDITOR_SETTING
- AI_SETTING
- EXPORT_SETTING

方針:

- 分類ごとにテーブルを分ける
- User設定とProject設定を混在させない
- よく使う設定はカラム化する
- 将来拡張用にSETTING_JSONを持てる構造にする
- 未登録時はDefault値を返す

## 5. APIまとめ

主なAPI:

```text
GET /api/settings/user
PUT /api/settings/user

GET /api/projects/{projectId}/settings/project
PUT /api/projects/{projectId}/settings/project

GET /api/projects/{projectId}/settings/editor
PUT /api/projects/{projectId}/settings/editor

GET /api/projects/{projectId}/settings/ai
PUT /api/projects/{projectId}/settings/ai

GET /api/projects/{projectId}/settings/export
PUT /api/projects/{projectId}/settings/export
```

GET時に設定レコードが存在しない場合はDefault値を返す。

PUT時にレコードが存在しない場合はUpsertする。

## 6. Frontendまとめ

主なComponent:

- SettingsPage.vue
- SettingsSideMenu.vue
- UserSettingsPanel.vue
- ProjectSettingsPanel.vue
- EditorSettingsPanel.vue
- AiSettingsPanel.vue
- ExportSettingsPanel.vue
- SettingActionBar.vue

主なStore:

- UserSettingStore
- ProjectSettingStore
- EditorSettingStore
- AiSettingStore
- ExportSettingStore

制御:

- isDirtyで未保存変更を管理
- 保存前にFrontend Validation
- 403時はPermission再取得
- 権限不足時はreadonly/disabled

## 7. Backendまとめ

主な部品:

- UserSettingController
- ProjectSettingController
- SettingService
- SettingDefaultService
- SettingValidationService
- PermissionService
- CurrentUserService

方針:

- ControllerはHTTP入出力に集中
- 設定取得・保存はSettingServiceへ集約
- 初期値はSettingDefaultServiceで管理
- 保存前ValidationはSettingValidationServiceで実施
- Project共通設定はPermissionServiceで認可

## 8. 権限制御まとめ

| 設定 | 更新可能Role |
| --- | --- |
| User Setting | Owner / Editor / Viewer本人 |
| Project Setting | Owner |
| Editor Setting | Owner / Editor / Viewer本人 |
| AI Setting | Owner |
| Export Setting | Owner |

User Settingは個人設定のため、Ownerであっても他User分は更新しない。

## 9. Validationまとめ

最低限以下を検証する。

- theme: light / dark / system
- language: ja / en
- defaultZoom: 0.2〜3.0
- density: compact / normal / comfortable
- gridSize: 4〜64
- defaultLinkType: bezier / straight / step / smoothstep
- reviewStrictness: low / normal / high
- dslDetailLevel: simple / standard / detailed
- pdfPageSize: A4 / A3
- pdfOrientation: portrait / landscape
- mermaidDirection: LR / TB

## 10. Testへ渡す観点

19_テスト仕様では以下を確認する。

- 未登録時にDefault値が返る
- User Settingを本人が更新できる
- ViewerがProject Settingを更新できない
- EditorがEditor Settingを更新できる
- OwnerがAI Settingを更新できる
- Validation Error時に400を返す
- Project未参加時に403を返す
- 403時にFrontendがPermission再取得する
- isDirty時に画面遷移確認が出る
