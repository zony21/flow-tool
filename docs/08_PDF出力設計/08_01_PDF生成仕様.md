# 08_01_PDF生成仕様

## 1. 目的

PDF出力は、FlowVersionの構造化データをもとに、人がレビュー・承認・共有できる固定レイアウト成果物を生成する機能である。
図そのものを正とせず、Project / Flow / Lane / Stage / Node / Link / Comment のSSOTからPDFを生成する。

## 2. 採用理由

PDFはブラウザ・OS・閲覧者の環境差が少なく、設計レビュー、顧客提示、監査証跡、印刷、保管に向いている。
AI Flow Designerでは、PDFを単なるスクリーンショットではなく、構造化データから再生成可能な正式成果物として扱う。

## 3. 基本方針

- 出力元は保存済みFlowVersion Snapshotとする。
- 編集中未保存データはPDF出力対象にしない。
- フロー図、メタ情報、一覧表、コメント、AIメモを同一PDFへ含める。
- A4縦、A4横、A3横を初期対応とする。
- 大規模Flowでは自動縮尺またはページ分割を選択できる。
- 日本語フォントの埋め込みを前提にする。

## 4. 入力データ

PDF生成Serviceは以下を受け取る。

| 項目 | 内容 |
| --- | --- |
| projectId | Project識別子 |
| flowId | Flow識別子 |
| versionId | 出力対象Version |
| exportOptions | 用紙、向き、余白、分割、表示対象 |
| requestedBy | 出力実行ユーザー |

## 5. 出力構成

1. 表紙
2. 改訂情報
3. フロー図
4. Lane一覧
5. Stage一覧
6. Node一覧
7. Link一覧
8. Comment一覧
9. AI専用メモ一覧
10. 出力条件

## 6. 処理フロー

1. APIでPDF出力要求を受け付ける。
2. 権限を検証する。
3. FlowVersion Snapshotを取得する。
4. ExportOptionsを検証する。
5. PDF用ViewModelへ変換する。
6. Canvas座標をPDF座標へ変換する。
7. 図形、線、文字、画像、コメントを描画する。
8. 一覧ページを生成する。
9. PDFバイト配列を生成する。
10. ダウンロードレスポンスまたはExport履歴へ保存する。

## 7. 完了条件

- 同じFlowVersionと同じOptionsから同一内容のPDFを再生成できる。
- Node / Link / Comment が欠落しない。
- 日本語が文字化けしない。
- ページ番号、出力日時、FlowVersionが表示される。
- 権限のないユーザーは出力できない。
