# 08_20_PDF出力設計まとめ

## 1. 目的
08_PDF出力設計の内容を総括し、実装時に参照すべき判断基準を整理する。

## 2. 設計思想
PDFは画面の見た目を保存する機能ではない。SSOTであるFlowVersion Snapshotから、レビュー・承認・共有できる正式成果物を生成する機能である。

## 3. 実装の中核
- FlowVersion Snapshotを入力とする。
- ExportOptionsで出力内容を制御する。
- Canvas座標をPDF座標へ変換する。
- Node、Link、Lane、Stage、Comment、画像を描画する。
- 一覧表で詳細情報を補完する。
- 日本語フォントを正しく扱う。

## 5. 完了条件
