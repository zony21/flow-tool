# 08_04_レイアウト変換設計

## 1. 目的
Canvas座標をPDF座標へ変換し、画面表示とPDF出力の位置関係を一致させる。

## 2. 座標変換方針
Canvasの論理座標を基準とし、PDFの描画可能領域へスケール変換する。PDF座標は左上原点として扱う変換層を設け、PDFライブラリ依存をService外へ漏らさない。

## 3. 変換式
- scaleX = drawableWidth / canvasBounds.width
- scaleY = drawableHeight / canvasBounds.height
- scale = modeがfitの場合はmin(scaleX, scaleY)
- pdfX = marginLeft + (canvasX - minX) * scale
- pdfY = marginTop + (canvasY - minY) * scale

## 4. 対象要素
Node、Lane、Stage、Comment、Link制御点、画像矩形を同一変換式で変換する。

## 5. テスト観点
- 負の座標を含むFlow
- 右下へ大きく広がるFlow
- Lane / Stageのみ存在するFlow
- 画像Nodeを含むFlow

## 6. 完了条件
画面上の相対位置とPDF上の相対位置が一致し、要素が余白外へはみ出さない。
