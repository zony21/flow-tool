# 08_02_出力対象データ設計

## 1. 目的
PDFへ出力するデータ範囲を明確化し、SSOTから漏れなく成果物を生成できるようにする。

## 2. 出力対象
| 区分 | 対象 |
| --- | --- |
| Project | 名称、説明、所有者、作成日、更新日 |
| Flow | 名称、説明、種別、ステータス |
| Version | Version番号、Snapshot作成日時、作成者、変更理由 |
| Lane | 名称、順序、色、責務説明 |
| Stage | 名称、順序、工程説明 |
| Node | 種別、名称、説明、座標、サイズ、責務、AIメモ |
| Link | 接続元、接続先、ラベル、条件、線種 |
| Comment | 独立コメント、Node紐付けコメント、AI専用メモ |

## 3. 除外対象
- 論理削除済みデータ
- ユーザーが出力オプションで非表示にした要素
- 編集中で未保存の一時状態
- Undo/Redo用の操作履歴

## 4. データ取得条件
PDF生成はFlowVersion Snapshotを基準にする。現在テーブルの最新値ではなく、指定Version時点の構造を復元して出力する。

## 5. 完了条件
- Flow図に表示されるNodeと一覧表のNode件数が一致する。
- Link一覧に存在するLinkは図中でも表現される。
- Commentは独立・Node紐付けの両方が出力できる。
