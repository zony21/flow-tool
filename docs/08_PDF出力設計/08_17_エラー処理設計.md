# 08_17_エラー処理設計

## 1. 目的
PDF生成時の異常を分類し、ユーザーと開発者が原因を特定できるようにする。

## 2. エラー分類
| 分類 | HTTP | 内容 |
| --- | --- | --- |
| 入力エラー | 400 | ExportOptions不正 |
| 権限エラー | 403 | Project / Flow閲覧権限なし |
| 対象なし | 404 | Project / Flow / Versionが存在しない |
| データ不整合 | 409 | Snapshot内でNode参照が欠落 |
| 生成失敗 | 500 | PDFライブラリ、フォント、画像取得失敗 |

## 3. ユーザー表示
ユーザーには原因分類と対処方針を表示する。内部例外やファイルパスは表示しない。

## 4. ログ
ログにはrequestId、projectId、flowId、versionId、userId、errorCodeを記録する。

## 5. 完了条件
エラー時に空PDFや壊れたPDFを返さず、適切なHTTPステータスを返す。
