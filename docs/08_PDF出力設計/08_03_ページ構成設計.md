# 08_03_ページ構成設計

## 1. 目的
PDFのページ構成を固定化し、設計レビュー時に必要な情報を同じ順序で確認できるようにする。

## 2. 標準ページ構成
1. 表紙
2. 改訂情報
3. フロー図ページ
4. Node一覧
5. Link一覧
6. Lane / Stage一覧
7. Comment一覧
8. AI専用メモ一覧
9. 出力条件

## 3. 表紙
表紙にはProject名、Flow名、Version、出力日時、出力者、機密区分を表示する。

## 4. フロー図ページ
Canvas全体を1ページへ縮尺するモードと、ページ分割するモードを持つ。初期値は自動判定とする。

## 5. 一覧ページ
一覧ページは表形式で出力する。列幅は日本語表示を前提に折り返し可能とし、ID、名称、説明、関連要素を表示する。

## 6. 完了条件
- どのFlowでも表紙から出力条件まで同じ順序で出力される。
- 非表示オプション指定時もページ番号が破綻しない。
