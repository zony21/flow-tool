# 08_18_性能設計

## 1. 目的
大規模FlowでもPDF生成が実用的な時間で完了し、Backendのメモリ使用量が過大にならないようにする。

## 2. 性能目標
| 規模 | 目標 |
| --- | --- |
| Node 100件 | 5秒以内 |
| Node 500件 | 15秒以内 |
| Node 1000件 | 30秒以内または非同期化検討 |

## 3. 対策
- Snapshot取得は必要な関連データを一括取得する。
- 画像はサイズ上限を設ける。
- 一覧表はストリーム的にページ追加する。
- 不要な中間画像化を避ける。
- 同一Version、同一Optionsの再生成キャッシュを将来検討する。

## 4. メモリ制御
PDF生成中の画像展開、フォント埋め込み、ページバッファがメモリを圧迫しないよう、上限を設定する。

## 5. 完了条件
Node 500件規模のFlowでタイムアウトせず、Backendが安定して応答する。
