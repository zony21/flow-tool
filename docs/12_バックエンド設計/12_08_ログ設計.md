# 12_08_ログ設計

## 1. 本書の目的

本書は、AI Flow Designer のバックエンドにおけるログ設計を定義する。

AI Flow Designer は、構造化フローをSingle Source of Truthとして保持するWebアプリケーションである。Flow保存、Version作成、Export、画像管理、編集ロック、Undo/Redo補助情報など、後から原因調査が必要になる処理が多い。

そのため、本システムのログは単なる障害記録ではなく、以下を実現するための運用基盤として扱う。

- 障害原因の追跡
- Flow保存失敗時の原因調査
- Export失敗時の原因調査
- 編集ロック競合の確認
- 権限エラーの確認
- API性能劣化の検出
- 監査証跡の補助
- AI実装時のデバッグ容易化

## 2. 基本方針

ログ設計は以下の方針とする。

- ASP.NET Core標準のILoggerを利用する
- 構造化ログを前提とする
- 文字列連結ログは禁止
- traceIdを必ず出力する
- userIdを可能な限り出力する
- projectId / flowId を可能な限り出力する
- OAuth token / JWT / 個人情報は出力しない
- 通常操作はInformation
- 業務上の想定内エラーはWarning
- システム障害はError
- 処理時間を記録する
- 将来の外部ログ基盤連携を妨げない

## 3. ログレベル

|Level|用途|
|---|---|
|Trace|原則使用しない。局所調査用|
|Debug|開発環境の詳細確認|
|Information|正常な重要処理|
|Warning|想定内だが注意が必要な事象|
|Error|処理失敗・復旧可能な障害|
|Critical|システム継続困難な障害|

## 4. Information対象

Informationログは以下で出力する。

- ユーザーログイン成功
- Project作成
- Flow作成
- Flow保存成功
- FlowVersion作成
- FlowVersion復元
- Template作成
- Export成功
- 画像アップロード成功
- 編集ロック取得
- 編集ロック解除
- 自動保存成功

## 5. Warning対象

Warningログは以下で出力する。

- Validation失敗
- 権限不足
- 編集ロック競合
- Revision競合
- NotFound
- Template名重複
- 削除済データへのアクセス
- 想定範囲内のExport不可
- 画像サイズ超過
- SVG検証失敗

## 6. Error対象

Errorログは以下で出力する。

- DB保存失敗
- Transaction失敗
- Export処理失敗
- PDF生成失敗
- 画像保存失敗
- ファイル移動失敗
- 予期しない例外
- 外部連携失敗
- GitHub OAuth検証失敗

## 7. Critical対象

Criticalは原則として以下に限定する。

- DB接続不能が継続
- ストレージルートが消失
- 設定ファイル読込不能
- 起動時Migration失敗
- アプリケーション継続不能

## 8. 必須ログ項目

すべての重要ログには以下を含める。

|項目|説明|
|---|---|
|TraceId|HTTPリクエスト単位の追跡ID|
|UserId|ログインユーザーID|
|ProjectId|対象Project ID|
|FlowId|対象Flow ID|
|Action|処理名|
|Result|Success / Failed|
|ElapsedMs|処理時間|
|ErrorCode|エラー時のコード|
|RequestPath|APIパス|
|HttpMethod|HTTPメソッド|

## 9. ログスコープ

ASP.NET Core の BeginScope を利用し、リクエスト単位で共通情報を付与する。

```csharp
using (_logger.BeginScope(new Dictionary<string, object>
{
    ["TraceId"] = traceId,
    ["UserId"] = currentUser.UserId,
    ["ProjectId"] = projectId,
    ["FlowId"] = flowId
}))
{
    await _flowService.SaveAsync(request);
}
```

## 10. APIリクエストログ

すべてのAPIリクエストについて、最低限以下を記録する。

- Method
- Path
- StatusCode
- ElapsedMs
- UserId
- TraceId

正常終了はInformation、4xxはWarning、5xxはErrorとする。

## 11. Request Bodyログ

Request Bodyは原則ログ出力しない。

理由:

- Flow保存時に大量データとなる
- 構造化フローには業務情報が含まれる可能性がある
- 画像や拡張JSONを含む可能性がある
- 個人情報混入リスクがある

必要な場合は、件数やIDのみ記録する。

例:

```text
NodeCount=120
LinkCount=300
CommentCount=40
```

- FlowId
- ProjectId
- RequestedRevision
- CurrentRevision
- ErrorCode
- ExceptionType
- TransactionState

## 14. Versionログ

FlowVersion作成時:

- VersionId
- FlowId
- ProjectId
- VersionNo
- SnapshotSize
- CreatedBy

FlowVersion復元時:

- VersionId
- FlowId
- ProjectId
- RestoreFromVersionNo
- NewRevision

## 15. Exportログ

Export処理では以下を記録する。

- ExportType
- FlowId
- ProjectId
- VersionId
- NodeCount
- LinkCount
- OutputSize
- ElapsedMs
- Result

ExportType:

- MermaidFlowchart
- MermaidSequence
- PDF
- JSON
- AIDSL
- ApiSpec
- DbUpdateList
- PlcList
- CommunicationList
- DesignDraft

## 16. 画像ログ

画像アップロード時:

- ImageId
- FileNameHash
- ContentType
- FileSize
- Hash
- StorageKey
- Result

ファイル名そのものはログに出さない。必要な場合はHash化する。

## 17. 認証ログ

GitHub OAuthログイン時:

- UserId
- GitHubUserId
- LoginResult
- IsNewUser
- ElapsedMs

出力禁止:

- Access Token
- Refresh Token
- Authorization Code
- JWT

## 18. 権限ログ

権限不足時:

- UserId
- ProjectId
- FlowId
- RequiredRole
- ActualRole
- Action
- Result

## 19. 編集ロックログ

- ExpiredAt

編集ロック競合:

- FlowId
- RequestUserId
- LockOwnerUserId
- ExpiredAt

編集ロック解除:

- FlowId
- UserId

Reason:

## 20. 性能ログ

以下は性能ログ対象とする。

- Flow保存
- Flow読込
- Flow一覧
- Export
- Version比較
- Template複写
- 画像アップロード
- 1000ノード以上の描画用データ取得

閾値例:

|処理|Warning閾値|
|---|---:|
|Flow保存|3000ms|
|Flow読込|2000ms|
|Export PDF|10000ms|
|Version比較|5000ms|
|画像アップロード|5000ms|

## 21. ログ出力例

```csharp
_logger.LogInformation(
    "Flow saved. FlowId={FlowId}, ProjectId={ProjectId}, RevisionBefore={RevisionBefore}, RevisionAfter={RevisionAfter}, NodeCount={NodeCount}, LinkCount={LinkCount}, ElapsedMs={ElapsedMs}",
    flowId,
    projectId,
    revisionBefore,
    revisionAfter,
    nodeCount,
    linkCount,
    elapsedMs);
```

## 22. 禁止ログ例

以下は禁止。

```csharp
_logger.LogInformation("token=" + token);
_logger.LogError(ex.ToString());
_logger.LogInformation(JsonSerializer.Serialize(request));
_logger.LogInformation("connectionString=" + connectionString);
```

## 23. 推奨ログ例

```csharp
_logger.LogError(
    ex,
    "Failed to save flow. FlowId={FlowId}, ProjectId={ProjectId}, ErrorCode={ErrorCode}",
    flowId,
    projectId,
    "FLOW_SAVE_FAILED");
```

## 24. 環境別ログ設定

開発環境:

- Debug以上
- Console出力
- 詳細ログを許可

検証環境:

- Information以上
- API性能ログ有効

本番環境:

- Information以上
- 個人情報・機密情報出力禁止
- ログローテーション必須

## 25. 保存期間

初期方針:

|ログ|保存期間|
|---|---:|
|通常APIログ|30日|
|エラーログ|90日|
|監査補助ログ|180日|
|認証ログ|180日|
|性能ログ|90日|

## 27. テスト観点

- 正常APIでInformationログが出る
- Validation失敗でWarningログが出る
- DB例外でErrorログが出る
- traceIdがレスポンスとログで一致する
- Tokenがログへ出力されない
- Flow保存時にNodeCount/LinkCountが記録される
- Export処理時間が記録される

## 28. 完了条件

- 主要APIに構造化ログが入っている
- 共通ログMiddlewareが実装されている
- 例外MiddlewareとtraceIdが連携している
- 認証情報がログに漏れない
- 障害時にProjectId/FlowId/UserIdで追跡できる
