# 12_03_レイヤ責務設計

## 1. 目的

本書は、AI Flow Designer Backendのレイヤ責務を定義する。

Backendは、Controller、Application Service、Domain、Repository、Infrastructureが明確に分離されている必要がある。責務が曖昧になると、Controller肥大化、Repositoryへの業務ロジック混入、DB依存の拡散、SSOT検証漏れが発生する。

## 2. レイヤ一覧

```text
Api Layer
Application Layer
Domain Layer
Infrastructure Layer
Persistence Layer
```

## 3. Api Layer

### 3.1 責務

- HTTP Request受信
- Route定義
- Authorize属性付与
- Request DTO受信
- Service呼び出し
- Response DTO返却
- HTTP Status返却

### 3.2 配置

```text
FlowDesigner.Api/Controllers
FlowDesigner.Api/Middlewares
FlowDesigner.Api/Filters
```

### 3.3 禁止事項

- 業務ロジックを書く。
- Repositoryを直接呼ぶ。
- DbContextを直接使う。
- EntityをResponseとして返す。
- Transactionを開始する。

## 4. Application Layer

### 4.1 責務

- UseCase実行
- 業務ルールの呼び出し
- DTO Validation
- Domain Rule実行
- Repository Interface呼び出し
- Transaction境界制御
- 認可確認
- DTO変換
- 例外送出

### 4.2 配置

```text
FlowDesigner.Application/Services
FlowDesigner.Application/DTOs
FlowDesigner.Application/Validators
FlowDesigner.Application/Interfaces
```

### 4.3 代表Service

- AuthService
- ProjectService
- FlowService
- FlowQueryService
- FlowSaveService
- FlowVersionService
- TemplateService
- ExportService
- ImageFileService
- ProjectAuthorizationService

### 4.4 禁止事項

- EF Core DbContextを直接利用する。
- Infrastructure具象クラスへ依存する。
- HTTP固有処理に依存する。

## 5. Domain Layer

### 5.1 責務

- SSOTの意味定義
- Entity定義
- Value Object定義
- Domain Rule定義
- Domain Exception定義

### 5.2 配置

```text
FlowDesigner.Domain/Entities
FlowDesigner.Domain/ValueObjects
FlowDesigner.Domain/Enums
FlowDesigner.Domain/Rules
```

### 5.3 禁止事項

- ASP.NET Coreへ依存する。
- EF Coreへ依存する。
- API DTOへ依存する。
- Repositoryへ依存する。

## 6. Infrastructure Layer

### 6.1 責務

- Repository実装
- EF Core DbContext
- Migration
- GitHub OAuth通信
- JWT生成
- File Storage
- PDF / Mermaid / JSON / AI DSL Export実装
- 外部ライブラリ利用

### 6.2 配置

```text
FlowDesigner.Infrastructure/Persistence
FlowDesigner.Infrastructure/Repositories
FlowDesigner.Infrastructure/Auth
FlowDesigner.Infrastructure/Storage
FlowDesigner.Infrastructure/Export
```

### 6.3 禁止事項

- Controllerへ依存する。
- API Responseを生成する。
- 業務ルールを独自判断する。

## 7. Persistence Layer

PersistenceはInfrastructure内のDB関連領域である。

責務:

- AppDbContext
- Entity Configuration
- Migration
- Query最適化
- DB Provider差異吸収

SQLite / SQL Server差異はここへ閉じる。

## 8. レイヤ別データ形式

| レイヤ | 主なデータ形式 |
| --- | --- |
| Api | Request / Response DTO |
| Application | DTO / Application Model / Domain Entity |
| Domain | Entity / Value Object |
| Infrastructure | EF Entity / Storage Model |
| Persistence | Table / Column / Migration |

## 9. データ変換方針

```text
Request DTO
  ↓ Validator
Application Model
  ↓ Domain Rule
Domain Entity
  ↓ Repository
EF Entity / DB
```

Response時:

```text
DB / EF Entity
  ↓ Repository
Domain Entity
  ↓ Service / Mapper
Response DTO
```

EntityをControllerから直接返さない。

## 10. Flow保存時の責務分担

| 処理 | 担当 |
| --- | --- |
| Request受信 | FlowController |
| 認証確認 | Middleware / Authorize |
| Project権限確認 | ProjectAuthorizationService |
| Revision確認 | FlowSaveService |
| 構造検証 | FlowStructureValidator / Domain Rule |
| 差分分類 | FlowSaveService |
| Entity取得 | Repository |
| Transaction | FlowSaveService |
| SaveChanges | FlowSaveService / UnitOfWork |
| Response作成 | FlowSaveService |

## 11. Export時の責務分担

| 処理 | 担当 |
| --- | --- |
| Export要求受信 | ExportController |
| 権限確認 | ProjectAuthorizationService |
| Flow / Snapshot取得 | FlowQueryService / VersionService |
| Export入力Model作成 | ExportService |
| 形式別生成 | Infrastructure Export Service |
| 結果返却 | ExportController |

## 12. 完了条件

- 各レイヤの責務が明確である。
- Controllerに業務ロジックが入らない。
- RepositoryにTransactionや業務判断が入らない。
- Domainが外部技術に依存しない。
- AIが本書を読んで責務分離を保った実装ができる。
