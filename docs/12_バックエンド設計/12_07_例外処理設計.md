# 12_07_例外処理設計

## 1. 目的

本書は、AI Flow Designer Backendにおける例外処理設計を定義する。

Backendでは、Controller / Service / Repository / EF Core / 認証処理 / Export処理 / 画像管理処理で発生する例外を、Frontendが一貫して扱える形式へ変換する必要がある。

本システムはSSOTを正とするため、例外処理は単なるエラー返却ではなく、構造化データの整合性を守るための重要機能として扱う。

## 2. 基本方針

- Controllerでtry-catchを多用しない。
- 例外はService層で意味付けする。
- Repository層では業務例外を生成しない。
- 共通Exception MiddlewareでAPIレスポンス形式へ変換する。
- EF Core例外を外部へ直接返さない。
- 内部例外の詳細はログへ記録する。
- APIレスポンスには必要最小限の情報のみ返す。
- Frontendで判定しやすいerrorCodeを必ず返す。
- 入力検証エラーと業務ルール違反を区別する。
- 想定外例外はInternalServerErrorとして扱う。

## 3. 例外分類

| 分類 | HTTP | 概要 |
| --- | ---: | --- |
| ValidationException | 400 | DTO入力値が不正 |
| BusinessRuleException | 409 | 業務ルール違反 |
| NotFoundException | 404 | 対象データなし |
| UnauthorizedException | 401 | 未認証 |
| ForbiddenException | 403 | 権限不足 |
| ConflictException | 409 | Revision競合・編集ロック競合 |
| FileStorageException | 500 | 画像・ファイル保存失敗 |
| ExportException | 500 | Mermaid / PDF / JSON / AI DSL出力失敗 |
| DatabaseException | 500 | DB永続化失敗 |
| UnexpectedException | 500 | 想定外例外 |

## 4. 共通エラーレスポンス形式

```json
{
  "success": false,
  "errorCode": "FLOW_REVISION_CONFLICT",
  "message": "Flow has been updated by another user.",
  "details": [
    {
      "field": "revision",
      "message": "Requested revision is older than current revision."
    }
  ],
  "traceId": "00-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx-xxxxxxxxxxxxxxxx-00"
}
```

## 5. フィールド定義

| 項目 | 必須 | 説明 |
| --- | --- | --- |
| success | 必須 | 失敗時はfalse |
| errorCode | 必須 | Frontend判定用コード |
| message | 必須 | 利用者または開発者向け概要 |
| details | 任意 | 項目別エラー |
| traceId | 必須 | ログ追跡用ID |

## 6. errorCode命名規則

```text
{DOMAIN}_{REASON}
```

例:

```text
PROJECT_NOT_FOUND
FLOW_NOT_FOUND
FLOW_REVISION_CONFLICT
FLOW_LOCKED_BY_OTHER_USER
NODE_INVALID_POSITION
LINK_INVALID_ENDPOINT
TEMPLATE_NAME_DUPLICATED
IMAGE_UPLOAD_FAILED
EXPORT_MERMAID_FAILED
```

Domain例:

- AUTH
- USER
- PROJECT
- FLOW
- LANE
- STAGE
- NODE
- LINK
- COMMENT
- TEMPLATE
- VERSION
- EXPORT
- IMAGE
- SETTING
- SYSTEM

## 7. ValidationException

DTOの入力値が不正な場合に使用する。

対象例:

- 必須項目なし
- 最大文字数超過
- GUID形式不正
- enum範囲外
- 座標値不正
- サイズが0以下
- LinkのSource / Target未指定
- NodeType未指定
- FlowId未指定

ValidationExceptionはFluentValidationまたは独自Validatorで検出する。

## 8. BusinessRuleException

業務ルール違反の場合に使用する。

対象例:

- Viewerが編集しようとした。
- Flowが編集ロック中。
- 削除対象LaneにNodeがあるが削除方法が未指定。
- Stage削除時にNode移動先が未指定。
- Template名が重複。
- Flow Version復元時に対象Versionが削除済。
- NodeTypeが無効。
- Linkの始点または終点Nodeが削除済。

## 9. NotFoundException

対象データが存在しない場合に使用する。

対象例:

- Projectが存在しない。
- Flowが存在しない。
- Nodeが存在しない。
- Linkが存在しない。
- Templateが存在しない。
- FlowVersionが存在しない。

論理削除済データも、通常APIではNotFoundとして扱う。

## 10. ConflictException

同時編集やRevision不一致を扱う。

対象例:

- Flow保存時のrevisionが最新でない。
- 編集ロックを他ユーザーが保持している。
- Version復元中にFlowが更新された。
- Template複写時に同名が作成された。

Frontendでは再読込・差分確認・キャンセル導線を表示する。

## 11. DatabaseException

DBアクセス失敗時に使用する。

DbUpdateExceptionをそのままControllerへ返してはならない。
Service層または共通MiddlewareでDatabaseExceptionへ変換する。

ログには以下を記録する。

- DbContext操作名
- 対象テーブル
- 対象ID
- ユーザーID
- traceId
- InnerException

APIレスポンスへSQL文や接続文字列を返してはならない。

## 12. FileStorageException

画像・ファイル保存で失敗した場合に使用する。

対象例:

- 画像アップロード失敗
- ストレージ書き込み失敗
- ファイル移動失敗
- Content-Type不正
- サイズ超過

## 13. ExportException

Export処理で失敗した場合に使用する。

対象例:

- Mermaid生成失敗
- PDF生成失敗
- JSON生成失敗
- AI DSL生成失敗
- Snapshot読み込み失敗

Export対象のflowId、snapshotId、exportTypeをログへ記録する。

## 14. Exception Middleware

Middlewareは以下を担当する。

1. 例外を捕捉する。
2. traceIdを取得する。
3. 例外種別をHTTPステータスへ変換する。
4. 標準ErrorResponseを生成する。
5. 必要情報をログ出力する。

Controllerでは原則try-catchしない。

## 15. ログ連携

例外発生時は以下をログへ出力する。

- traceId
- userId
- projectId
- flowId
- action
- errorCode
- exceptionType
- message
- elapsedMs

OAuth token、JWT、接続文字列、個人情報は出力しない。

## 16. テスト観点

- ValidationExceptionが400で返る。
- NotFoundExceptionが404で返る。
- ForbiddenExceptionが403で返る。
- ConflictExceptionが409で返る。
- UnexpectedExceptionが500で返る。
- ErrorResponseにtraceIdが含まれる。
- DbUpdateExceptionがDatabaseExceptionへ変換される。
- StackTraceがAPIレスポンスへ出ない。

## 17. 完了条件

- 例外分類とHTTPステータスが定義されている。
- 共通ErrorResponse形式が定義されている。
- Exception Middlewareの責務が明確である。
- FrontendがerrorCodeで分岐できる。
- AIが本書を読んで例外処理を実装できる。
