# 12_20_バックエンド設計まとめ

## 1. 目的

本書は、12_バックエンド設計の総まとめである。

Backendは、AI Flow DesignerのSSOTを守る中核であり、保存、検証、Version Snapshot、Export、認証・認可、画像管理、Template、ログ、例外、テストを担う。

## 2. Backendの責務

Backendが担当するもの:

- GitHub OAuth認証
- JWT認証
- Project単位認可
- Project / Flow管理
- Lane / Stage / Node / Link / Comment保存
- Flow構造Validation
- Version Snapshot作成・復元
- Template作成・適用
- Mermaid / PDF / JSON / AI DSL Export
- 画像アップロード・取得・削除
- Transaction管理
- 例外処理
- ログ出力
- テスト容易性確保

Backendが担当しないもの:

- Frontend Canvas表示
- Frontend一時状態管理
- Undo / Redo履歴管理
- Canvas DOMからのPDF生成

## 3. レイヤ構成

```text
Api
 -> Application
 -> Domain
Infrastructure
 -> Application / Domain
```

原則:

- Controllerは薄くする。
- Serviceが業務処理とTransactionを管理する。
- Repositoryは永続化処理のみ担当する。
- DomainはSSOTの意味とルールを表す。
- Infrastructureは外部技術依存を閉じ込める。

## 6. 実装チェックリスト

- [ ] ControllerからRepositoryを直接呼ばない。
- [ ] ControllerからDbContextを直接呼ばない。
- [ ] EntityをAPI Responseへ直接返さない。
- [ ] RepositoryでSaveChangesしない。
- [ ] RepositoryでTransactionを開始しない。
- [ ] Serviceで認可確認を行う。
- [ ] ServiceでTransaction境界を管理する。
- [ ] Flow保存前に構造Validationを行う。
- [ ] LinkのSource / Target整合性を検証する。
- [ ] Loopを禁止しない。
- [ ] Exportは保存済みFlowまたはSnapshotを対象とする。
- [ ] SnapshotはBackendで作成する。
- [ ] OAuth token / JWTをログに出さない。
- [ ] traceIdをErrorResponseに含める。
