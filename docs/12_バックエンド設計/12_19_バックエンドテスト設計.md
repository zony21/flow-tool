# 12_19_バックエンドテスト設計

## 1. 目的

本書は、AI Flow Designer Backendにおけるテスト設計を定義する。

BackendはSSOTの保存、検証、Version Snapshot、Export、認証・認可、Transaction、画像管理を担当するため、単体テストだけでは品質を担保できない。Domain、Service、Repository、API、Export、Transactionを分けて検証する。

## 2. テスト分類

| 分類 | 対象 |
| --- | --- |
| Unit Test | Domain Rule / Validator / Mapper |
| Service Test | Application Service |
| Repository Test | EF Core / SQLite |
| API Test | Controller / Middleware |
| Transaction Test | Rollback / Commit |
| Export Test | Mermaid / PDF / JSON / AI DSL |
| Auth Test | GitHub OAuth / JWT / 認可 |
| Performance Test | 大規模Flow |

## 3. Unit Test

対象:

- FlowStructureValidator
- NodeTypeValidator
- LinkValidator
- SnapshotSerializer
- ExportInputBuilder
- DTO Mapper

観点:

- Linkが存在しないNodeを参照するとError。
- Loopは許可される。
- NodeId重複を検出する。
- decision Nodeの条件不足を検出する。
- image NodeのimageFileId不足を検出する。

## 4. Service Test

対象:

- ProjectService
- FlowSaveService
- FlowQueryService
- FlowVersionService
- TemplateService
- ExportService
- ImageFileService
- ProjectAuthorizationService

観点:

- FlowSaveServiceが認可確認を行う。
- FlowSaveServiceがRevision競合を検出する。
- FlowSaveServiceがTransactionを利用する。
- Template適用時にID再採番される。
- Version復元時に現在状態を保存してから復元する。
- ExportServiceがSnapshot指定を優先する。

## 5. Repository Test

RepositoryはSQLiteのテストDBを利用して検証する。

観点:

- FlowId単位でNode / Link / Commentを一括取得できる。
- 論理削除済データが通常検索に出ない。
- RepositoryでSaveChangesを呼ばない。
- ProjectMember権限確認ができる。
- VersionNo最新値を取得できる。

## 6. API Test

対象:

- AuthController
- ProjectController
- FlowController
- VersionController
- ExportController
- ImageController

観点:

- 未認証で401になる。
- 権限不足で403になる。
- Validation Errorが400になる。
- NotFoundが404になる。
- Conflictが409になる。
- ErrorResponseにtraceIdが含まれる。
- EntityがResponseへ直接出ない。

## 7. Transaction Test

観点:

- Flow保存成功時に全要素が保存される。
- Link保存失敗時にNodeもRollbackされる。
- Template適用失敗時にFlowが変更されない。
- Version復元失敗時に現在Flowが保持される。
- 画像メタデータ保存失敗時に補償処理が行われる。

## 8. Export Test

観点:

- Mermaid Flowchartを生成できる。
- Mermaid Sequenceを生成できる。
- JSONにSchemaVersionが含まれる。
- AI DSLにLane責務、Stage目的、Node処理、Link条件が含まれる。
- SnapshotからExportできる。
- 未対応ExportTypeでExportExceptionになる。

## 9. Auth Test

観点:

- JWTがない場合401。
- JWT期限切れで401。
- ViewerがFlow保存すると403。
- EditorがFlow保存できる。
- ViewerがExportできる。
- ProjectMemberでないユーザーはProjectにアクセスできない。
- OAuth tokenやJWTがログに出ない。

## 10. Performance Test

初期目標:

- 1000Node / 10000Link Flow読込 3秒以内。
- 1000Node / 10000Link Flow保存 5秒以内。
- PDF出力 10秒以内。

CIで毎回実行する必要はないが、Release前に実行する。

## 11. テストデータ

用意するFlow:

- 最小Flow
- Lane / StageありFlow
- 分岐ありFlow
- LoopありFlow
- 大規模Flow
- image NodeありFlow
- Validation ErrorありFlow
- SnapshotありFlow

## 13. 完了条件

- Unit / Service / Repository / API / Transaction / Export / Authのテスト方針が定義されている。
- SSOT保存・検証・Exportの品質を検証できる。
- AIが本書を読んでBackendテストを実装できる。
