# 12_06_トランザクション設計

## 目的
Flow保存時にLane・Stage・Node・Link・Commentを一括保存し、整合性を保証する。

## 基本方針
- Service層でトランザクション開始
- Repositoryでは開始しない
- Entity Framework CoreのTransactionを利用
- 失敗時は全件ロールバック

## 保存対象
1. Flow
2. Lane
3. Stage
4. Node
5. Link
6. Comment

## 処理フロー
1. 入力検証
2. Transaction開始
3. Flow更新
4. Lane更新
5. Stage更新
6. Node更新
7. Link更新
8. Comment更新
9. Commit

例外発生時はRollbackする。

## 実装規約
- SaveChangesはServiceで集約
- Repositoryは永続化処理のみ担当
- ネストしたTransactionは禁止
