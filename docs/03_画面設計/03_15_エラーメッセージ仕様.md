# 03_15_エラーメッセージ仕様

## 1. 本書の目的

本書は、AI Flow Designer の画面で表示するエラーメッセージ仕様を定義する。
エラー時に利用者が次に何をすればよいか判断できることを目的とする。

## 2. 画面概要

エラー表示はToast、画面内メッセージ、ErrorDialogを使い分ける。
内部例外、スタックトレース、DBエラー詳細をそのまま表示してはならない。

## 3. 利用者

- 全利用者

## 4. 権限

権限不足時は、操作不可理由を明確に表示する。
ただし、権限の内部構造や不要なセキュリティ情報は表示しない。

## 5. レイアウト

```text
[Toast]
[画面内エラー領域]
[ErrorDialog]
[入力項目ごとのエラー]
```

## 6. 表示項目

| 種別 | 表示場所 | 用途 |
| --- | --- | --- |
| Toast | 画面右上 | 一時通知 |
| InlineError | 入力項目下 | バリデーションエラー |
| ErrorDialog | 中央 | 復旧が必要な重大エラー |
| Banner | 画面上部 | セッション切れ、通信障害 |

## 7. 入力項目

エラーメッセージ仕様自体に入力項目はない。

## 8. ボタン

ErrorDialogでは以下を使用する。

- 閉じる
- 再試行
- ログインへ戻る

## 9. ダイアログ

重大エラーやセッション切れではErrorDialogを使用する。
入力エラーではダイアログを使用せず、項目下に表示する。

## 10. バリデーション

画面ごとのバリデーション結果を、対象項目に紐付けて表示する。
複数エラーがある場合は、最初のエラー項目へフォーカスする。

## 11. API

APIから返るエラーコードを画面メッセージへ変換する。

- 400: 入力値が不正です
- 401: ログイン状態が切れました
- 403: 操作権限がありません
- 404: 対象データが存在しません
- 409: 他の更新と競合しました
- 500: 処理中にエラーが発生しました

## 12. DB

DB例外はAPI層で画面用エラーへ変換する。
画面にDB例外名を表示しない。

## 13. 状態遷移

```text
操作実行
  ↓
成功 -> 成功Toast
失敗 -> エラー種別判定
  ├ 入力エラー
  ├ 権限エラー
  ├ セッション切れ
  └ システムエラー
```

## 14. エラー処理

代表メッセージは以下とする。

- 保存に失敗しました
- 入力値を確認してください
- 対象データが存在しません
- 操作権限がありません
- ログイン状態が切れました
- 通信に失敗しました

## 15. 未保存データ

保存失敗時は未保存状態を維持する。
利用者が再保存できる状態を保つ。

## 16. テスト観点

- 入力エラーが項目下に表示されること
- APIエラーが適切なメッセージに変換されること
- 401でログイン誘導されること
- 保存失敗時に編集内容が消えないこと
- 内部例外が表示されないこと

## 17. 完了条件

本仕様は、画面横断のエラー表示、APIエラー変換、未保存保持方針が明確になった場合に完了とする。
