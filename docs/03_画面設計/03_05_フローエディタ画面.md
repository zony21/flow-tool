# 03_05_フローエディタ画面

## 1. 本書の目的

本書は、AI Flow Designer の中核であるフローエディタ画面の詳細仕様を定義する。
フローエディタ画面は、担当レーン、工程レーン、ノード、接続、コメントを編集し、AIが理解可能な構造化データを作成する画面である。

## 2. 画面の役割

本画面の役割は以下である。

- Laneの作成・編集・削除
- Stageの作成・編集・削除
- Nodeの作成・編集・移動・削除
- Linkの作成・編集・削除
- Commentの作成・編集・削除
- FlowVersionの保存
- Mermaid、PDF、JSON出力への導線
- Undo/Redo操作
- 将来のAIレビュー導線

## 3. レイアウト

画面はExcel風のエディタ構成とする。

```text
[上部ツールバー]
[左: ノードパレット] [中央: フローキャンバス] [右: プロパティパネル]
                     [上: 担当レーン]
                     [左: 工程レーン]
```

## 4. 上部ツールバー

ツールバーには以下を配置する。

- 戻る
- 保存
- バージョン作成
- Undo
- Redo
- ズームイン
- ズームアウト
- 表示リセット
- Mermaid出力
- PDF出力
- JSON出力
- 将来AIレビュー

保存ボタンは未保存変更がある場合に強調表示する。

## 5. ノードパレット

ノードパレットには初期ノード種別を表示する。

- 開始
- 終了
- 処理
- 判定
- 六角形
- 吹き出し
- 画像

将来的にNodeType管理画面から種別を追加できるようにする。

## 6. フローキャンバス

キャンバスはVue Flowで実装する。
以下の操作を提供する。

- ノード配置
- ノード移動
- ノード選択
- 複数選択
- 接続線作成
- ズーム
- パン
- 範囲選択

キャンバス上の見た目は構造化データの表示結果であり、正は保存データである。

## 7. 担当レーン

担当レーンは上方向に表示する。
担当レーンは機器、システム、担当者、外部システムを表す。

保持情報は以下とする。

- 担当名
- カテゴリ
- アイコン画像
- 色
- 説明
- 表示順

## 8. 工程レーン

工程レーンは左方向に表示する。
工程は処理段階、時系列、業務工程を表す。

保持情報は以下とする。

- 工程名
- 説明
- 表示順
- 色

## 9. ノード操作

### 9.1 ノード追加

ノードは以下の方法で追加する。

- ノードパレットからドラッグ＆ドロップ
- キャンバスのダブルクリック

追加時には内部IDを採番し、表示名とは分離する。

### 9.2 ノード移動

ノード移動時は座標を更新する。
座標から担当レーンと工程を判定できる場合でも、保存上はLane IDとStage IDを明示的に保持する。

### 9.3 ノード削除

ノード削除時は確認を行う。
接続されているLinkがある場合は、関連Linkも削除するか確認する。

## 10. 接続操作

接続はノード間の矢印として作成する。
Linkは以下を保持する。

- FROM Node ID
- TO Node ID
- データ名
- 通信種別
- 条件
- 説明

ループ接続は許可する。
接続線は初期状態でBezierを使用する。
将来的にStraight、Step、SmoothStepへ切替可能にする。

## 11. コメント操作

コメントは以下の2種類を扱う。

- 独立コメント
- ノード紐付けコメント

通常コメントとは別に、AIだけが読むAIメモを保持できるようにする。

## 12. プロパティパネル

右側プロパティパネルは、選択対象に応じて表示内容を切り替える。

### 12.1 Node選択時

- 表示名
- ノード種別
- 担当レーン
- 工程
- 処理種別
- 説明
- AIメモ
- 座標
- サイズ

### 12.2 Link選択時

- FROM
- TO
- データ名
- 通信種別
- 条件
- 説明

### 12.3 Lane選択時

- 担当名
- カテゴリ
- 色
- アイコン
- 説明
- 表示順

### 12.4 Stage選択時

- 工程名
- 説明
- 色
- 表示順

### 12.5 Comment選択時

- コメント本文
- 紐付けNode
- AI向けフラグ
- 表示位置

## 13. レーン・工程削除仕様

LaneまたはStageを削除する場合、対象にNodeが存在するときは以下を選択させる。

- ノードも削除
- 別レーンまたは別工程へ移動
- キャンセル

自動削除は禁止する。

## 14. 保存仕様

保存時には以下を一括保存する。

- FlowVersion
- Lane
- Stage
- Node
- Link
- Comment
- 座標
- サイズ
- 表示順

保存失敗時は画面上の編集状態を破棄しない。

## 15. Undo/Redo

Undo/RedoはCommand Patternで実装する。
対象操作は以下とする。

- Node追加
- Node移動
- Node削除
- Link追加
- Link削除
- Lane追加
- Lane削除
- Stage追加
- Stage削除
- プロパティ変更

## 16. 出力導線

ツールバーから以下へ遷移または実行できる。

- Mermaid flowchart
- Mermaid sequenceDiagram
- PDF
- JSON

将来的にAI DSL、API仕様、DB更新一覧、通信一覧、PLC一覧、設計書ドラフト、コード生成へ拡張する。

## 17. API連携

使用APIは以下とする。

- GET /api/projects/{projectId}/flows/{flowId}/versions/latest
- POST /api/projects/{projectId}/flows/{flowId}/versions
- PUT /api/flow-versions/{flowVersionId}
- POST /api/flow-versions/{flowVersionId}/export/mermaid
- POST /api/flow-versions/{flowVersionId}/export/pdf
- POST /api/flow-versions/{flowVersionId}/export/json

## 18. DB連携

主に以下を使用する。

- FLOW
- FLOW_VERSION
- LANE
- STAGE
- NODE
- LINK
- COMMENT
- NODE_TYPE

## 19. エラー表示

想定エラーは以下とする。

- フローの読み込みに失敗しました
- 保存に失敗しました
- 接続元または接続先が不正です
- 対象ノードが存在しません
- 出力に失敗しました
- ログイン状態が切れました

## 20. テスト観点

- ノードを追加できること
- ノードを移動できること
- ノード属性を編集できること
- Linkを作成できること
- ループ接続を作成できること
- Lane削除時に選択肢が表示されること
- Stage削除時に選択肢が表示されること
- 保存後に再読込して構造が維持されること
- Mermaid、PDF、JSON出力へ遷移できること
- Undo/Redoが動作すること

## 21. 完了条件

本画面は、構造化データを正としてFlowを作成・編集・保存・出力できる画面仕様が定義され、実装者がVue3、PrimeVue、Vue Flow、Piniaで実装可能な粒度になった場合に完了とする。
