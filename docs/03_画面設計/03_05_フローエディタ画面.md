# 03_05_フローエディタ画面

## 1. 本書の目的

本書は、AI Flow Designer の中核であるフローエディタ画面の詳細仕様を定義する。
フローエディタ画面は、担当レーン、工程レーン、ノード、接続、コメントを編集し、AIが理解可能な構造化データを作成する画面である。

## 2. 画面の役割

本画面の役割は以下である。

- Laneの作成・編集・削除
- Stageの作成・編集・削除
- Nodeの作成・編集・移動・削除
- Linkの作成・編集・削除
- Commentの作成・編集・削除
- FlowVersionの保存
- Mermaid、PDF、JSON出力への導線
- Undo/Redo操作
- 将来のAIレビュー導線
- AGF/AGV搬送フロー時の搬送属性編集
- AGF/AGV搬送フロー時のロケーション管理導線
- AGF/AGV搬送フロー時の搬送表出力導線

## 3. レイアウト

画面はExcel風のエディタ構成とする。

```text
[上部ツールバー]
[左: ノードパレット] [中央: フローキャンバス] [右: プロパティパネル]
                     [上: 担当レーン]
                     [左: 工程レーン]
```

## 4. 上部ツールバー

ツールバーには以下を配置する。

- 戻る
- 保存
- バージョン作成
- Undo
- Redo
- ズームイン
- ズームアウト
- 表示リセット
- 出力
- コメント
- 設備/分類設定
- 将来AIレビュー

保存ボタンは未保存変更がある場合に強調表示する。

出力は1つの導線に集約し、押下時に出力ダイアログを表示する。

Transport Flowの場合のみ、ロケーション管理および搬送表出力を利用可能にする。

## 5. ノードパレット

ノードパレットには初期ノード種別を表示する。

- 開始
- 終了
- 処理
- 判定
- 六角形
- 吹き出し
- 画像

将来的にNodeType管理画面から種別を追加できるようにする。

## 6. フローキャンバス

キャンバスはVue Flowで実装する。
以下の操作を提供する。

- ノード配置
- ノード移動
- ノード選択
- 複数選択
- 接続線作成
- ズーム
- パン
- 範囲選択

キャンバス上の見た目は構造化データの表示結果であり、正は保存データである。

## 7. 担当レーン

担当レーンは上方向に表示する。
担当レーンは機器、システム、担当者、外部システムを表す。

保持情報は以下とする。

- 担当名
- カテゴリ
- アイコン画像
- 色
- 説明
- 表示順

## 8. 工程レーン

工程レーンは左方向に表示する。
工程は処理段階、時系列、業務工程を表す。

保持情報は以下とする。

- 工程名
- 説明
- 表示順
- 色

## 9. ノード操作

### 9.1 ノード追加

ノードは以下の方法で追加する。

- ノードパレットからドラッグ＆ドロップ
- キャンバスのダブルクリック

追加時には内部IDを採番し、表示名とは分離する。

### 9.2 ノード移動

ノード移動時は座標を更新する。
座標から担当レーンと工程を判定できる場合でも、保存上はLane IDとStage IDを明示的に保持する。

### 9.3 ノード削除

ノード削除時は確認を行う。
接続されているLinkがある場合は、関連Linkも削除するか確認する。

## 10. 接続操作

接続はノード間の矢印として作成する。
Linkは以下を保持する。

- FROM Node ID
- TO Node ID
- データ名
- 通信種別
- 条件
- 説明

ループ接続は許可する。
接続線は初期状態でBezierを使用する。
将来的にStraight、Step、SmoothStepへ切替可能にする。

## 11. コメント操作

コメントは以下の2種類を扱う。

- 独立コメント
- ノード紐付けコメント

通常コメントとは別に、AIだけが読むAIメモを保持できるようにする。

コメント種別は以下を扱う。

- 補足
- 注意
- 未確定
- 例外
- API

## 12. プロパティパネル

右側プロパティパネルは、選択対象に応じて表示内容を切り替える。

### 12.1 Node選択時

- 表示名
- ノード種別
- 担当レーン
- 工程
- 処理種別
- 説明
- AIメモ
- 座標
- サイズ

FlowTypeがTransportの場合のみ、以下を追加表示する。

- メーカーコマンド
- ロケーション
- 設備
- R/W
- 処理区分

### 12.2 Link選択時

- FROM
- TO
- データ名
- 通信種別
- 条件
- 説明

### 12.3 Lane選択時

- 担当名
- カテゴリ
- 色
- アイコン
- 説明
- 表示順

### 12.4 Stage選択時

- 工程名
- 説明
- 色
- 表示順

### 12.5 Comment選択時

- コメント本文
- 紐付けNode
- 種別
- AI向けフラグ
- 表示位置

## 13. レーン・工程削除仕様

LaneまたはStageを削除する場合、対象にNodeが存在するときは以下を選択させる。

- ノードも削除
- 別レーンまたは別工程へ移動
- キャンセル

自動削除は禁止する。

## 14. 保存仕様

保存時には以下を一括保存する。

- FlowVersion
- Lane
- Stage
- Node
- Link
- Comment

Transport Flowの場合は、Nodeの搬送属性も保存対象に含める。

## 15. FlowType制御

Flowは以下の種別を持つ。

| 種別 | 説明 |
| --- | --- |
| Normal | 通常フロー |
| Transport | AGF/AGV搬送フロー |

既存FlowはNormalとして扱う。

Normalでは通常のFlow編集機能のみを表示する。
Transportでは通常機能に加え、搬送属性編集、ロケーション管理、搬送表出力を表示する。

## 16. ロケーション管理ダイアログ

ロケーション管理はフローエディタ内のダイアログとして提供する。

理由は、ロケーションは搬送フロー編集中に追加・修正される頻度が高く、Node詳細から即時利用できる必要があるためである。

ダイアログで扱う項目は以下とする。

| 項目 | 説明 |
| --- | --- |
| ロケーション名 | P1、A1、ST1など |
| 種類 | 経由点、荷役場所、充電位置、待機場所など |
| 説明 | 任意補足 |
| 表示順 | 選択肢表示順 |

## 17. 搬送表出力

Transport Flowのみ搬送表出力を利用可能とする。

生成処理は以下とする。

1. LinkをたどってNode実行順を決定する
2. 実行順にNoを採番する
3. Node名を処理欄へ出力する
4. Nodeのメーカーコマンドを動作欄へ出力する
5. Nodeのロケーションをロケ欄へ出力する
6. Nodeの設備を設備欄へ出力する
7. R/WをReadまたはWriteとして出力する
8. コマンドまたはR/Wから処理区分を判定する

出力形式はMarkdownを初期対応とし、将来的にExcel、PDFへ拡張する。

## 18. テスト観点

- Normal FlowではTransport専用項目が表示されないこと
- Transport FlowではNode詳細に搬送属性が表示されること
- ロケーション管理ダイアログで追加したロケーションをNode詳細で選択できること
- 搬送表出力がTransport Flowのみ表示されること
- 保存後に搬送属性が失われないこと

## 19. 完了条件

フローエディタがNormal FlowとTransport Flowを明確に切り替え、通常フローの編集品質を維持したままAGF/AGV搬送表生成に必要な情報を構造化できること。
