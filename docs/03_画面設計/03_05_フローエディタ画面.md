# 03_05_フローエディタ画面

## Transport ロケーション設定仕様（追記）

## 基本方針

ロケーションはTransport Flow専用情報とする。

ロケーションはAGF/AGVが動作する位置情報であり、Node単位で設定する。

例:

- P1（経由点）
- A1（荷役位置）
- ST1（充電位置）


## 設定場所

FlowType = Transport の場合のみ、Node詳細パネルに表示する。

Node選択時:

通常項目
+
Transport属性

- メーカーコマンド
- ロケーション
- 設備
- R/W
- 処理区分


## 操作仕様

1. ユーザーがNodeを選択
2. 右側詳細パネルを表示
3. ロケーションプルダウンから選択
4. Node.locationIdとして保存


## ロケーション追加

ロケーションは独立管理画面ではなく、FlowEditor内Dialogで追加する。

理由:

搬送設計中に新しい停止位置・荷役位置が発生するため。

例:

1. Node作成
2. P10へ移動を作りたい
3. P10が未登録
4. Node詳細からロケーション追加Dialogを開く
5. P10登録
6. 登録後そのままNodeへ設定可能


## 保存責務

ロケーションマスタ:

TRANSPORT_LOCATION

保持:

- locationId
- projectId
- name
- locationType
- description


Node:

保持:

- locationId


## 搬送テンプレート連携

搬送表生成時:

Node
↓
locationId
↓
TRANSPORT_LOCATION取得
↓
ロケ欄へ出力


重要:

ロケーション表を直接編集して搬送設計を管理しない。

正:
Flow + Node + Transport属性

出力:
搬送テンプレート
