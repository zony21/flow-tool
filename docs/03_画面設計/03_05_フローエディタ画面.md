# 03_05 Flow Editor Screen Design

## 1. Purpose

The Flow Editor edits the structured Flow model and visualizes Lane, Stage, Node, Link, and Comment data.

The structured model is the Single Source of Truth. Screen layout and exported files are not authoritative data sources.

## 2. Transport Flow Display

Transport-specific controls are displayed only when `FlowType = TRANSPORT`.

Normal Flow must not display or persist Transport-specific controls.

## 3. Node Detail Transport Section

The Node Detail panel may contain attributes that describe the action executed by the selected Node.

Allowed Node-level Transport attributes:

- Command
- Equipment
- R/W Type

Location must not be selected, created, or stored from Node Detail.

## 4. Location Responsibility

A Location represents the physical position of client equipment or a facility point within a Project.

Examples:

- Conveyor loading position
- AGF pickup position
- Charging station position
- Inspection station position
- Warehouse entrance position

Location is Project-level equipment/facility master data.

Location is not a Node property.

## 5. Prohibited Node Detail Behavior

The following functions must not be implemented in Node Detail:

- Location selection
- Create New Location button
- Location creation dialog
- Automatic assignment of a newly created Location to a Node
- Saving `Node.LocationId`

## 6. Location Management

Location must be managed as Project-level client equipment/facility position data.

The management entry point must be provided outside Node Detail, such as Project Transport Settings or Equipment/Location settings.

The exact management screen structure will be defined separately.

## 7. Transport Template Relationship

Transport Template generation must not derive Location from Node screen layout or Node coordinates.

When a generated template requires a physical location, the location must be resolved from the Project-level equipment/facility structure associated with the transport process.

The resolution rule must be defined by the Transport Template Generator specification before implementation.

## 8. Authoritative Rule

Correct:

```text
Project
  -> Client Equipment / Facility Position
  -> Location Master

Flow
  -> Node
  -> Command / Equipment / R/W Type
```

Incorrect:

```text
Node
  -> LocationId
```

This rule supersedes the previous design that assigned Location directly from Node Detail.
