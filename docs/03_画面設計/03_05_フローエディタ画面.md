# 03_05 Flow Editor Screen Design

## 1. Purpose

The Flow Editor edits the structured Flow model. The structured model is the Single Source of Truth; screen layout and exported files are derived representations.

## 2. Transport Flow Display

Transport controls are displayed only when `FlowType = TRANSPORT`.

Normal Flow does not display or persist `CommandId`, `EquipmentId`, `LocationId`, or Transport R/W values. Normalization stores null master references and `RwType = NONE`.

## 3. Location Responsibility

Location is Project-level Transport master data. A Location represents a reusable physical point such as a pickup position, loading position, charging station, or waiting position.

Project Transport Settings is responsible for creating, editing, activating, deactivating, and deleting Location master data.

A Transport Node does not create or own a Location. It may reference one existing active Project Location through `Node.LocationId`.

```text
Project Transport Settings
  -> Manage Location master data

Node Detail
  -> Select an existing Location
```

## 6. Node Detail Transport Section

For a Transport Flow, Node Detail displays selectors for the supported Transport references and R/W value.

The section may include:

```text
Command
Location
Equipment
R/W Type
```

### 6.1 Command Selector

The Command selector is filtered by the Manufacturer Vehicle Types assigned to the Node's Action Unit.

Available options are active, non-deleted Commands whose `ManufacturerVehicleTypeId` belongs to the Action Unit assignment set.

Recommended display:

```text
Manufacturer A / AGF / TravelToPosture
Manufacturer A / AGF / Loading
Manufacturer B / AGV / MoveTask
```

The stored Node value remains:

```text
Node.CommandId
```

Manufacturer and Vehicle Type are resolved through the Command relationship.

If no Manufacturer Vehicle Type is assigned to the Action Unit:

A Command outside the Action Unit assignment scope must not be newly saved.

An existing inactive Command reference remains visible with an inactive indication and must not be silently cleared.

### 6.2 Location Selector

The Location selector:

- loads valid Locations belonging to the current Project;
- updates the selected `Node.LocationId`;
- is disabled in read-only mode;
- remains hidden for Normal Flow.

Node Detail is not the primary Location master management screen and does not provide Location creation in this implementation cycle.

### 6.3 Equipment Selector

Equipment remains Project-level facility or system data.

The current `Node.EquipmentId` responsibility is not replaced by the global AGF / AGV Manufacturer Vehicle Type assignment in this cycle.

Do not use Equipment to infer Manufacturer, Vehicle Type, or Command availability.

## 7. Save and Reload

Changing the Location or Command selector updates the corresponding Node reference in the in-memory Flow and marks the Flow dirty. It does not automatically save the Flow.

Action Unit Manufacturer Vehicle Type assignments are structured Flow data and must be preserved through the Flow save and load path.

Reloading the saved Transport Flow restores:

```text
Action Unit Manufacturer Vehicle Type assignments
Node.CommandId
Node.LocationId
Node.EquipmentId
Node.RwType
```

## 8. Version Snapshot and Duplication

The Flow Version Snapshot preserves the Manufacturer Vehicle Type IDs assigned to each Action Unit.

Flow duplication:

```text
regenerates Action Unit IDs
regenerates assignment IDs
remaps assignments to duplicated Action Units
retains global ManufacturerVehicleTypeId references
retains valid Node.CommandId references
```

## 9. Transport Template Relationship

The future Transport Template Generator resolves:

```text
Action
  <- Node.CommandId
  <- Command belonging to Manufacturer + Vehicle Type

Location
  <- Node.LocationId
  <- referenced TransportLocation display value or code
```

Location must not be inferred from Node coordinates, Equipment, Lane, Stage, or screen layout.

Command Manufacturer or Vehicle Type must not be inferred from Location or Equipment.

## 10. Test Conditions

Confirm that:

- one Action Unit can assign multiple Manufacturer Vehicle Types;
- duplicate assignment is prevented;
- inactive records cannot be newly assigned;
- existing inactive assignments remain visible;
- the Node Command selector contains only Commands from assigned Manufacturer Vehicle Types;
- Command options display Manufacturer and Vehicle Type;
- a Command outside the assignment scope is rejected during save;
- assignments survive save, reload, Version Snapshot, and duplication;
- existing Location behavior remains unchanged;
- Normal Flow behavior remains unchanged.
