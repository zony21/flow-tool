# 03_16_設備管理画面

## 1. 本書の目的

本書は、設備管理画面の仕様を定義する。

設備管理画面は、Transport Flowおよび設備連携フローで利用する実設備をProject単位で管理する画面である。

## 2. 画面の役割

本画面の役割は以下である。

- 設備一覧表示
- 設備追加
- 設備編集
- 設備削除
- 設備カテゴリ管理

## 3. 表示項目

| 項目 | 説明 |
| --- | --- |
| 設備名 | SH1、PLC01、WCS、RCS、A1コンベアなど |
| カテゴリ | PLC、RCS、WCS、AGF、コンベア等 |
| 説明 | 任意補足 |
| 更新日時 | 最終更新日時 |

## 4. 設備カテゴリ

初期カテゴリは以下とする。

- PLC
- RCS
- WCS
- AGF
- AGV
- AMR
- コンベア
- シャッター
- ロボット
- センサー
- 安全機器
- その他

カテゴリは将来的にユーザー追加可能とする。

## 5. 操作

### 5.1 追加

設備名、カテゴリ、説明を登録する。

設備名とカテゴリは必須とする。

### 5.2 編集

設備名、カテゴリ、説明を変更する。

Transport Nodeで利用中の設備も編集可能とするが、削除は不可とする。

### 5.3 削除

未使用設備のみ削除可能とする。

## 6. 画面遷移

```text
プロジェクト詳細
  ↓
設備管理
```

## 7. API

利用APIはEquipment APIとする。

- GET /api/projects/{projectId}/equipments
- POST /api/projects/{projectId}/equipments
- PUT /api/projects/{projectId}/equipments/{equipmentId}
- DELETE /api/projects/{projectId}/equipments/{equipmentId}

## 8. DB

主にEQUIPMENTを利用する。

## 9. テスト観点

- 設備を追加できること
- 設備カテゴリを設定できること
- Nodeで利用中の設備を削除できないこと
- Transport Node詳細で設備を選択できること

## 10. 完了条件

Project内の実設備を構造化して管理し、Transport FlowのNode詳細および搬送表生成に利用できる状態であること。
