# 03_11_ログイン画面

## 1. 本書の目的

本書は、ログイン画面の詳細仕様を定義する。
ログイン画面は、GitHub OAuthを利用してAI Flow Designerへログインする入口である。

## 2. 画面概要

未ログインユーザーが保護された画面へアクセスした場合、本画面へ誘導する。
ログイン成功後はプロジェクト一覧画面へ遷移する。

## 3. 利用者

- システム設計者
- 実装者
- レビュー担当者
- 管理者

## 4. 権限

未ログイン状態ではログイン操作のみ可能とする。
ログイン済みユーザーが本画面へアクセスした場合はプロジェクト一覧画面へリダイレクトする。

## 5. レイアウト

```text
[アプリ名]
[説明文]
[GitHubでログインボタン]
[利用上の注意]
[エラー表示]
```

## 6. 表示項目

| 項目 | 内容 |
| --- | --- |
| アプリ名 | AI Flow Designer |
| 説明文 | 構造化フロー設計ツールであることを説明 |
| ログインボタン | GitHub OAuth開始 |
| エラー表示 | OAuth失敗時のメッセージ |

## 7. 入力項目

本画面には手入力項目を設けない。
認証情報はGitHub OAuth画面で入力する。

## 8. ボタン

- GitHubでログイン

## 9. ダイアログ

初期実装ではダイアログを使用しない。
OAuth失敗時は画面内メッセージまたはToastで通知する。

## 10. バリデーション

手入力項目がないため画面側バリデーションは行わない。
OAuthコールバック時にstate検証を行う。

## 11. API

- GET /api/auth/github/login
- GET /api/auth/github/callback
- GET /api/auth/me
- POST /api/auth/logout

## 12. DB

- USER
- USER_LOGIN_HISTORY

## 13. 状態遷移

```text
未ログイン
  ↓ GitHubログイン
OAuth認可
  ↓ 成功
プロジェクト一覧
  ↓ 失敗
ログイン画面エラー表示
```

## 14. エラー処理

- GitHubログインに失敗しました
- 認証状態を確認できませんでした
- 利用許可されていないユーザーです
- セッションの作成に失敗しました

## 15. 未保存データ

ログイン画面では未保存データを扱わない。

## 16. テスト観点

- 未ログイン時にログイン画面が表示されること
- GitHubログインボタンでOAuthへ遷移すること
- ログイン成功後にプロジェクト一覧へ遷移すること
- ログイン失敗時にエラーが表示されること
- ログイン済みでアクセスした場合にリダイレクトされること

## 17. 完了条件

本画面は、GitHub OAuthによるログイン導線と失敗時の扱いが明確に定義された場合に完了とする。
