# 03_04_フロー一覧画面

## 1. 本書の目的

本書は、フロー一覧画面の詳細仕様を定義する。
フロー一覧画面は、プロジェクト配下に存在する作業単位のFlowを管理し、フローエディタへ遷移するための画面である。

## 2. 画面の役割

Flowは、プロジェクト内の作業単位を表す。
例として、RFID読取り、搬送要求、ラベル印刷、AGF搬送、異常停止などをFlowとして管理する。

本画面では以下を行う。

- Flow一覧表示
- Flow新規作成
- Flow編集
- Flow削除
- Flow複製
- テンプレートから作成
- フローエディタ画面への遷移

## 3. レイアウト

```text
[ヘッダー]
[パンくず: プロジェクト一覧 > 対象プロジェクト]
[画面タイトル: フロー一覧] [新規作成] [テンプレートから作成]
[検索・絞り込み領域]
[フロー一覧テーブル]
[ページング]
```

## 4. 表示項目

| 項目 | 内容 | 備考 |
| --- | --- | --- |
| フロー名 | Flow表示名 | 必須 |
| カテゴリ | 作業分類 | 任意 |
| 説明 | Flow説明 | 任意 |
| 最新バージョン | 最新FlowVersion | 自動表示 |
| ノード数 | 最新版のNode件数 | 自動集計 |
| 更新日時 | 最終更新日時 | 降順標準 |
| 操作 | 編集、複製、削除、出力 | ボタン表示 |

## 5. 操作仕様

### 5.1 新規作成

新規作成ボタン押下でFlow作成ダイアログを表示する。
入力項目は以下とする。

- フロー名
- カテゴリ
- 説明
- 初期テンプレート選択有無

作成時に初期FlowVersionを作成する。

### 5.2 テンプレートから作成

テンプレートから作成ボタン押下でテンプレート選択ダイアログを表示する。
テンプレートにはレーン、工程、ノードまで含めることができる。

### 5.3 編集

編集ボタン押下でフローエディタ画面へ遷移する。
対象Flowの最新FlowVersionを読み込む。

### 5.4 複製

複製ボタン押下で既存Flowを複製する。
複製対象には以下を含める。

- Lane
- Stage
- Node
- Link
- Comment
- FlowVersionの初期版

### 5.5 削除

削除ボタン押下で確認ダイアログを表示する。
初期実装では論理削除とし、過去バージョンの扱いはDB設計に従う。

## 6. バリデーション

| 項目 | 条件 |
| --- | --- |
| フロー名 | 必須、最大100文字 |
| カテゴリ | 最大100文字 |
| 説明 | 最大1000文字 |

## 7. 状態管理

Piniaで以下を管理する。

- 対象Project
- Flow一覧
- 選択中Flow
- 検索条件
- 読込中状態
- 保存中状態
- エラー状態

## 8. API連携

使用APIは以下とする。

- GET /api/projects/{projectId}/flows
- POST /api/projects/{projectId}/flows
- GET /api/flows/{flowId}
- PUT /api/flows/{flowId}
- DELETE /api/flows/{flowId}
- POST /api/flows/{flowId}/duplicate
- POST /api/projects/{projectId}/flows/from-template

## 9. DB連携

主に以下を使用する。

- PROJECT
- FLOW
- FLOW_VERSION
- LANE
- STAGE
- NODE
- LINK
- COMMENT
- TEMPLATE

## 10. エラー表示

想定エラーは以下とする。

- フロー一覧の取得に失敗しました
- フローの作成に失敗しました
- フローの更新に失敗しました
- フローの削除に失敗しました
- フローの複製に失敗しました
- テンプレートからの作成に失敗しました

## 11. テスト観点

- 対象プロジェクトのFlow一覧が表示されること
- Flowを新規作成できること
- 最新FlowVersionが作成されること
- フローエディタへ遷移できること
- Flowを複製できること
- テンプレートからFlowを作成できること
- 削除確認ダイアログが表示されること
- APIエラー時にメッセージが表示されること

## 12. 完了条件

本画面は、プロジェクト配下のFlowを管理し、フローエディタへ正しく接続できる粒度で仕様が定義された場合に完了とする。
