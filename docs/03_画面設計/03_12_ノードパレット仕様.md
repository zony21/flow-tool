# 03_12_ノードパレット仕様

## 1. 本書の目的

本書は、フローエディタ画面左側に配置するノードパレットの詳細仕様を定義する。
ノードパレットは、利用者が構造化データ上のNodeを作成する入口であり、単なる図形選択ではなく意味を持つNodeTypeを選択するためのUIである。

## 2. 画面概要

ノードパレットはフローエディタ画面の左側に固定表示する。
利用者はノード種別を選択し、キャンバスへドラッグ＆ドロップすることでNodeを追加する。

## 3. 利用者

- システム設計者
- 業務設計者
- レビュー担当者

## 4. 権限

参照のみ権限ではノードパレットを非活性または非表示にする。
編集権限がある場合のみノード追加を許可する。

## 5. レイアウト

```text
[検索]
[カテゴリ]
[ノード種別一覧]
  - 開始
  - 終了
  - 処理
  - 判定
  - 六角形
  - 吹き出し
  - 画像
```

## 6. 表示項目

| 項目 | 内容 |
| --- | --- |
| ノード種別名 | 開始、終了、処理など |
| アイコン | 種別を表すアイコン |
| 説明 | 種別の利用目的 |
| カテゴリ | 基本、通信、制御、注釈など |

## 7. 入力項目

- 検索キーワード
- カテゴリ絞り込み

## 8. ボタン

基本的にはボタンではなくドラッグ操作を主とする。
将来的に「選択して追加」ボタンを提供してもよい。

## 9. ダイアログ

画像ノードを選択した場合は、画像選択ダイアログへつなげる。
その他の基本ノードではダイアログを表示しない。

## 10. バリデーション

- 編集権限がない場合は追加不可
- NodeTypeが無効化されている場合は追加不可
- 画像ノードで画像未選択の場合は作成不可

## 11. API

- GET /api/node-types
- POST /api/flow-versions/{flowVersionId}/nodes

## 12. DB

- NODE_TYPE
- NODE
- FLOW_VERSION

## 13. 状態遷移

```text
パレット表示
  ↓
ノード種別選択
  ↓
キャンバスへドラッグ
  ↓
Node作成
  ↓
プロパティパネル表示
```

## 14. エラー処理

- ノード種別の取得に失敗しました
- ノードを追加できませんでした
- 編集権限がありません

## 15. 未保存データ

ノード追加後はフローエディタ全体を未保存状態にする。

## 16. テスト観点

- 初期ノード種別が表示されること
- 検索で絞り込めること
- ドラッグ＆ドロップでNodeを追加できること
- 追加後に未保存状態になること
- 編集権限なしでは追加できないこと

## 17. 完了条件

本仕様は、NodeTypeを選択してNodeを追加でき、追加結果が構造化データとして保持される仕様が定義された場合に完了とする。
