# 03_09_バージョン管理画面

## 1. 本書の目的

本書は、Flowのバージョン管理画面の詳細仕様を定義する。
FlowVersionは、設計変更履歴を保持し、AIや実装者がどの時点のフローを参照しているかを明確にするための重要情報である。

## 2. 画面概要

バージョン管理画面では、対象Flowに紐付くFlowVersionを一覧表示し、参照、複製、最新版化、差分確認、説明編集を行う。
初期実装では一覧・参照・新バージョン作成を優先する。

## 3. 利用者

- システム設計者
- レビュー担当者
- 実装者
- 管理者

## 4. 権限

初期実装ではログイン済みユーザーが操作可能とする。
将来的には、参照のみ、編集可能、管理者で操作を分ける。

## 5. レイアウト

```text
[ヘッダー]
[パンくず: プロジェクト > フロー > バージョン管理]
[対象Flow情報]
[バージョン一覧テーブル]
[操作ボタン: 詳細 / 複製 / 最新化 / 戻る]
```

## 6. 表示項目

| 項目 | 内容 |
| --- | --- |
| バージョン番号 | 1.0、1.1、2.0など |
| 最新フラグ | 最新版かどうか |
| 変更概要 | 変更内容の短い説明 |
| 作成者 | バージョン作成者 |
| 作成日時 | バージョン作成日時 |
| ノード数 | 対象バージョンのNode件数 |
| 接続数 | 対象バージョンのLink件数 |

## 7. 入力項目

新バージョン作成時に以下を入力する。

- バージョン番号
- 変更概要
- 変更理由

## 8. ボタン

- 詳細表示
- このバージョンを開く
- 最新版として複製
- 新バージョン作成
- 戻る

## 9. ダイアログ

新バージョン作成ダイアログを使用する。
削除や最新版変更など影響の大きい操作にはConfirmDialogを使用する。

## 10. バリデーション

| 項目 | 条件 |
| --- | --- |
| バージョン番号 | 必須、同一Flow内で重複不可 |
| 変更概要 | 最大200文字 |
| 変更理由 | 最大1000文字 |

## 11. API

- GET /api/flows/{flowId}/versions
- GET /api/flow-versions/{flowVersionId}
- POST /api/flows/{flowId}/versions
- POST /api/flow-versions/{flowVersionId}/duplicate
- PUT /api/flow-versions/{flowVersionId}/summary

## 12. DB

- FLOW
- FLOW_VERSION
- NODE
- LINK
- COMMENT
- USER

## 13. 状態遷移

```text
一覧表示
  ↓
バージョン選択
  ├ 詳細表示
  ├ エディタで開く
  └ 複製して新バージョン作成
```

## 14. エラー処理

- バージョン一覧の取得に失敗しました
- 対象バージョンが存在しません
- バージョン作成に失敗しました
- バージョン番号が重複しています

## 15. 未保存データ

フローエディタに未保存変更がある状態でバージョンを切り替える場合は、遷移前に確認する。

## 16. テスト観点

- バージョン一覧が表示されること
- 最新版が判別できること
- バージョンを選択して開けること
- 新バージョンを作成できること
- 重複バージョン番号でエラーになること

## 17. 完了条件

本画面は、FlowVersionを一覧・参照・作成でき、フローエディタと連携できる粒度で定義された場合に完了とする。
