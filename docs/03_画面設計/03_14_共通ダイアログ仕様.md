# 03_14_共通ダイアログ仕様

## 1. 本書の目的

本書は、AI Flow Designerで使用する共通ダイアログの仕様を定義する。
ダイアログは、作成、編集、削除確認、テンプレート選択、出力条件指定、レーン・工程削除時の移動選択などで使用する。

## 2. 画面概要

共通ダイアログはPrimeVue DialogおよびConfirmDialogを基本とする。
破壊的操作では必ず確認を行い、利用者が意図しない削除や変更を防止する。

## 3. 利用者

- システム設計者
- レビュー担当者
- 管理者

## 4. 権限

編集権限がない場合、作成・更新・削除系ダイアログは表示しない、または実行ボタンを非活性にする。

## 5. レイアウト

```text
[タイトル]
[説明文]
[入力項目]
[エラー表示]
[キャンセル] [実行]
```

## 6. 表示項目

| 項目 | 内容 |
| --- | --- |
| タイトル | 操作内容を明示する |
| 説明文 | 影響範囲や注意点を説明する |
| 入力領域 | 操作に必要な入力項目 |
| エラー表示 | 入力エラー、APIエラーを表示 |

## 7. 入力項目

ダイアログごとに異なるが、代表例は以下である。

- プロジェクト名
- フロー名
- バージョン番号
- 変更概要
- 移動先Lane
- 移動先Stage
- 出力形式

## 8. ボタン

- キャンセル
- 実行
- 削除
- 保存
- 作成

破壊的操作では実行ボタン名を「削除」など具体化する。

## 9. ダイアログ種類

| ダイアログ | 目的 |
| --- | --- |
| 作成ダイアログ | Project、Flow、Versionなどを作成する |
| 削除確認 | 削除前に確認する |
| Lane削除 | Nodeを削除するか移動するか選択する |
| Stage削除 | Nodeを削除するか移動するか選択する |
| テンプレート選択 | テンプレートからFlowを作成する |
| 出力条件 | 出力形式とオプションを選択する |
| 未保存確認 | 未保存変更がある状態で遷移する際に確認する |

## 10. バリデーション

- 必須項目未入力は実行不可
- 文字数超過はエラー表示
- 選択必須項目未選択は実行不可
- 削除確認では対象名を表示し誤認を防止する

## 11. API

ダイアログ自体はAPIを持たない。
各操作元画面のAPIを呼び出す。

## 12. DB

ダイアログの入力結果は、各操作対象DBへ反映される。

## 13. 状態遷移

```text
操作ボタン押下
  ↓
ダイアログ表示
  ↓
入力または確認
  ├ キャンセル
  └ 実行
       ↓
     成功/失敗
```

## 14. エラー処理

- 入力値が不正です
- 対象データが存在しません
- 操作に失敗しました
- 編集権限がありません

## 15. 未保存データ

未保存確認ダイアログでは、保存して遷移、破棄して遷移、キャンセルの選択肢を設ける。

## 16. テスト観点

- 必要な場面でダイアログが表示されること
- キャンセルで処理されないこと
- 必須未入力で実行できないこと
- 実行成功時に閉じること
- 実行失敗時にエラーが表示されること

## 17. 完了条件

本仕様は、共通ダイアログの構造、入力、ボタン、Validation、エラー処理が画面横断で統一された場合に完了とする。
