# 03_10_ユーザー設定画面

## 1. 本書の目的

本書は、ユーザー設定画面の詳細仕様を定義する。
ユーザー設定画面は、ログインユーザーごとの表示設定、操作設定、将来のAI利用設定を管理する画面である。

## 2. 画面概要

初期実装では、表示テーマ、キャンバス初期表示、エディタ操作設定を中心に扱う。
将来的には通知設定、AIレビュー設定、既定出力形式、ショートカット設定を追加できる構成とする。

## 3. 利用者

- システム設計者
- 実装者
- レビュー担当者
- 管理者

## 4. 権限

ログイン済みユーザーは自分の設定を参照・更新できる。
管理者が他ユーザー設定を変更する機能は将来対応とする。

## 5. レイアウト

```text
[ヘッダー]
[画面タイトル: ユーザー設定]
[プロフィール情報]
[表示設定]
[エディタ設定]
[出力設定]
[保存 / キャンセル]
```

## 6. 表示項目

| 項目 | 内容 |
| --- | --- |
| GitHubユーザー名 | OAuthから取得したユーザー名 |
| 表示名 | アプリ内表示名 |
| メール | 取得可能な場合のみ表示 |
| 最終ログイン日時 | セッション情報から表示 |

## 7. 入力項目

| 項目 | 内容 |
| --- | --- |
| 表示テーマ | light / dark / system |
| キャンバス初期ズーム | 初期表示倍率 |
| Grid表示 | Gridを表示するか |
| Snap有効 | ノード移動時にGrid吸着するか |
| 既定出力形式 | Mermaid / PDF / JSON |

## 8. ボタン

- 保存
- キャンセル
- 初期値に戻す

## 9. ダイアログ

初期値に戻す場合は確認ダイアログを表示する。
保存せずに画面遷移する場合は未保存確認を行う。

## 10. バリデーション

| 項目 | 条件 |
| --- | --- |
| 表示名 | 最大100文字 |
| キャンバス初期ズーム | 25〜200の範囲 |
| 既定出力形式 | 定義済み形式のみ |

## 11. API

- GET /api/users/me/settings
- PUT /api/users/me/settings
- POST /api/users/me/settings/reset

## 12. DB

- USER
- USER_SETTING

## 13. 状態遷移

```text
設定読込
  ↓
編集
  ├ 保存
  ├ キャンセル
  └ 初期値へ戻す
```

## 14. エラー処理

- ユーザー設定の取得に失敗しました
- ユーザー設定の保存に失敗しました
- 設定値が不正です
- ログイン状態が切れました

## 15. 未保存データ

入力値変更後に未保存状態とする。
未保存状態で画面遷移する場合はConfirmDialogを表示する。

## 16. テスト観点

- 設定値が表示されること
- 表示テーマを変更できること
- Grid/Snap設定を保存できること
- 不正なズーム値で保存できないこと
- 初期値へ戻せること

## 17. 完了条件

本画面は、ユーザー個別設定を表示・更新でき、フローエディタの表示・操作へ反映できる仕様になった場合に完了とする。
