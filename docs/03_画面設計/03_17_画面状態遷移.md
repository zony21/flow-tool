# 03_17_画面状態遷移

## 1. 本書の目的

本書は、AI Flow Designer の画面状態遷移を定義する。
各画面が読込中、表示中、編集中、未保存、保存中、エラー、セッション切れなどの状態を一貫して扱えるようにする。

## 2. 画面概要

画面状態はPiniaで管理する。
画面ごとに独自実装を行わず、共通状態モデルに従う。

## 3. 利用者

- 全利用者

## 4. 権限

権限により編集状態へ遷移できるかを制御する。
参照のみユーザーは編集中状態へ遷移できない。

## 5. レイアウト

状態に応じて以下を表示する。

- Loading表示
- Empty表示
- Error表示
- Editing表示
- Unsaved表示
- Saving表示

## 6. 表示項目

| 状態 | 表示内容 |
| --- | --- |
| loading | 読込中表示 |
| loaded | 通常表示 |
| editing | 編集中表示 |
| dirty | 未保存表示 |
| saving | 保存中表示 |
| error | エラー表示 |
| unauthorized | 権限なし表示 |
| sessionExpired | セッション切れ表示 |

## 7. 入力項目

状態遷移仕様自体に入力項目はない。

## 8. ボタン

状態に応じてボタン活性を制御する。

- loading中は主要操作非活性
- saving中は保存ボタン非活性
- dirty中は保存ボタン強調
- unauthorizedでは編集ボタン非表示

## 9. ダイアログ

dirty状態で画面遷移する場合、未保存確認ダイアログを表示する。

## 10. バリデーション

保存前に画面単位のValidationを実行する。
Validation失敗時はdirty状態を維持する。

## 11. API

画面状態はAPIレスポンスにより遷移する。

- 成功: loadedまたはsaved
- 400: validationError
- 401: sessionExpired
- 403: unauthorized
- 500: error

## 12. DB

DB更新成功後に保存済み状態へ遷移する。
DB更新失敗時は未保存状態を維持する。

## 13. 状態遷移

```text
initial
  ↓
loading
  ├ loaded
  │   ├ editing
  │   │   ├ dirty
  │   │   │   ├ saving
  │   │   │   │   ├ saved
  │   │   │   │   └ error
  │   │   │   └ discard
  │   │   └ loaded
  │   └ error
  ├ unauthorized
  └ sessionExpired
```

## 14. エラー処理

エラー時は状態をerrorにし、操作可能な復旧手段を表示する。
保存失敗時はdirtyを維持する。

## 15. 未保存データ

dirty状態では以下の操作前に確認する。

- 画面遷移
- バージョン切替
- ログアウト
- ブラウザ更新
- Flow切替

## 16. テスト観点

- 読込中にLoadingが表示されること
- 編集後にdirtyになること
- 保存成功でdirtyが解除されること
- 保存失敗でdirtyが維持されること
- 401でsessionExpiredになること
- 403でunauthorizedになること

## 17. 完了条件

本仕様は、全画面が共通状態遷移に従い、未保存データとエラーを一貫して扱える場合に完了とする。
