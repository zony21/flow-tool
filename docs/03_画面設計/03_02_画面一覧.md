# 03_02_画面一覧

## 1. 本書の目的

本書は、AI Flow Designer で提供する画面一覧を定義する。
画面一覧は、画面設計、API設計、DB設計、権限設計、テスト仕様を対応付けるための基準資料とする。

## 2. 画面分類

画面は以下に分類する。

- 認証画面
- プロジェクト管理画面
- フロー管理画面
- フロー編集画面
- 出力画面
- テンプレート画面
- 設定画面
- 管理画面
- Transport管理画面

## 4. 主要画面の位置付け

### 4.1 ログイン画面

GitHub OAuth認証を開始する画面である。
未ログイン時は認証が必要な画面へ遷移させず、本画面へ誘導する。

### 4.2 プロジェクト一覧画面

ユーザーが最初に操作する主要画面である。
作成済みプロジェクトを一覧表示し、新規作成、編集、削除、フロー一覧遷移を提供する。

### 4.3 フロー一覧画面

プロジェクト配下の作業単位フローを管理する。
RFID読取り、搬送要求、ラベル印刷、AGF搬送、異常停止などの単位でフローを作成する。

### 4.4 フローエディタ画面

本システムの中核画面である。
担当レーン、工程レーン、ノード、接続、コメントを編集し、AIが理解可能な構造化データを作成する。

AGF/AGV搬送フローの場合は、通常のFlow編集に加えて搬送属性を扱う。
搬送属性はメーカー、コマンド、ロケーション、設備、R/W、処理区分で構成する。

### 4.5 出力画面

構造化データから Mermaid、PDF、JSONを生成する。
将来的にAI DSL、API仕様、DB更新一覧、通信一覧、PLC一覧、設計書ドラフト、コード生成へ拡張する。

Transport Flowでは搬送表出力を追加する。

### 4.6 Transport管理画面

AGF/AGV搬送フローで利用するマスタを管理する。
メーカー管理、コマンド管理、設備管理は独立画面とし、ロケーション管理はフロー編集時の利用頻度が高いためフローエディタ内ダイアログとして扱う。

## 5. 画面遷移概要

```text
ログイン
  ↓
プロジェクト一覧
  ↓
プロジェクト詳細
  ↓
フロー一覧
  ↓
フローエディタ
  ├ 出力画面
  ├ バージョン一覧
  ├ テンプレート選択
  ├ ロケーション管理ダイアログ
  └ Transport管理
       ├ AGF/AGVメーカー管理
       ├ AGF/AGVコマンド管理
       └ 設備管理
```

## 6. 権限との対応

初期実装ではGitHubログイン済みユーザーを編集可能ユーザーとして扱う。
将来的には画面単位で以下を制御する。

- 参照可否
- 作成可否
- 更新可否
- 削除可否
- 出力可否
- 管理可否

Transport管理画面はProject設定権限または管理権限で制御する。

## 9. テスト観点

- 画面一覧に定義された画面へ遷移できること
- 初期対応画面と将来対応画面が区別されていること
- 各画面が関連APIと対応していること
- 各画面が関連DBと対応していること
- 権限追加時に画面制御へ展開できること
- Transport管理画面が通常Flowへ不要に露出しないこと

## 10. 完了条件

本書は、初期実装対象画面が明確であり、後続の個別画面設計書へ展開できる状態になった場合に完了とする。
