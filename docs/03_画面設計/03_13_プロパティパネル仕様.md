# 03_13_プロパティパネル仕様

## 1. 本書の目的

本書は、フローエディタ画面右側に配置するプロパティパネルの詳細仕様を定義する。
プロパティパネルは、選択中のLane、Stage、Node、Link、Commentの意味情報を編集するための重要UIである。

## 2. 画面概要

プロパティパネルは選択対象に応じて表示内容を切り替える。
未選択時はFlow概要または操作ガイドを表示する。

## 3. 利用者

- システム設計者
- 業務設計者
- レビュー担当者

## 4. 権限

参照のみ権限では入力項目を読み取り専用とする。
編集権限がある場合のみ更新できる。

## 5. レイアウト

```text
[選択対象タイトル]
[基本情報]
[構造情報]
[AI向け情報]
[表示情報]
[操作ボタン]
```

## 6. 表示項目

選択対象別に以下を表示する。

| 対象 | 主な項目 |
| --- | --- |
| Node | 表示名、種別、Lane、Stage、処理種別、説明、AIメモ、座標、サイズ |
| Link | FROM、TO、データ名、通信種別、条件、説明 |
| Lane | 担当名、カテゴリ、色、アイコン、説明、表示順 |
| Stage | 工程名、説明、色、表示順 |
| Comment | コメント本文、紐付けNode、AI向けフラグ、表示位置 |

## 7. 入力項目

各対象の編集可能項目を入力項目とする。
内部IDは表示のみ、または開発者向け表示に限定する。

## 8. ボタン

- 適用
- 削除
- 複製
- キャンセル

編集内容はリアルタイム反映または適用ボタン反映のどちらかを画面設計方針で統一する。
初期実装では入力変更時にStoreへ反映し、保存ボタンでDBへ保存する。

## 9. ダイアログ

削除操作ではConfirmDialogを表示する。
Lane/Stage削除時にNodeが存在する場合は移動・削除・キャンセルの選択ダイアログを表示する。

## 10. バリデーション

| 対象 | 条件 |
| --- | --- |
| Node表示名 | 必須、最大100文字 |
| Link FROM/TO | 必須、存在するNode ID |
| Lane担当名 | 必須、最大100文字 |
| Stage工程名 | 必須、最大100文字 |
| Comment本文 | 最大2000文字 |

## 11. API

プロパティパネル単体で即時API保存しない。
フローエディタ保存時に以下へ反映する。

- PUT /api/flow-versions/{flowVersionId}

将来的に個別更新APIを設けてもよい。

## 12. DB

- LANE
- STAGE
- NODE
- LINK
- COMMENT
- FLOW_VERSION

## 13. 状態遷移

```text
対象未選択
  ↓
対象選択
  ↓
プロパティ表示
  ↓
編集
  ↓
未保存状態
  ↓
保存
```

## 14. エラー処理

- 入力値が不正です
- 対象データが存在しません
- 編集権限がありません
- 保存に失敗しました

## 15. 未保存データ

プロパティ変更後はフローエディタ全体を未保存状態にする。
画面遷移時には未保存確認を行う。

## 16. テスト観点

- Node選択時にNode項目が表示されること
- Link選択時にLink項目が表示されること
- 編集後に未保存状態になること
- 必須未入力でエラーになること
- 参照権限では読み取り専用になること

## 17. 完了条件

本仕様は、選択対象ごとに意味情報を編集でき、保存時に構造化データへ正しく反映できる仕様となった場合に完了とする。
