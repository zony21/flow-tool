# 03_19_共通UI仕様

## 1. 本書の目的

本書は、AI Flow Designer の全画面で共通利用するUI仕様を定義する。
Header、Sidebar、Footer、Toast、ConfirmDialog、ErrorDialog、Loading、Session切れ表示などを統一し、画面ごとの実装ばらつきを防止する。

## 2. 画面概要

共通UIは各画面の土台として利用する。
画面個別仕様は本書の共通UI仕様を前提に設計する。

## 3. 利用者

- 全利用者

## 4. 権限

共通UIはログイン状態と権限に応じて表示を切り替える。
未ログイン時はログイン画面へ誘導する。

## 5. レイアウト

```text
[Header]
[Sidebar] [Main Content]
[Footer]
[Toast]
[Dialog Layer]
[Loading Overlay]
```

## 6. 表示項目

### 6.1 Header

- アプリ名
- 現在プロジェクト名
- 現在フロー名
- ログインユーザー
- ユーザーメニュー

### 6.2 Sidebar

- プロジェクト一覧導線
- フロー一覧導線
- テンプレート導線
- 設定導線

### 6.3 Footer

- バージョン情報
- 保存状態
- 最終更新日時

## 7. 入力項目

共通UI自体に原則入力項目はない。
検索や設定入力は個別画面で扱う。

## 8. ボタン

- ユーザーメニュー
- ログアウト
- Sidebar開閉
- 通知閉じる

## 9. ダイアログ

全画面で以下の共通ダイアログを使用する。

- ConfirmDialog
- ErrorDialog
- UnsavedConfirmDialog
- SessionExpiredDialog

## 10. バリデーション

共通UIでは入力バリデーションは行わない。
ただし、セッション状態と権限状態は常に確認する。

## 11. API

- GET /api/auth/me
- POST /api/auth/logout
- GET /api/system/version

## 12. DB

- USER
- USER_SETTING

## 13. 状態遷移

```text
アプリ起動
  ↓
認証状態確認
  ├ 未ログイン -> ログイン画面
  └ ログイン済 -> メイン画面
       ├ 通常
       ├ Loading
       ├ Error
       └ SessionExpired
```

## 14. エラー処理

- 認証状態を確認できませんでした
- ログイン状態が切れました
- 通信に失敗しました

## 15. 未保存データ

共通UIは画面の未保存状態を受け取り、FooterやHeaderで保存状態を表示する。
ログアウト時に未保存変更がある場合は確認する。

## 16. テスト観点

- Headerが全画面で表示されること
- Sidebarを開閉できること
- Toastが表示されること
- Loadingが表示されること
- Session切れでログイン誘導されること
- 未保存状態が表示されること

## 17. 完了条件

本仕様は、全画面で共通UIを一貫して利用でき、認証・権限・未保存・エラー表示を統一できる状態になった場合に完了とする。
