# 03_06_プロジェクト詳細画面

## 1. 本書の目的

本書は、プロジェクト詳細画面の詳細仕様を定義する。
プロジェクト詳細画面は、プロジェクトの基本情報を確認・編集し、配下のFlow管理へ遷移するための画面である。

## 2. 画面の役割

本画面では以下を行う。

- プロジェクト基本情報の表示
- プロジェクト基本情報の編集
- プロジェクト配下Flowの概要表示
- フロー一覧画面への遷移
- プロジェクト削除操作

## 3. レイアウト

```text
[ヘッダー]
[パンくず: プロジェクト一覧 > プロジェクト詳細]
[画面タイトル: プロジェクト詳細]
[基本情報カード]
[配下Flow概要]
[操作ボタン: 保存 / フロー一覧へ / 削除]
```

## 4. 表示項目

| 項目 | 内容 | 編集可否 |
| --- | --- | --- |
| プロジェクト名 | プロジェクト表示名 | 可 |
| 説明 | プロジェクト説明 | 可 |
| 表示順 | 一覧表示順 | 可 |
| 作成者 | 作成ユーザー | 不可 |
| 作成日時 | 作成日時 | 不可 |
| 更新者 | 最終更新ユーザー | 不可 |
| 更新日時 | 最終更新日時 | 不可 |
| Flow件数 | 配下Flow件数 | 不可 |

## 5. 操作仕様

### 5.1 保存

保存ボタン押下で入力内容をProject APIへ送信する。
保存成功後、画面上の未保存状態を解除する。

### 5.2 フロー一覧へ遷移

フロー一覧へボタン押下で対象プロジェクトのフロー一覧画面へ遷移する。

### 5.3 削除

削除ボタン押下で確認ダイアログを表示する。
配下Flowが存在する場合の制御はAPI側の削除可否判定に従う。

## 6. バリデーション

| 項目 | 条件 |
| --- | --- |
| プロジェクト名 | 必須、最大100文字 |
| 説明 | 最大1000文字 |
| 表示順 | 0以上の整数 |

## 7. API連携

- GET /api/projects/{projectId}
- PUT /api/projects/{projectId}
- DELETE /api/projects/{projectId}
- GET /api/projects/{projectId}/flows/summary

## 8. DB連携

- PROJECT
- FLOW
- USER

## 9. エラー表示

- プロジェクト情報の取得に失敗しました
- プロジェクト情報の保存に失敗しました
- プロジェクトの削除に失敗しました
- 対象プロジェクトが存在しません

## 10. テスト観点

- プロジェクト情報が表示されること
- 編集内容を保存できること
- 必須未入力で保存できないこと
- フロー一覧画面へ遷移できること
- 削除確認が表示されること
- APIエラー時にメッセージが表示されること

## 11. 完了条件

本画面は、Projectの基本情報を表示・編集・削除でき、Flow管理画面へ接続できる粒度で定義された場合に完了とする。
