# 01_02_システム構成

## 1. 本書の目的

本書は、AI Flow Designer のシステム構成を定義する。

Frontend、Backend、DB、認証、Export、外部サービスの関係を整理し、各レイヤの責務が混在しないようにする。

## 2. 全体構成

```text
Browser
  ↓ HTTPS
Vue 3 Frontend
  ↓ REST API
ASP.NET Core Backend
  ↓ EF Core
SQLite / SQL Server

GitHub OAuth
  ↑↓
Backend Auth
```

Frontendはユーザー操作、画面表示、Flow描画を担当する。Backendは認証、認可、業務処理、DB保存、Export生成を担当する。DBは構造化Flowデータを保持する。

## 3. Frontend構成

FrontendはVue 3、TypeScript、PrimeVue、Vue Flow、Piniaで構成する。

主な責務:

- 画面表示
- ユーザー操作受付
- Flowエディタ描画
- 入力Validationの補助
- API呼び出し
- Toast / Dialog表示
- 未保存変更の保護

FrontendはDBへ直接接続しない。

## 4. Backend構成

BackendはASP.NET Core .NET 8で構成する。

主な責務:

- API提供
- GitHub OAuth認証
- Project権限制御
- Flow保存
- AutoSave
- Version作成・復元
- Template管理
- Export生成
- Validation

BackendはEntityをAPIへ直接返却せず、DTOへ変換して返却する。

## 5. DB構成

初期DBはSQLiteとする。

将来、運用規模が大きくなった場合はSQL Serverへ移行する。

DBは構造化Flowデータの正であり、Project、Flow、Lane、Stage、Node、Link、Comment、Version、Template、Setting、Lock、ExportHistoryを保持する。

## 6. 認証構成

認証はGitHub OAuthを利用する。

GitHubから取得した外部ユーザー情報は、内部USER_ACCOUNTと紐付ける。

以後のAPI認証はBackendが発行するTokenで行う。

## 7. Export構成

ExportはBackendで生成する。

対象:

- Mermaid Flowchart
- Mermaid Sequence
- JSON
- PDF

将来:

- AI DSL
- API仕様
- DB更新一覧
- 通信一覧

## 8. レイヤ責務

|レイヤ|責務|
|---|---|
|View|画面表示|
|Store|Frontend状態管理|
|API Client|Backend API呼び出し|
|Controller|HTTP入口|
|Service|業務処理|
|Repository|DB操作|
|Entity|DB構造|
|DTO|API入出力|

## 9. 重要な制約

- FrontendはDBへ直接接続しない
- Frontendの権限制御は補助であり、Backend認可を必須とする
- Vue Flow内部状態を保存の正にしない
- ExportはDB構造から生成する
- Flow保存はTransactionで行う

## 10. テスト観点

- FrontendからAPIを呼び出せる
- 未認証時に保護APIへアクセスできない
- 権限なしProjectを参照できない
- Flow保存でDBへ正しく反映される
- ExportがDB構造から生成される

## 11. 完了条件

- システムのレイヤ構成が明確である
- Frontend、Backend、DB、認証、Exportの責務が分離されている
- 03画面設計、04DB設計、05API設計へ展開できる
