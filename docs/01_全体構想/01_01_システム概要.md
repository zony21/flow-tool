# 01_01_システム概要

## 1. 本書の目的

本書は、AI Flow Designer のシステム概要を定義する。

本書では、システムが何を実現するか、どのような構造で設計されるか、主要な管理単位がどのように関係するかを整理する。

## 2. システムの目的

AI Flow Designer は、業務フロー、システムフロー、設備連携フローを構造化データとして設計するWebアプリケーションである。

ユーザーはブラウザ上でFlowを視覚的に編集し、その内容をMermaid、PDF、JSON、将来のAI向け出力へ展開できる。

## 3. 中核となる考え方

本システムでは、画面上の図ではなく、DBに保存される構造化Flowデータを正とする。

Flowは以下の構造で管理する。

```text
Project
  └─ Flow
      ├─ Lane
      ├─ Stage
      ├─ Node
      ├─ Link
      └─ Comment
```

この構造を正とすることで、画面表示、Export、Version、Template、AI向け出力を同じ情報源から生成できる。

## 4. 主要概念

|概念|説明|
|---|---|
|Project|Flowを束ねる最上位単位|
|Flow|設計対象のフロー|
|Lane|処理担当主体|
|Stage|処理工程|
|Node|処理・判断・開始・終了などの単位|
|Link|Node間の意味的接続|
|Comment|補足説明|
|Template|再利用可能な構造|
|Version|復元可能なSnapshot|
|Export|成果物生成|

## 5. 利用の基本流れ

基本的な利用流れは以下である。

```text
GitHubログイン
  ↓
Project作成または選択
  ↓
Flow作成または選択
  ↓
Flowエディタで編集
  ↓
保存 / AutoSave
  ↓
Version作成 / Template作成 / Export
```

## 6. 技術構成概要

FrontendはVue 3、TypeScript、PrimeVue、Vue Flow、Piniaで構成する。

BackendはASP.NET Core .NET 8で構成する。

DBは初期段階ではSQLiteを利用し、将来SQL Serverへ移行できる設計とする。

認証はGitHub OAuthを利用する。

## 7. 初期開発の対象

初期開発では以下を対象とする。

- GitHub OAuth認証
- Project管理
- Flow管理
- Flowエディタ
- Lane / Stage / Node / Link / Comment編集
- Undo / Redo
- AutoSave
- Version管理
- Template管理
- Export
- User設定

## 8. 初期対象外

以下は将来拡張とする。

- リアルタイム共同編集
- AIチャット
- 本格的なコード生成
- 外部レビュー機能
- 複数DB同時運用

ただし、将来拡張できるように、構造化データとAI向け出力の設計は初期段階から考慮する。
