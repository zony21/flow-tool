# 01_05_画面遷移

## 1. 本書の目的

本書は、AI Flow Designer の全体画面遷移を定義する。

詳細な画面仕様は03_画面設計で扱う。本書では、ログインからProject、Flow、Flowエディタ、Template、Version、Export、設定までの主要導線を整理する。

## 2. 基本方針

画面遷移はVue Routerで制御する。

認証が必要な画面ではRouter Guardを利用する。Project権限が必要な画面では、画面表示前またはAPI取得時にBackendで権限を確認する。

## 3. 全体画面遷移

```text
/login
  ↓
/home
  ↓
/projects
  ↓
/projects/:projectId
  ↓
/projects/:projectId/flows
  ↓
/projects/:projectId/flows/:flowId/editor
      ├─ version
      ├─ template
      ├─ export
      └─ settings
```

## 4. ログイン導線

未認証ユーザーが保護画面へアクセスした場合、`/login` へ遷移する。

GitHub OAuth成功後、元の遷移先が安全な内部URLであればそこへ戻す。不正なredirect先の場合はProject一覧へ遷移する。

## 5. Project導線

Project一覧では、ユーザーが参照権限を持つProjectのみ表示する。

主な遷移:

```text
/projects
  ├─ Project作成
  ├─ Project詳細
  └─ Project編集
```

ProjectAdmin以上はProject設定を編集できる。

## 6. Flow導線

Project詳細またはFlow一覧からFlowエディタへ遷移する。

```text
/projects/:projectId/flows
  ├─ Flow作成
  ├─ Flow編集
  ├─ Flow複写
  └─ Flow無効化
```

ViewerはFlowを閲覧できるが、編集はできない。

## 7. Flowエディタ導線

Flowエディタは本システムの中心画面である。

```text
Flowエディタ
  ├─ Node / Link編集
  ├─ PropertyPanel編集
  ├─ Template適用
  ├─ Version作成 / 復元
  ├─ Export
  └─ Flow一覧へ戻る
```

Editor以上で編集ロックを取得できた場合のみ編集可能とする。

## 8. Template導線

Template管理画面では、Templateの一覧、詳細、作成、適用を扱う。

FlowエディタからTemplateを適用する場合、適用後の変更はUndo対象とする。

Template適用時はID再採番を行う。

## 9. Version導線

Version管理画面では、Version一覧、詳細、比較、復元を扱う。

Version復元は編集操作であるため、編集権限と編集ロックを必須とする。

復元前には現在状態を復元前Versionとして保存する。

## 10. Export導線

Export画面では、形式選択、Validation、Preview、Downloadを行う。

Viewer以上でExport可能とする。

Export前にValidationを実行し、エラーがある場合はExport不可とする。

## 11. 設定導線

設定画面では、ユーザー単位の表示設定、Grid / Snap、AutoSave、Shortcut、Export既定値を設定する。

設定変更はFlowエディタへ反映される。

## 12. 未保存変更の扱い

Dirty状態のまま画面遷移しようとした場合、UnsavedChangesDialogを表示する。

選択肢:

- 保存して移動
- 保存せず移動
- キャンセル

FlowエディタではAutoSave状態も考慮する。

## 13. エラー遷移

|状況|遷移または表示|
|---|---|
|401|/login|
|403|Forbidden表示|
|404|NotFound表示|
|500|ErrorDialogまたはErrorPanel|
|Session切れ|/login|
|編集ロック競合|読み取り専用で開く|

## 14. 権限別導線

|Role|主な導線|
|---|---|
|Viewer|閲覧、Export、Version比較|
|Editor|Flow編集、Template適用、Version作成|
|ProjectAdmin|Project設定、メンバー管理、Version復元|
|Admin|標準Template管理、システム管理|

## 15. APIとの関係

画面遷移時には、必要に応じて以下を取得する。

- currentUser
- Project権限
- Project詳細
- Flow一覧
- Flow詳細
- EditorLock状態
- UserSetting

## 16. テスト観点

- 未認証で保護画面へアクセスするとLoginへ遷移する
- 認証後に元画面へ戻れる
- 権限なしProjectを開けない
- Dirty状態で遷移確認が出る
- ViewerはFlow編集できない
- Session切れ時にLoginへ戻る

## 17. 完了条件

- ログインからFlow編集までの導線が定義されている
- Template、Version、Export、設定への導線が定義されている
- エラー時、未保存時、権限別の挙動が明確である
