# 01_03_データ構造

## 1. 本書の目的

本書は、AI Flow Designer の全体データ構造を定義する。

本書では、Project、Flow、Lane、Stage、Node、Link、Commentを中心とした構造化データの関係を整理し、DB設計・API設計・画面設計へつなげる。

## 2. 基本方針

本システムでは、図面ではなく構造化データを正とする。

Flowエディタ上の図形や線は表示結果であり、保存、Export、Version、Template、AI向け出力ではDBに保存された構造化データを利用する。

## 3. データ階層

基本階層は以下である。

```text
Project
  └─ Flow
      ├─ Lane
      ├─ Stage
      ├─ Node
      ├─ Link
      └─ Comment
```

Projectは最上位管理単位である。FlowはProject配下に存在する。Flow内には担当主体であるLane、工程であるStage、処理単位であるNode、接続であるLink、補足であるCommentが存在する。

## 4. 各データの役割

|データ|役割|
|---|---|
|Project|複数Flowを束ねる|
|Flow|フロー設計の単位|
|Lane|誰が担当するかを表す|
|Stage|どの工程かを表す|
|Node|処理や判断を表す|
|Link|Node間の意味的な接続を表す|
|Comment|補足説明を表す|

## 5. NodeとLane / Stageの関係

NodeはLaneとStageに所属する。

これにより、AIやExport処理は「どの担当が、どの工程で、どの処理を行うか」を判断できる。

画面上の位置だけでLaneやStage所属を判定しない。

## 6. Linkの意味

Linkは単なる線ではない。

Linkは以下の情報を持つ。

- 接続元Node
- 接続先Node
- ラベル
- データ名
- 通信種別
- 条件
- 表示上の線種

ループ接続は許可する。

## 7. Commentの種類

Commentには以下がある。

- 独立Comment
- Node紐付けComment

Node紐付けCommentは、対象Nodeの補足情報として扱う。

## 8. VersionとTemplate

VersionはFlow構造のSnapshotである。

Templateは再利用可能なFlow構造である。

どちらもLane、Stage、Node、Link、Commentの関係を保持する。

## 9. Exportとの関係

Exportは構造化データから生成する。

対象:

- Mermaid Flowchart
- Mermaid Sequence
- JSON
- PDF
- 将来AI DSL
