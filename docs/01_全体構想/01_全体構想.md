# 01_全体構想

## 1. 本書の目的

本書は、AI Flow Designer の全体構想を定義する。

00_全体方針が設計思想と判断基準を示す文書であるのに対し、本書はアプリケーション全体の構成、主要機能、利用導線、システム構成、開発範囲を整理する。

## 2. システム概要

AI Flow Designer は、構造化Flowデータを中心にしたWebアプリケーションである。

ユーザーはブラウザ上でProjectを作成し、Project配下にFlowを作成する。Flowエディタでは、Lane、Stage、Node、Link、Commentを編集し、その内容をDBへ保存する。

保存されたFlow構造から、Mermaid、PDF、JSON、将来のAI向け出力を生成する。

## 3. 全体アーキテクチャ

本システムは以下の構成とする。

```text
Browser
  ↓
Vue 3 Frontend
  ↓ REST API
ASP.NET Core Backend
  ↓ EF Core
SQLite / SQL Server
```

Frontendは画面表示と操作を担当する。Backendは認証、認可、保存、Export、Version管理を担当する。DBは構造化Flowデータを保持する。

## 4. 主要構成要素

|構成要素|役割|
|---|---|
|Project|Flowを束ねる最上位単位|
|Flow|設計対象のフロー|
|Lane|担当主体|
|Stage|工程|
|Node|処理、判断、開始、終了など|
|Link|Node間の意味的接続|
|Comment|補足説明|
|Template|再利用可能なFlow構造|
|Version|復元可能なSnapshot|
|Export|成果物生成|

## 5. 利用者導線

基本導線は以下とする。

```text
GitHub Login
  ↓
Home
  ↓
Project一覧
  ↓
Project詳細
  ↓
Flow一覧
  ↓
Flowエディタ
  ↓
Export / Version / Template
```

未認証ユーザーはログイン画面へ誘導する。Project権限のないユーザーは該当Projectを参照できない。

## 6. 機能構成

初期開発で扱う機能は以下とする。

- GitHub OAuth認証
- Project管理
- Flow管理
- Flowエディタ
- Lane / Stage編集
- Node / Link / Comment編集
- Undo / Redo
- AutoSave
- Template管理
- Version管理
- Export
- User設定

## 7. Flowエディタ構想

Flowエディタは本システムの中核である。

画面上ではVue Flowを利用してNodeとLinkを描画する。ただし、保存の正はVue Flowの内部状態ではなく、DBに保存される構造化Flowデータとする。

Flowエディタでは以下を扱う。

- Node追加、移動、削除
- Link作成、編集、削除
- Lane / Stageの所属管理
- PropertyPanelでの詳細編集
- Undo / Redo
- AutoSave
- 編集ロック

## 8. Export構想

Exportは、構造化Flowデータから成果物を生成する機能である。

初期対象:

- Mermaid Flowchart
- Mermaid Sequence
- PDF
- JSON

将来対象:

- AI DSL
- API仕様
- DB更新一覧
- 通信一覧
- PLC一覧
- 設計書ドラフト

## 9. Version構想

Versionは、Flowの復元可能な履歴である。

Version作成時は、Flow構造をSnapshot JSONとして保存する。復元時は現在状態を復元前Versionとして保存したうえで、選択Versionの構造へ戻す。

Undo / Redoは一時操作履歴であり、Versionとは責務を分ける。

## 10. Template構想

Templateは、再利用可能なFlow構造である。

TemplateにはLane、Stage、Node、Link、Commentを含めることができる。適用時は既存IDをそのまま使わず、新しいIDへ再採番する。

## 11. 認証・認可構想

認証はGitHub OAuthを利用する。

認可はProject単位のRoleで制御する。

基本Role:

- Viewer
- Editor
- ProjectAdmin
- Admin

Frontendの表示制御は補助であり、Backend APIで必ず権限を確認する。

## 12. DB構想

DBは構造化Flowデータの正となる。

初期DBはSQLiteとし、将来的にSQL Serverへ移行可能な設計とする。

主要テーブルはProject、Flow、Lane、Stage、FlowNode、FlowLink、FlowComment、Template、FlowVersion、UserSetting、EditorLock、ExportHistoryである。

## 13. 初期開発範囲

初期開発では、単独編集を前提とする。

リアルタイム共同編集、AIチャット、本格的なコード生成、外部レビュー機能は将来拡張とする。

ただし、将来拡張できるように、構造化データとAI向け出力の設計は初期段階から行う。
