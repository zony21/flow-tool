# 02_16_バージョン管理要件

## 1. 本書の目的

本書は、AI Flow Designer のFlowバージョン管理要件を定義する。

Flowバージョン管理は、設計変更履歴を安全に保持し、比較・復元・レビューを可能にするための機能である。

## 2. 基本方針

- Flow単位でVersionを保持する
- VersionはFlow構造のSnapshotとして保存する
- Undo / Redo履歴はVersionに含めない
- AutoSave一時状態はVersionに含めない
- Version復元前に現在状態を復元前Versionとして保存する
- Version復元後はUndo / Redo履歴をクリアする

## 3. Version対象データ

Version Snapshotに含める対象:

- Flow基本情報
- Lane
- Stage
- Node
- Link
- Comment
- Layout
- Export設定
- 拡張JSON

Version Snapshotに含めない対象:

- 編集ロック
- Undo / Redo履歴
- 選択状態
- Hover状態
- Viewport一時状態
- AutoSave一時データ

## 4. Version作成タイミング

Versionは以下のタイミングで作成できる。

- ユーザーが明示的にVersion作成した時
- Version復元前
- 将来的にリリース前保存時
- 将来的に大規模変更前

初期実装では明示作成と復元前作成を必須とする。

## 5. Version番号

Version番号はFlow単位で採番する。

例:

```text
v1
v2
v3
```

DB上では連番、表示上では `v{number}` として扱う。

## 6. Versionコメント

Version作成時にコメントを入力できる。

用途:

- 変更理由
- レビュー観点
- 復元時の判断材料

コメント未入力でも作成可能とする。

## 7. Snapshot形式

SnapshotはJSON形式で保持する。

要件:

- schemaVersionを持つ
- Flow全体構造を再現できる
- ID参照関係を保持する
- 将来Migration可能な形式とする

## 8. Version一覧要件

表示項目:

- Version番号
- 作成日時
- 作成者
- コメント
- Node数
- Link数
- Comment数

## 9. Version比較要件

比較対象:

- Version A
- Version B

比較対象要素:

- Lane追加・更新・削除
- Stage追加・更新・削除
- Node追加・更新・削除・移動・リサイズ
- Link追加・更新・削除
- Comment追加・更新・削除

## 10. 差分分類

差分分類:

- Added
- Updated
- Deleted
- Moved
- Resized

差分表示はVersion管理画面で確認できること。

## 11. Version復元要件

復元手順:

1. 復元対象Versionを選択する
2. 権限を確認する
3. 編集ロックを確認する
4. 現在状態を復元前Versionとして保存する
5. SnapshotをFlowへ反映する
6. Revisionを更新する
7. Undo / Redo履歴をクリアする
8. Flowエディタへ戻る

## 12. 復元権限

Version復元はEditor以上が可能。

ただし、Flow編集ロック取得済みであることを必須とする。

ViewerはVersion一覧と比較のみ可能。

## 13. 保持数

初期方針:

- Version数に明確な上限は設けない
- ただし性能要件ではFlowあたり100 Versionを試験対象とする

将来的にProject設定で保持数上限を設定可能にする。

## 14. 削除ルール

初期実装ではVersion削除は必須機能としない。

将来対応する場合:

- ProjectAdmin以上のみ削除可
- 削除は論理削除
- 復元済み履歴との関連に注意する

## 15. DB要件

主テーブル:

- FLOW_VERSION

保持項目候補:

- FLOW_VERSION_ID
- FLOW_ID
- VERSION_NO
- COMMENT
- SNAPSHOT_JSON
- CREATED_BY
- CREATED_AT
- IS_DELETED

## 16. API要件

対象API:

- Version一覧取得
- Version詳細取得
- Version作成
- Version比較
- Version復元

API詳細は05_API設計に従う。

## 17. 異常系

- Versionが存在しない
- Snapshot JSONが不正
- 権限不足
- 編集ロックなし
- Revision競合
- 復元中DB更新失敗

## 18. テスト観点

- Versionを作成できる
- Version一覧を表示できる
- Version比較できる
- Version復元できる
- 復元前Versionが作成される
- 復元後Undo / Redo履歴が消える
- Viewerは復元できない

## 19. 完了条件

- Flow単位でVersionを保持できる
- Snapshotから復元できる
- 比較要件が定義されている
- 権限と編集ロック要件が明確である
