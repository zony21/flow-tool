# 02_17_AI出力要件

## 1. 本書の目的

本書は、AI Flow Designer のAI出力要件を定義する。

AI出力は、構造化FlowをAIが高精度で理解できる形式へ変換するための機能であり、本システムの中核価値である。

## 2. 基本方針

- AI出力の正は構造化Flowデータである
- 図面画像をAI理解の正としない
- Node / Link / Lane / Stage / Comment の意味情報を欠落させない
- AI DSLは独自仕様として固定する
- 将来的なLLM変更に依存しすぎない形式にする

## 3. AI出力対象

対象:

- JSON
- AI DSL
- Mermaid
- AI用テキスト
- 設計書ドラフト

将来対象:

- API仕様
- DB更新一覧
- PLC一覧
- 通信一覧
- テスト観点一覧

## 4. AI理解性要件

AIが理解するために必須の情報:

- Flow名
- Flow説明
- Lane担当
- Stage工程
- Node表示名
- Node種別
- Node説明
- Link接続元
- Link接続先
- Linkデータ名
- Link通信種別
- Link条件
- Comment本文

## 5. Node出力要件

Nodeは以下を出力する。

- nodeId
- displayName
- nodeType
- laneName
- stageName
- description
- input / output相当情報
- extensionJson

座標だけでは意味を判断しない。

## 6. Link出力要件

Linkは以下を出力する。

- linkId
- fromNode
- toNode
- dataName
- communicationType
- condition
- description
- loop有無

Linkは単なる矢印ではなく、処理間の意味的関係として扱う。

## 7. Lane出力要件

Laneは担当主体として出力する。

例:

- WCS
- RCS
- AGF
- PLC
- MES
- ERP
- 作業者

LaneはAIが「誰が処理を担うか」を判断する基準となる。

## 8. Stage出力要件

Stageは工程として出力する。

StageはAIが「いつ・どの工程で処理が行われるか」を判断する基準となる。

## 9. Comment出力要件

Commentは以下に分類して出力する。

- Node紐付けComment
- 独立Comment

Node紐付けCommentは対象Nodeの補足説明として扱う。

独立CommentはFlow全体または近傍要素の補足として扱う。

## 10. AI DSL要件

AI DSLは独自仕様として固定する。

要件:

- Flow構造を欠落なく表現できる
- 人間にも読める
- AIにも解析しやすい
- schemaVersionを持つ
- 将来変更しない前提で設計する

## 11. JSON出力要件

JSON出力では内部構造を機械可読形式で出力する。

要件:

- schemaVersionを含む
- Project / Flow / Lane / Stage / Node / Link / Commentを含む
- ID参照を保持する
- 必要に応じて監査項目を除外できる

## 12. AI用テキスト要件

AI用テキストは、JSONやDSLより自然言語寄りの説明を生成する。

用途:

- ChatGPTへ貼り付ける
- 設計レビューへ使う
- 開発者向け説明に使う

## 13. 出力前Validation

AI出力前に以下を検証する。

- Node名不足
- Link接続先不足
- Lane未設定
- Stage未設定
- Link説明不足
- 画像Nodeの説明不足

## 14. 品質基準

AI出力は以下を満たす。

- 担当が分かる
- 工程が分かる
- 処理順序が分かる
- 条件分岐が分かる
- 通信内容が分かる
- 補足コメントが分かる

## 15. 禁止事項

- 図面座標だけで意味を判断する
- Linkラベルだけを唯一の情報源にする
- Lane / Stage情報を省略する
- 画像Nodeの画像だけを出力して説明を省略する
- AI DSL仕様を頻繁に変更する

## 16. テスト観点

- Node / Link / Lane / Stageが出力される
- 担当と工程が欠落しない
- 条件付きLinkが出力される
- Commentが適切に出力される
- ループ構造が表現される
- JSONとAI DSLの内容が矛盾しない

## 17. 完了条件

- AIがFlow構造を理解できる情報が出力される
- 図面に依存せず構造化データから生成できる
- JSON / AI DSL / AI用テキストへ展開できる
