# 02_20_受入条件

## 1. 本書の目的

本書は、AI Flow Designer の要件定義に対する受入条件を定義する。

受入条件は、実装完了時にシステムが要求を満たしているかを判断するための基準である。

## 2. 基本方針

- 受入条件は機能、非機能、性能、セキュリティ、Export、AI理解性を対象とする
- 実装完了判定は画面表示だけでなく、DB、API、Export結果まで確認する
- 主要操作はE2Eテストで確認する
- 大規模Flowの性能確認を行う

## 3. 認証受入条件

- GitHub OAuthでログインできる
- 未認証で保護画面へアクセスするとログイン画面へ遷移する
- 認証済みでProject一覧を表示できる
- JWTなしのAPIアクセスは401になる
- ログアウトできる

## 4. Project受入条件

- Projectを作成できる
- Project一覧を表示できる
- Project詳細を表示できる
- ProjectAdminがProject設定を変更できる
- 権限のないProjectは表示されない

## 5. Flow受入条件

- Flowを作成できる
- Flow一覧を表示できる
- Flowエディタを開ける
- Flowを保存できる
- Flowを複写できる
- Flowを論理削除できる

## 6. Editor受入条件

- Nodeを追加、移動、編集、削除できる
- Linkを作成、再接続、編集、削除できる
- Laneを追加、編集、削除できる
- Stageを追加、編集、削除できる
- Commentを追加、編集、削除できる
- 複数選択できる
- コピー、貼付、Alt複製ができる
- Grid / Snapが動作する

## 7. Undo / Redo受入条件

- Node追加をUndo / Redoできる
- Node移動をUndo / Redoできる
- Link作成をUndo / Redoできる
- Template適用を1操作としてUndoできる
- Undo 1000回の試験に耐える

## 8. AutoSave受入条件

- 編集後dirty状態になる
- 30秒間隔でAutoSaveされる
- AutoSave成功後dirtyが解除される
- AutoSave失敗時に編集内容が失われない
- 手動保存できる

## 9. Version受入条件

- Versionを作成できる
- Version一覧を表示できる
- Version比較できる
- Version復元できる
- 復元前に現在状態がVersion保存される
- 復元後Undo / Redo履歴がクリアされる

## 10. Template受入条件

- Template一覧を表示できる
- Flow全体からTemplateを作成できる
- 選択範囲からTemplateを作成できる
- TemplateをFlowへ適用できる
- 適用時にIDが再採番される
- ViewerはTemplate適用できない

## 11. Export受入条件

- Mermaid Flowchartを出力できる
- Mermaid Sequenceを出力できる
- PDFを出力できる
- JSONを出力できる
- Export前Validationが動作する
- ExportしてもFlow構造は変更されない

## 12. AI出力受入条件

- JSONにFlow / Lane / Stage / Node / Link / Commentが含まれる
- Nodeの担当Laneが出力される
- Nodeの工程Stageが出力される
- Linkのfrom / toが出力される
- LinkのdataName / communicationType / conditionが出力される
- Commentが出力される
- AI DSLが将来追加できる構造である

## 13. 権限受入条件

- ViewerはFlow編集できない
- EditorはFlow編集できる
- ProjectAdminはメンバー管理できる
- Editorはメンバー管理できない
- 最後のProjectAdminを削除または降格できない
- 他ProjectのFlowを参照できない

## 14. セキュリティ受入条件

- 未認証APIは401になる
- 権限不足APIは403になる
- 不正な画像参照が拒否される
- 不正ファイルアップロードが拒否される
- XSS文字列が実行されない
- JWTやTokenがログ出力されない

## 15. 性能受入条件

- 1000 NodeのFlowを開ける
- 10000 LinkのFlowを開ける
- Flowエディタ初期表示が目標時間内に収まる
- AutoSave APIが目標時間内に応答する
- Exportが目標時間内に完了する
- Undo 1000回の試験に耐える

## 16. バックアップ・復旧受入条件

- DBバックアップを取得できる
- 画像ストレージをバックアップできる
- バックアップから復元できる
- 復元後にFlowを開ける
- 画像Nodeが表示される
- AutoSave失敗時に編集内容が失われない

## 17. テスト受入条件

- 単体テストが主要Serviceをカバーする
- API結合テストが主要APIをカバーする
- E2Eテストが主要画面遷移をカバーする
- 性能試験データで性能確認できる
- 回帰テストを実行できる

## 18. ドキュメント受入条件

- 02_要件定義が実装判断に使える
- 03_画面設計と矛盾しない
- 04_DB設計と矛盾しない
- 05_API設計と矛盾しない
- 06_フローエディタ設計と矛盾しない
- 11 / 12の実装設計へ展開できる

## 19. 完了判定

本章の受入条件を満たすことで、AI Flow Designer の初期開発要件を満たしたと判断する。

未達条件がある場合は、実装タスクまたは残課題として管理する。

## 20. 完了条件

- 機能受入条件が定義されている
- 非機能受入条件が定義されている
- 性能、セキュリティ、AI出力の受入条件が定義されている
- テスト仕様へ展開できる
