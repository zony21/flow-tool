# 02_13_監査ログ要件

## 1. 本書の目的

本書は、AI Flow Designer の監査ログ要件を定義する。

監査ログは、Project、Flow、Version、権限、Templateなどの重要操作について、誰が、いつ、何を行ったかを追跡できるようにするための要件である。

## 2. 基本方針

- 重要操作を監査対象とする
- 作成者、更新者、更新日時は主要テーブルの監査カラムで保持する
- 権限変更やVersion復元など影響の大きい操作は監査ログ対象とする
- 初期実装では監査ログ閲覧画面は必須としない
- 将来的に監査ログ検索画面を追加可能にする

## 3. 監査対象イベント

監査対象:

- ログイン成功
- ログイン失敗
- Project作成
- Project更新
- Project削除
- Flow作成
- Flow削除
- Flow複写
- Version作成
- Version復元
- Template作成
- Template更新
- Template削除
- メンバー追加
- メンバー削除
- 権限変更
- 画像アップロード
- 画像削除

## 4. 監査対象外

初期実装で詳細監査対象外とするもの:

- Node移動1回ごとの詳細ログ
- Link編集1回ごとの詳細ログ
- Hoverや選択状態
- Zoom / Pan
- AutoSave成功ごとの詳細ログ

ただし、Flow保存結果は通常の監査カラムで追跡する。

## 5. 記録項目

監査ログに記録する項目候補:

- auditLogId
- occurredAt
- userId
- githubLogin
- projectId
- flowId
- targetType
- targetId
- actionType
- result
- errorCode
- ipAddress
- userAgent
- traceId
- summary

## 6. actionType

代表的なactionType:

- LOGIN_SUCCESS
- LOGIN_FAILURE
- PROJECT_CREATE
- PROJECT_UPDATE
- PROJECT_DELETE
- FLOW_CREATE
- FLOW_DELETE
- FLOW_DUPLICATE
- VERSION_CREATE
- VERSION_RESTORE
- TEMPLATE_CREATE
- TEMPLATE_UPDATE
- TEMPLATE_DELETE
- MEMBER_ADD
- MEMBER_REMOVE
- ROLE_CHANGE
- IMAGE_UPLOAD
- IMAGE_DELETE

## 7. result

resultは以下とする。

- SUCCESS
- FAILURE

失敗時はerrorCodeを記録する。

## 8. 保存方式

初期実装では以下のいずれかとする。

- 監査ログ専用テーブルへ保存
- アプリケーションログへ出力

推奨は監査ログ専用テーブルである。

将来的に検索画面を追加するため、DB保存を前提に設計する。

## 9. 保持期間

初期方針:

- 監査ログ保持期間は1年を目安とする
- 個人開発段階では自動削除は必須としない
- 将来、保持期間を設定化する

## 10. 個人情報配慮

監査ログに不要な個人情報を保存しない。

保存しない情報:

- JWT
- GitHubアクセストークン
- ファイル実体パス
- 入力本文全文
- 画像バイナリ

## 11. 監査カラム要件

主要テーブルには以下を保持する。

- SYSTEM_CREATE_USER_ID
- SYSTEM_CREATE_DATETIME
- SYSTEM_UPDATE_USER_ID
- SYSTEM_UPDATE_DATETIME
- IS_DELETED

## 12. 監査ログ検索要件

将来の検索条件:

- 日時From / To
- userId
- projectId
- flowId
- actionType
- result
- traceId

## 13. Backend要件

- 重要操作のService層で監査ログを記録する
- 失敗時も可能な範囲で監査ログを残す
- traceIdと関連付ける
- 監査ログ記録失敗で本処理を失敗させるかは操作ごとに判断する

## 14. テスト観点

- Project作成時に監査ログが残る
- Version復元時に監査ログが残る
- 権限変更時に監査ログが残る
- 失敗時にerrorCodeが記録される
- JWTやTokenが記録されない

## 15. 完了条件

- 監査対象イベントが定義されている
- 記録項目が定義されている
- 個人情報やTokenを記録しない方針が明確である
- 将来の監査ログ検索へ拡張可能である
