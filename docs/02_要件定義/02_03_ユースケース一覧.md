# 02_03_ユースケース一覧

## 1. 本書の目的

本書は、AI Flow Designer の主要ユースケースを定義する。

ユースケースは、画面設計、API設計、DB設計、テスト仕様、実装タスクの基準となる。

## 2. アクター一覧

|アクター|説明|
|---|---|
|未認証ユーザー|ログイン前のユーザー|
|Viewer|Projectを閲覧できるユーザー|
|Editor|Flowを編集できるユーザー|
|ProjectAdmin|Projectとメンバーを管理できるユーザー|
|Admin|システム全体を管理できるユーザー|

## 4. UC-001 GitHubでログインする

目的:

- GitHub OAuthによりユーザーを認証する。

正常フロー:

1. ユーザーがログイン画面を開く
2. GitHubログインボタンを押す
3. GitHub認証画面へ遷移する
4. 認証成功後、Callbackへ戻る
5. JWTを取得する
6. Project一覧へ遷移する

異常フロー:

- GitHub認証失敗
- Callback不正
- ユーザー作成失敗

## 5. UC-005 Flowを編集する

目的:

- 構造化Flowを編集する。

前提:

- ユーザーがEditor以上である
- Flowが存在する
- 編集ロックを取得できる

正常フロー:

1. Flowエディタ画面を開く
2. Flow構造を取得する
3. 編集ロックを取得する
4. Node / Link / Lane / Stage / Commentを編集する
5. Command履歴へ追加する
6. AutoSave対象にする
7. 保存する

異常フロー:

- 編集ロック取得失敗
- Revision競合
- 保存失敗
- 権限不足

## 6. UC-006 Nodeを追加・編集する

操作:

- Node追加
- Node移動
- Nodeリサイズ
- Node削除
- Nodeプロパティ更新

業務ルール:

- NodeはLane / Stageに所属する
- NodeTypeは拡張可能
- Node削除時は関連Linkも削除対象
- 画像Nodeは画像メタデータを参照する

## 7. UC-007 Linkを追加・編集する

操作:

- Link作成
- Link再接続
- Link削除
- Link条件編集
- Link通信種別編集

業務ルール:

- LinkはfromNodeとtoNodeを持つ
- ループは許可する
- Linkは通信や条件など意味情報を保持する

## 8. UC-010 Undo / Redoする

前提:

- Editor以上
- 編集ロック取得済
- Command履歴が存在する

業務ルール:

- 選択状態はUndo対象外
- Command単位で取り消す
- Template適用は1操作としてUndoする

## 9. UC-014 Versionを復元する

前提:

- Editor以上
- 編集ロック取得済
- 対象Versionが存在する

正常フロー:

1. Version管理画面を開く
2. 復元対象Versionを選択する
3. 復元確認を行う
4. 現在状態を復元前Versionとして保存する
5. 対象VersionをFlowへ反映する
6. Flowエディタへ戻る

## 10. UC-017 FlowをExportする

前提:

- Viewer以上
- Flowが存在する

正常フロー:

1. Export画面を開く
2. Export形式を選択する
3. 出力前Validationを行う
4. プレビューを表示する
5. ダウンロードする

## 11. UC-019 メンバーを管理する

前提:

- ProjectAdmin以上

操作:

- メンバー追加
- 権限変更
- メンバー削除

業務ルール:

- 最後のProjectAdminは削除不可
- Admin権限はProject画面では付与しない
- USER_ACCOUNT自体は削除しない

## 12. 共通異常系

全ユースケース共通の異常:

- 未認証
- 権限なし
- 対象データなし
- Revision競合
- Validationエラー
- 通信失敗
- サーバーエラー

## 13. 完了条件

- 主要アクターが定義されている
- 主要ユースケースが一覧化されている
- 正常フローと異常フローの基準がある
- 画面・API・テストへ展開できる
