# 02_09_性能要件

## 1. 本書の目的

本書は、AI Flow Designer の性能要件を定義する。

性能要件は、Flowエディタ、API、DB、Export、Version、Undo / Redoの実装および性能試験の基準とする。

## 2. 基本方針

- 大規模Flowでも実用可能な操作感を維持する
- 1000 Node、10000 Linkを性能試験対象とする
- Undo 1000回を性能試験対象とする
- 初期実装では単一サーバー構成を前提とする
- 将来的にSQL Serverへ移行可能にする

## 3. 対象データ規模

|対象|目標規模|
|---|---:|
|Project数 / ユーザー|100|
|Flow数 / Project|500|
|Lane数 / Flow|50|
|Stage数 / Flow|100|
|Node数 / Flow|1000|
|Link数 / Flow|10000|
|Comment数 / Flow|1000|
|Version数 / Flow|100|
|Template数 / Project|500|

## 4. 画面表示性能

|処理|目標|
|---|---:|
|ログイン後Project一覧表示|3秒以内|
|Project詳細表示|3秒以内|
|Flow一覧表示|3秒以内|
|Flowエディタ初期表示|5秒以内|
|1000 Node描画|3秒以内|
|10000 Link描画|5秒以内|
|プロパティ切替|200ms以内|
|選択切替|200ms以内|

## 5. エディタ操作性能

|操作|目標|
|---|---:|
|Nodeドラッグ追従|体感遅延なし|
|Node追加|300ms以内|
|Node削除|500ms以内|
|Link作成|500ms以内|
|複数選択|500ms以内|
|Zoom / Pan|体感遅延なし|
|Fit View|1秒以内|

## 6. Undo / Redo性能

|操作|目標|
|---|---:|
|Undo 1回|500ms以内|
|Redo 1回|500ms以内|
|Undo 1000回連続|操作可能|
|100 Node貼付Undo|2秒以内|
|Template適用Undo|2秒以内|

## 7. AutoSave性能

AutoSave間隔:

```text
30秒
```

目標:

- 差分保存を基本とする
- AutoSave API応答は3秒以内
- 保存中でも編集を極力妨げない
- 失敗時は次回再試行できる

## 8. API応答性能

|API分類|目標|
|---|---:|
|一覧取得|2秒以内|
|詳細取得|3秒以内|
|Flow保存|5秒以内|
|AutoSave|3秒以内|
|Version作成|5秒以内|
|Version復元|10秒以内|
|Template適用前取得|3秒以内|
|Export生成|10秒以内|

## 9. Export性能

|形式|目標|
|---|---:|
|Mermaid Flowchart|5秒以内|
|Mermaid Sequence|5秒以内|
|JSON|3秒以内|
|PDF|10秒以内|
|AI DSL|5秒以内|

PDFはFlow規模により時間が増える可能性があるため、長時間処理時はProgress表示を行う。

## 10. DB性能

要件:

- ProjectId、FlowId、IsDeletedにIndexを設定する
- Flow読込時に必要なデータを効率よく取得する
- 一覧取得では巨大JSONやVersion Snapshotを取得しない
- Template一覧ではTemplate JSON本文を取得しない
- Version一覧ではSnapshot本文を取得しない

## 11. メモリ要件

フロントエンド:

- 大規模Flow読込時もブラウザが操作不能にならないこと
- 画像本体を不要にStoreへ保持しないこと
- ViewModel Cacheを適切に破棄すること

バックエンド:

- ExportやVersion作成時に過剰なメモリ使用を避けること

## 12. 同時利用要件

初期想定:

- 同時ログインユーザー: 50
- 同一Project同時閲覧: 20
- 同一Flow同時閲覧: 10
- 同一Flow編集者: 1

同一Flowの同時編集は初期対象外であり、編集ロックにより1人に制限する。

## 13. 性能劣化時の要件

性能劣化が発生した場合:

- Loading表示を行う
- Progress表示を行う
- ユーザー操作を完全に固めない
- 保存失敗時は再試行可能にする
- Export長時間処理ではキャンセルを将来検討する

## 14. 性能試験データ

性能試験用に以下を用意する。

- 小規模Flow: 50 Node / 100 Link
- 中規模Flow: 300 Node / 1000 Link
- 大規模Flow: 1000 Node / 10000 Link
- Comment大量Flow
- ImageNode大量Flow
- Version大量Flow
- Template大量Project

## 15. ボトルネック想定

想定ボトルネック:

- Vueの大量再描画
- Linkラベル大量DOM
- Flow全体deep watch
- AutoSave時の全量送信
- Version Snapshotの巨大化
- PDF生成
- Template JSONの巨大化

## 16. 性能対策方針

- 差分保存
- ViewModel Cache
- Linkラベル簡略表示
- deep watch禁止
- 一覧取得で本文取得しない
- Index設計
- 大量処理のProgress表示

- 主要性能目標が数値化されている
- 性能試験対象データ規模が定義されている
- ボトルネックと対策方針が定義されている
- 06、11、12、13、19章の設計へ展開できる
