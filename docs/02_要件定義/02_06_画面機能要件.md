# 02_06_画面機能要件

## 1. 本書の目的

本書は、AI Flow Designer の画面機能要件を定義する。

03_画面設計ではUIレイアウトやコンポーネント構成を定義するが、本書では各画面が業務上満たすべき機能要件、前提条件、入力、出力、エラー条件を定義する。

## 2. 共通画面要件

全画面共通で以下を満たす。

- 認証が必要な画面ではJWTを検証する
- Project配下画面ではProject権限を検証する
- API失敗時は共通エラー形式で表示する
- Loading状態を表示する
- 権限不足時は操作を非表示または非活性にする
- 破壊的操作は確認ダイアログを表示する

## 3. ログイン画面要件

画面責務:

- GitHub OAuthログインを開始する

入力:

- なし
- redirect URLはクエリパラメータで受け取る

出力:

- 認証成功時、Project一覧へ遷移
- redirect指定がある場合は元画面へ遷移

エラー条件:

- OAuth失敗
- redirect不正

## 4. Project一覧画面要件

画面責務:

- ユーザーが参加しているProjectを一覧表示する
- Projectを新規作成する

入力:

- 検索キーワード
- 権限フィルタ
- 更新日条件

出力:

- Project一覧
- Project詳細への遷移

業務ルール:

- 自分が参照権限を持つProjectのみ表示する
- Project作成時、作成者をProjectAdminにする

## 5. Project詳細画面要件

画面責務:

- Project基本情報を表示する
- Flow、Template、Member、Settingへの導線を提供する

前提条件:

- Projectが存在する
- Viewer以上の権限がある

出力:

- Project概要
- Flowサマリー
- 最近更新されたFlow

## 6. Flow一覧画面要件

画面責務:

- Project配下のFlowを一覧表示する
- Flowを作成、複写、削除する

業務ルール:

- ViewerはFlowを閲覧できる
- Editor以上はFlowを作成できる
- ロック中Flowは削除不可
- 削除は論理削除とする

## 7. Flowエディタ画面要件

画面責務:

- Flow構造を編集する
- Node / Link / Lane / Stage / Commentを編集する
- AutoSave、Undo / Redo、Version、Template、Exportへ連携する

前提条件:

- Flowが存在する
- Viewer以上の権限がある

編集可能条件:

- Editor以上
- 編集ロック取得済

入力:

- Node操作
- Link操作
- Lane / Stage操作
- Comment操作
- プロパティ入力
- ショートカット操作

出力:

- 更新されたFlow構造
- dirty状態
- Validation結果

業務ルール:

- Vue Flowの状態を正としない
- 構造化データを正とする
- すべての編集操作をCommand化する
- AutoSaveは差分のみ保存する

## 8. テンプレート管理画面要件

画面責務:

- Templateを検索、閲覧、作成、編集、削除、適用する

業務ルール:

- Viewerは閲覧のみ
- Editor以上は適用可能
- テンプレート適用はFlowエディタ上でCommand化する
- テンプレート内IDは適用時に再採番する

## 9. Version管理画面要件

画面責務:

- Flow Versionを一覧、比較、復元する

業務ルール:

- Viewerは一覧と比較が可能
- 復元はEditor以上かつ編集ロック取得済のみ可能
- 復元前に現在状態をVersionとして保存する
- 復元後はUndo / Redo履歴をクリアする

## 10. Export画面要件

画面責務:

- Flow構造から成果物を出力する

出力形式:

- Mermaid Flowchart
- Mermaid Sequence
- PDF
- JSON
- 将来 AI DSL

業務ルール:

- Viewer以上でExport可能
- Export前にValidationを実行する
- Flow構造を正として成果物を生成する

## 11. 設定画面要件

画面責務:

- ユーザー設定とProject設定を編集する

ユーザー設定:

- テーマ
- Grid / Snap
- ショートカット
- AutoSave
- Export既定値

Project設定:

- ProjectAdmin以上のみ編集可能

## 12. メンバー管理画面要件

画面責務:

- Projectメンバーと権限を管理する

業務ルール:

- ProjectAdmin以上のみ利用可能
- 最後のProjectAdminは削除不可
- USER_ACCOUNT自体は削除しない
- Admin権限はProject画面では付与しない

## 13. 共通ダイアログ要件

対象:

- ConfirmDialog
- InputDialog
- ValidationDialog
- ProgressDialog
- Toast
- LoadingOverlay

業務ルール:

- 削除や復元など破壊的操作は確認必須
- Validationエラーは項目単位と一覧の両方で表示できる
- APIエラーは共通形式で表示する

## 14. 画面間遷移要件

未保存変更がある状態で画面遷移する場合、確認ダイアログを表示する。

選択肢:

- 保存して移動
- 保存せず移動
- キャンセル

## 15. エラー条件

共通エラー:

- 未認証
- 権限なし
- 対象なし
- 通信失敗
- Revision競合
- 編集ロック競合
- Validationエラー

## 16. テスト観点

- 各画面が権限に応じて表示制御できる
- 各画面が必要APIを呼び出せる
- 未保存遷移確認が動作する
- Viewerでは編集操作ができない
- Editor以上で編集できる
- ProjectAdminのみメンバー管理できる

## 17. 完了条件

- 各画面の責務が定義されている
- 入力、出力、前提条件、業務ルールが定義されている
- 03_画面設計へ詳細展開できる
