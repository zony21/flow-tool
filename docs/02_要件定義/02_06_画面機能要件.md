# 02_06_画面機能要件

## 1. 本書の目的

本書は、AI Flow Designer の各画面が満たすべき機能要件を定義する。

03_画面設計がUIレイアウトやコンポーネントを定義するのに対し、本書では画面ごとの責務、入力、出力、業務ルール、遷移条件、エラー条件を要件として定義する。

## 2. 画面機能要件の共通方針

すべての画面は以下を満たす。

- 認証が必要な画面では未認証アクセスを拒否する
- Project配下画面ではProject権限を確認する
- APIエラーは共通エラー表示に変換する
- Loading状態を表示する
- 権限に応じて操作ボタンを制御する
- 入力項目は保存前にValidationする

## 3. ログイン画面要件

画面責務:

- GitHub OAuthログインを開始する
- 認証済みユーザーをProject一覧へ遷移させる

入力:

- redirectパラメータ

出力:

- GitHubログイン導線
- 認証エラー表示

業務ルール:

- redirectは内部URLのみ許可する
- 認証済みの場合はログイン画面を表示しない

## 4. Project一覧画面要件

画面責務:

- 参加Project一覧を表示する
- Projectを作成する
- Project詳細へ遷移する

入力:

- 検索キーワード
- 権限フィルタ
- 更新日条件

出力:

- Project一覧
- 自分の権限
- Flow件数

業務ルール:

- 自分が参加しているProjectのみ表示する
- Project作成時は作成者をProjectAdminにする

## 5. Project詳細画面要件

画面責務:

- Projectの概要を表示する
- Flow一覧、Template、設定、メンバー管理への導線を提供する

入力:

- projectId

出力:

- Project詳細
- 最近更新Flow
- 操作メニュー

業務ルール:

- Viewer以上のみ閲覧可
- ProjectAdmin以上のみ設定・メンバー管理へ遷移可

## 6. Flow一覧画面要件

画面責務:

- Project配下Flowを一覧表示する
- Flow作成、複写、削除を行う
- Flowエディタへ遷移する

入力:

- projectId
- 検索条件

出力:

- Flow一覧
- ロック状態
- Revision

業務ルール:

- ロック中Flowは削除不可
- Viewerは作成・複写・削除不可
- Flow削除は論理削除とする

## 7. Flowエディタ画面要件

画面責務:

- Flow構造を編集する
- Node / Link / Lane / Stage / Commentを操作する
- Undo / Redo、AutoSave、Version、Template、Exportへ連携する

入力:

- projectId
- flowId
- 編集操作

出力:

- Flow図
- プロパティ
- Validation
- 保存状態

業務ルール:

- 正は構造化データである
- 編集操作はCommand化する
- 編集にはロック取得が必要
- Viewerは読み取り専用
- AutoSaveは差分のみ対象とする

## 8. Template管理画面要件

画面責務:

- Templateを検索・表示・作成・編集・削除・適用する

入力:

- 検索条件
- Template作成情報
- Template編集情報

出力:

- Template一覧
- Template詳細
- Preview

業務ルール:

- 標準Template編集はAdminのみ
- Template適用はEditor以上
- Template JSONは保存前にValidationする

## 9. Version管理画面要件

画面責務:

- Flow Versionを一覧表示する
- Versionを比較する
- Versionを復元する

入力:

- flowId
- 比較対象Version
- 復元対象Version

出力:

- Version一覧
- 差分結果
- 復元結果

業務ルール:

- 復元にはEditor以上と編集ロックが必要
- 復元前に現在状態をVersion保存する
- 復元後はUndo / Redo履歴をクリアする

## 10. Export画面要件

画面責務:

- Flowを各形式へ出力する

入力:

- Export形式
- Exportオプション

出力:

- Mermaid
- PDF
- JSON
- AI DSL

業務ルール:

- Viewer以上でExport可能
- Export前にValidationする
- 出力形式ごとに固有Validationを行う

## 11. 設定画面要件

画面責務:

- ユーザー設定を編集する
- Project設定を編集する

入力:

- 表示設定
- Editor設定
- Shortcut設定
- AutoSave設定
- Export既定値

出力:

- 保存済み設定

業務ルール:

- Shortcut重複不可
- AutoSave間隔は最小値以上
- Project設定はProjectAdmin以上のみ

## 12. メンバー管理画面要件

画面責務:

- Projectメンバーを管理する

入力:

- GitHubユーザー名
- 権限

出力:

- メンバー一覧
- 権限変更結果

業務ルール:

- 最後のProjectAdminは削除不可
- Admin権限はProject画面で付与しない
- USER_ACCOUNT自体は削除しない

## 13. 共通エラー要件

各画面は以下のエラーを共通形式で扱う。

- 401 未認証
- 403 権限なし
- 404 対象なし
- 409 競合
- 422 Validation
- 500 システムエラー

## 14. 完了条件

- 画面ごとの責務が明確である
- 入力、出力、業務ルールが定義されている
- 03_画面設計へ展開できる
- API設計、Store設計、E2Eテストへ展開できる
