# 02_07_編集機能要件

## 1. 本書の目的

本書は、AI Flow Designer のFlowエディタにおける編集機能要件を定義する。

編集機能は、構造化データを正としてFlowを作成・変更するための中核機能である。

## 2. 基本方針

- 編集対象は構造化データである
- Vue Flowの表示状態を保存の正としない
- すべての編集操作はCommand化する
- 編集操作はUndo / Redo対象にする
- 編集後はAutoSave対象にする
- Flow Versionには編集後の構造をSnapshotとして保存できる

## 3. 編集対象

編集対象:

- Flow
- Lane
- Stage
- Node
- Link
- Comment

編集対象外:

- 選択状態
- Hover状態
- Zoom / Pan
- Tooltip表示
- 一時的なドラッグ中状態

## 4. Node編集要件

Nodeに対して以下を行えること。

- 追加
- 移動
- 複数移動
- リサイズ
- 削除
- コピー
- 貼付
- Alt複製
- 表示名編集
- 説明編集
- NodeType変更
- Lane / Stage変更
- extensionJson編集

## 5. Node業務ルール

- Nodeは必ずFlowに所属する
- NodeはLaneとStageに所属する
- NodeTypeは拡張可能である
- Node削除時は関連Linkも削除対象とする
- 画像Nodeは画像メタデータを参照する
- Nodeの座標はCanvas座標で保持する

## 6. Link編集要件

Linkに対して以下を行えること。

- 作成
- 再接続
- 削除
- dataName編集
- communicationType編集
- condition編集
- description編集
- linkType編集
- Bezier制御点編集

## 7. Link業務ルール

- LinkはfromNodeIdとtoNodeIdを必須とする
- fromNodeとtoNodeは同一Flow内に存在すること
- ループ接続を許可する
- 自己接続を許可する
- Linkは単なる線ではなく通信・条件・データ移動を表す

## 8. Lane編集要件

Laneに対して以下を行えること。

- 追加
- 名称変更
- カテゴリ変更
- アイコン変更
- 色変更
- 説明変更
- 並び替え
- 削除

## 9. Lane削除要件

Lane削除時は、配下Nodeの扱いを必ず選択する。

選択肢:

- 別Laneへ移動
- Nodeごと同時削除

暗黙削除は禁止する。

## 10. Stage編集要件

Stageに対して以下を行えること。

- 追加
- 名称変更
- 色変更
- 説明変更
- 並び替え
- 削除

## 11. Stage削除要件

Stage削除時もLane削除と同様に、配下Nodeの扱いを選択する。

選択肢:

- 別Stageへ移動
- Nodeごと同時削除

## 12. Comment編集要件

Commentに対して以下を行えること。

- 独立Comment追加
- Node紐付けComment追加
- 本文編集
- 移動
- リサイズ
- 削除
- Node紐付け解除

## 13. 複数選択要件

複数選択で以下を行えること。

- 複数Node移動
- 複数Node削除
- 複数Nodeコピー
- 選択範囲内Linkコピー
- 一括プロパティ変更

## 14. コピー・貼付要件

コピー対象:

- Node
- 選択範囲内Link
- Comment

貼付時:

- IDを再採番する
- Linkのfrom/toを新Nodeへ張り替える
- 座標を少しずらす
- 貼付後要素を選択状態にする

## 15. Grid / Snap要件

- Gridを表示できる
- Gridサイズを設定できる
- Snap有効/無効を切り替えられる
- Node移動、Resize、Comment移動、Link制御点移動でSnapする

## 16. プロパティ編集要件

選択対象に応じてプロパティを編集できること。

- Nodeプロパティ
- Linkプロパティ
- Laneプロパティ
- Stageプロパティ
- Commentプロパティ
- Flowプロパティ

## 17. Undo / Redo要件

以下はUndo / Redo対象とする。

- Node編集
- Link編集
- Lane編集
- Stage編集
- Comment編集
- Template適用
- 貼付
- 複製

Undo / Redo対象外:

- 選択状態
- Zoom / Pan
- Hover

## 18. AutoSave要件

編集後はdirtyとして扱う。

AutoSave対象:

- dirtyNodes
- dirtyLinks
- dirtyLanes
- dirtyStages
- dirtyComments
- deletedIds

## 19. 排他制御要件

編集にはFlow編集ロックが必要である。

編集ロックがない場合:

- 読み取り専用表示
- 編集操作不可
- Exportは可

## 20. Validation要件

編集時または保存時に以下を検証する。

- Node名必須
- NodeType存在
- Lane / Stage存在
- Link from/to存在
- 削除済Node参照なし
- 画像Nodeの画像参照

## 21. 異常系

- 編集ロックなし
- 権限不足
- Revision競合
- 不正なNode参照
- 不正なLink参照
- AutoSave失敗
- Undo / Redo失敗

## 22. テスト観点

- Node追加、移動、削除ができる
- Link作成、再接続、削除ができる
- Lane / Stage削除時に選択肢が出る
- Comment編集ができる
- Undo / Redoできる
- AutoSave対象になる
- Viewerでは編集できない

## 23. 完了条件

- 編集対象と編集対象外が分離されている
- 編集操作がCommand化される
- AutoSave / Version / UndoRedoと整合する
