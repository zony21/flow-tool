# 02_19_制約事項

## 1. 本書の目的

本書は、AI Flow Designer の制約事項を定義する。

制約事項は、初期開発で守るべき前提、対象外、技術的制限、将来拡張時の注意点を明確にするためのものである。

## 2. 技術制約

初期開発では以下の技術を使用する。

- Frontend: Vue3
- Language: TypeScript
- UI: PrimeVue
- Flow描画: Vue Flow
- 状態管理: Pinia
- Build: Vite
- Backend: ASP.NET Core .NET8
- ORM: Entity Framework Core
- DB: SQLite
- Auth: GitHub OAuth

## 3. DB制約

初期DBはSQLiteとする。

制約:

- 大規模同時書き込みには向かない
- 複雑な運用監視はSQL Serverより弱い
- 将来SQL Serverへ移行できるよう設計する
- SQL Server固有機能へ初期から依存しない

## 4. 認証制約

初期認証はGitHub OAuthのみとする。

対象外:

- 独自ID / パスワード認証
- SAML
- LDAP
- 多要素認証

将来的にSSOやMFAは追加検討する。

## 5. 共同編集制約

初期開発ではリアルタイム共同編集を対象外とする。

同一Flow編集は編集ロックにより1ユーザーに制限する。

他ユーザーは読み取り専用で開く。

## 6. コード生成制約

初期開発ではコード自動生成を対象外とする。

ただし、将来的にコード生成へ拡張できるよう、AI DSLや構造化データは意味情報を保持する。

## 7. AIチャット制約

初期開発ではAIチャット機能を実装しない。

AIへ渡すためのExportは対象とするが、アプリ内でLLMと会話する機能は対象外とする。

## 8. AI DSL制約

AI DSLは独自仕様として固定する。

制約:

- 一度確定した基本構文を頻繁に変更しない
- 破壊的変更を避ける
- schemaVersionを持つ
- JSON出力と矛盾しない

## 9. NodeType制約

NodeTypeは初期種類を持つが、拡張可能とする。

初期種類:

- 開始
- 終了
- 処理
- 判定
- 六角形
- 吹き出し
- 画像

## 10. Link制約

初期LinkはBezierを基本とする。

将来的に以下へ拡張する。

- 直線
- 折れ線
- Bezier切替

ループは許可する。

## 11. Export制約

初期Exportは以下のみ必須とする。

- Mermaid Flowchart
- Mermaid Sequence
- PDF
- JSON

AI DSL、API仕様、DB更新一覧、PLC一覧、通信一覧、設計書ドラフトは将来対応とする。

## 12. 画像制約

画像本体はDBへ保存しない。

DBには画像メタデータのみ保存する。

制約:

- 許可形式のみアップロード可能
- サイズ上限を設ける
- 画像参照にはProject権限が必要

## 13. 性能制約

性能試験対象は以下とする。

- 1000 Node
- 10000 Link
- Undo 1000回

これを超える規模は将来最適化対象とする。

## 14. ブラウザ制約

初期対応ブラウザ:

- Google Chrome 最新版
- Microsoft Edge 最新版

Safari / Firefox は将来検証対象とする。

## 15. モバイル制約

初期開発ではスマートフォン最適化を対象外とする。

FlowエディタはPCブラウザでの利用を前提とする。

## 16. オフライン制約

初期開発ではオフライン編集を対象外とする。

ネットワーク接続が必要である。

## 17. 運用制約

初期運用では小規模チーム利用を前提とする。

大規模組織向けの監査画面、SSO、細粒度権限は将来拡張とする。

## 18. 設計書制約

設計書は段階生成で詳細化する。

ただし既存章を削除してダミーへ戻してはならない。

更新時は既存内容を維持し、必要に応じて追記・統合する。

## 19. 完了条件

- 初期対象外が明確である
- 技術制約が明確である
- 将来拡張時の前提が明確である
- 実装判断時の迷いを減らせる
