# 02_12_権限要件

## 1. 本書の目的

本書は、AI Flow Designer の権限要件を定義する。

権限要件は、画面表示制御、API認可、Project管理、Flow編集、Template、Version、Export、メンバー管理の判断基準となる。

## 2. 権限種別

本システムの権限は以下とする。

- Admin
- ProjectAdmin
- Editor
- Viewer

## 3. Admin

Adminはシステム全体管理者である。

可能操作:

- 標準テンプレート管理
- 全Project参照
- システム設定
- 将来の監査ログ閲覧

初期実装ではAdmin機能は最小限とし、Project内操作ではProjectAdmin相当以上として扱う。

## 4. ProjectAdmin

ProjectAdminはProject単位の管理者である。

可能操作:

- Project情報更新
- Projectメンバー管理
- 権限変更
- Flow作成、編集、削除
- Template管理
- Version復元
- Export

## 5. Editor

EditorはFlow編集者である。

可能操作:

- Flow作成
- Flow編集
- Node / Link / Lane / Stage / Comment編集
- Template作成、適用
- Version作成、復元
- Export

不可操作:

- メンバー管理
- Project削除
- 他ユーザー権限変更

## 6. Viewer

Viewerは閲覧者である。

可能操作:

- Project閲覧
- Flow閲覧
- Version閲覧、比較
- Export

不可操作:

- Flow編集
- Template適用
- Version復元
- メンバー管理

## 8. API単位権限

Backend APIでは必ず権限確認を行う。

- GET系参照API: Viewer以上
- POST / PUT / DELETE系編集API: Editor以上
- Projectメンバー管理API: ProjectAdmin以上
- 標準テンプレート管理API: Adminのみ

## 9. Flow編集ロックとの関係

Flow編集には権限とロックの両方が必要である。

編集可能条件:

```text
Editor以上 AND Flow編集ロック取得済
```

Editor以上でもロック未取得の場合は読み取り専用とする。

復元には編集ロックも必要である。

## 12. Export権限

ExportはViewer以上で可能とする。

理由:

- Exportは閲覧データの派生成果物である
- 編集操作ではない

ただし、将来的にProject設定でExport制限を追加可能とする。

## 13. メンバー管理権限

ProjectAdmin以上のみ可能。

可能操作:

- メンバー一覧
- メンバー追加
- 権限変更
- メンバー削除

## 14. 権限変更ルール

- ProjectAdminはEditor / Viewer / ProjectAdminを付与できる
- Admin付与はAdminのみ可能
- 最後のProjectAdminを削除または降格してはならない
- 自分自身を最後のProjectAdminから降格できない

## 15. 権限チェック処理

権限チェックは以下の順に行う。

1. 認証済み確認
2. Project存在確認
3. Project参加確認
4. Role確認
5. Flow操作の場合はFlow所属確認
6. 編集操作の場合は編集ロック確認

## 16. データ参照制御

ユーザーは自分が参加していないProjectのデータを参照できない。

対象:

- Flow
- Version
- Template
- Image
- Export結果
- Comment

## 17. Frontend表示制御

Frontendでは権限に応じてボタンを非表示または非活性にする。

ただし、Frontend制御は補助であり、Backend認可を必須とする。

## 18. 異常系

|状況|HTTP|扱い|
|---|---:|---|
|未認証|401|ログインへ誘導|
|Project未参加|403|権限なし|
|Role不足|403|操作不可|
|ロックなし|409|読み取り専用|
|対象なし|404|NotFound|

## 19. テスト観点

- Viewerが編集APIを実行できない
- Editorがメンバー管理できない
- ProjectAdminが権限変更できる
- 最後のProjectAdminを削除できない
- 他ProjectのFlowを参照できない
- 編集ロックなしで保存できない

## 20. 完了条件

- Roleごとの操作範囲が定義されている
- 画面・API・Template・Version・Exportの権限が定義されている
- Backend認可チェックが必須である
- 最後のProjectAdmin保護が定義されている
