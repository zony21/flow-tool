# 02_01_システム目的

## 1. 本書の目的

本書は、AI Flow Designer のシステム目的を定義する。

本システムは、AIが高精度で理解できるシステムフローを作成するためのWebアプリケーションである。

ExcelやVisioのように図形を配置するだけのツールではなく、図形に意味を持たせ、構造化データを正として保持する。

## 2. 開発目的

AI Flow Designer の開発目的は、以下である。

- 人間が理解しやすいフロー図を作成する
- AIが理解しやすい構造化データを作成する
- 図面と設計情報のズレを防ぐ
- Mermaid、PDF、JSON、AI DSLなど複数形式へ出力する
- システム設計、業務設計、設備連携設計を再利用可能にする
- 将来的なAI実装支援、設計レビュー、仕様確認に利用できる基盤を作る

## 3. 解決したい課題

既存の図形エディタには以下の課題がある。

- 図形に意味がない
- 線が何を表すか曖昧
- 担当や工程が図面上の見た目に依存する
- AIが正確に読み取れない
- 設計書、API仕様、DB仕様、通信仕様と図面が分離する
- フロー更新時に各成果物を手作業で修正する必要がある
- 過去フローやテンプレートの再利用性が低い

AI Flow Designer は、これらを構造化データ中心の設計により解決する。

## 4. 中核コンセプト

中核コンセプトは Single Source of Truth である。

正となるのは図面ではなく、以下の構造化データである。

- Project
- Flow
- Lane
- Stage
- Node
- Link
- Comment
- Version
- Template

図面、Mermaid、PDF、JSON、AI DSLなどは、この構造化データから生成される派生成果物である。

## 5. 目標とする成果物

本システムは以下を生成できることを目標とする。

初期対象:

- フロー図
- Mermaid Flowchart
- Mermaid Sequence
- PDF
- JSON

将来対象:

- AI DSL
- API仕様
- DB更新一覧
- PLC一覧
- 通信一覧
- 設計書ドラフト

## 6. 利用者

想定利用者:

- システム設計者
- 業務設計者
- WCS / RCS / PLC / AGF 関連設計者
- フロントエンド開発者
- バックエンド開発者
- AIに設計内容を渡したい利用者
- 設計レビュー担当者

## 7. システムで重視する品質

重視する品質:

- 構造化データの正確性
- AI理解性
- 編集操作の安全性
- Version復元性
- Export再現性
- 共同利用時の権限制御
- 大規模Flowでも操作可能な性能

## 8. 成功条件

本システムの成功条件は以下である。

- ユーザーがFlowを構造化データとして作成できる
- AIがFlow内容を高精度に理解できる
- Node / Link / Lane / Stage の意味が明確である
- 図面とJSONが矛盾しない
- Export成果物が同一データから生成される
- Version復元により変更前へ戻せる
- Undo / Redo により編集ミスを回復できる

## 9. 非目的

初期開発で対象外とするもの:

- コード自動生成
- AIチャット機能
- リアルタイム共同編集
- PLCやWCSへの直接接続
- 外部システムへの自動デプロイ

ただし、将来拡張を妨げない設計とする。

## 10. 完了条件

- システム目的が明確である
- 解決課題が定義されている
- Single Source of Truth方針が要件として明文化されている
- 初期対象と将来対象が分離されている
