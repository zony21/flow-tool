# 02_02_システムスコープ

## 1. 本書の目的

本書は、AI Flow Designer のシステムスコープを定義する。

スコープを明確にすることで、初期開発で実装する範囲、将来拡張する範囲、対象外とする範囲を区別し、設計と実装のブレを防ぐ。

## 2. 初期開発スコープ

初期開発で対象とする範囲は以下である。

- GitHub OAuthログイン
- Project管理
- Flow管理
- Lane管理
- Stage管理
- Node管理
- Link管理
- Comment管理
- Flowエディタ
- Undo / Redo
- AutoSave
- Version管理
- Template管理
- Mermaid出力
- PDF出力
- JSON出力
- 画像Node用の画像管理
- 権限管理
- ユーザー設定

## 3. 将来拡張スコープ

将来対応する範囲は以下である。

- AI DSL出力
- API仕様生成
- DB更新一覧生成
- PLC一覧生成
- 通信一覧生成
- 設計書ドラフト生成
- MiniMap
- Link線種切替
- SVG出力
- SQL Server移行
- AIチャット機能
- コード生成
- リアルタイム共同編集

## 4. 初期対象外

初期開発では以下を対象外とする。

- AIチャットによる自動設計
- 直接的なソースコード生成
- PLCやWCSへの直接通信
- 外部DBへの直接反映
- リアルタイム同時編集
- スマートフォン最適化
- オフライン編集

## 5. システム境界

AI Flow Designer はWebアプリケーションとして提供する。

構成:

- Vue3 フロントエンド
- ASP.NET Core .NET8 バックエンド
- SQLite DB
- GitHub OAuth
- 画像ファイルストレージ

## 6. データ管理対象

本システムで管理する主なデータ:

- ユーザー
- Project
- Flow
- Flow Version
- Lane
- Stage
- NodeType
- Flow Node
- Flow Link
- Flow Comment
- Template
- User Setting
- 画像メタデータ

## 7. 管理しないデータ

初期開発では以下を管理しない。

- GitHubのパスワード
- GitHubアクセストークンの不要な長期保存
- 外部PLCの実データ
- 実機AGFの状態データ
- 外部DBの本番データ
- 生成コードの実行結果

## 8. 外部サービス連携

初期連携:

- GitHub OAuth

将来連携候補:

- GitHub Repository
- OpenAI API
- 外部ストレージ
- SQL Server
- CI/CD

## 9. ユーザー種別

システム内の主なユーザー種別:

- Admin
- ProjectAdmin
- Editor
- Viewer

各権限の詳細は権限要件で定義する。

## 10. 画面スコープ

初期対象画面:

- ログイン
- Project一覧
- Project詳細
- Flow一覧
- Flowエディタ
- テンプレート管理
- Version管理
- Export
- 設定
- メンバー管理

## 11. Exportスコープ

初期Export:

- Mermaid Flowchart
- Mermaid Sequence
- PDF
- JSON

将来Export:

- AI DSL
- API仕様
- DB更新一覧
- PLC一覧
- 通信一覧
- 設計書ドラフト

## 12. 性能スコープ

性能要件の対象:

- 1000 Node
- 10000 Link
- Undo 1000回
- AutoSave 30秒間隔
- 大規模Flowの初期表示

## 13. セキュリティスコープ

対象:

- GitHub OAuth
- JWT認証
- Project権限制御
- API認可
- 編集ロック
- 画像アクセス制御

## 14. 対象ブラウザ

初期対象:

- Google Chrome 最新版
- Microsoft Edge 最新版

Safari / Firefox は将来検証対象とする。

## 15. 完了条件

- 初期開発対象と将来対象が分離されている
- 対象外が明確である
- 外部連携範囲が明確である
- 性能・セキュリティの対象範囲が定義されている
