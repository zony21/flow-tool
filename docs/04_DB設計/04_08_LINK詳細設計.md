# 04_08_LINK詳細設計

## 1. 本書の目的

LINKテーブルは、Node間の接続を保持する。矢印の見た目ではなく、FROM、TO、データ名、通信種別、条件を構造化して保存する。

## 2. テーブル概要

LINKはFlowVersionに属し、from_node_idとto_node_idでNODEを参照する。ループ接続を許可する。

## 3. 採用理由

AIがどのデータがどこからどこへ送られるかを理解するため、接続線に意味情報を持たせる必要がある。

## 4. 利用機能

フローエディタ、Mermaid flowchart、sequenceDiagram、通信一覧、PLC一覧、AI DSL出力で利用する。

## 5. 関連画面

フローエディタ画面、プロパティパネル、出力画面。

## 6. 関連API

GET/PUT /api/flow-versions/{flowVersionId}。

## 7. ER上の位置

FLOW_VERSION 1:N LINK、NODE 1:N LINK(FROM)、NODE 1:N LINK(TO)。

## 8. テーブル定義

| 論理名 | 物理名 |
| --- | --- |
| 接続 | LINK |

## 9. カラム一覧

| カラム | 型 | NULL | 説明 |
| --- | --- | --- | --- |
| link_id | TEXT | NO | PK |
| flow_version_id | TEXT | NO | FLOW_VERSION FK |
| from_node_id | TEXT | NO | 接続元NODE |
| to_node_id | TEXT | NO | 接続先NODE |
| data_name | TEXT | YES | データ名 |
| communication_type | TEXT | YES | 通信種別 |
| condition_text | TEXT | YES | 条件 |
| description | TEXT | YES | 説明 |
| line_type | TEXT | NO | Bezier等 |
| display_order | INTEGER | NO | 表示順 |
| is_deleted | INTEGER | NO | 論理削除 |
| created_at | TEXT | NO | 作成日時 |
| created_by | TEXT | NO | 作成者 |
| updated_at | TEXT | NO | 更新日時 |
| updated_by | TEXT | NO | 更新者 |

## 10. PK
link_idをPKとする。

## 11. FK
flow_version_idはFLOW_VERSION、from_node_id/to_node_idはNODEを参照する。

## 12. Unique
同一FROM/TOの複数Linkは条件違いを表すため許可する。

## 13. Index
idx_link_version: flow_version_id, is_deleted。idx_link_from: from_node_id。idx_link_to: to_node_id。

## 14. Default値
line_type='Bezier'、display_order=0、is_deleted=0。

## 15. NULL可否
from_node_id、to_node_id、flow_version_id、line_typeはNOT NULL。

## 16. CHECK制約
line_typeはBezier/Straight/Step/SmoothStepを想定する。

## 17. 論理削除
Link削除時はis_deleted=1とする。

## 18. 監査カラム
共通監査カラムを保持する。

## 19. 更新タイミング
接続作成、接続先変更、データ名変更、条件変更、削除時。

## 20. 削除ルール
Node削除時は関連Link削除を確認し、必要に応じて論理削除する。

## 21. Versionとの関係
LINKはFlowVersion Snapshotの一部である。

## 22. Templateとの関係
Template適用時はfrom/toのNode IDを再採番後のIDへ張り替える。

## 23. サンプルデータ
```sql
insert into LINK(link_id, flow_version_id, from_node_id, to_node_id, data_name, communication_type, line_type, display_order, is_deleted, created_at, created_by, updated_at, updated_by)
values('lnk-001','fv-001','n-001','n-002','RFID番号','PLC通信','Bezier',1,0,'2026-07-03T00:00:00','u-001','2026-07-03T00:00:00','u-001');
```

## 24. SQL例
```sql
select * from LINK where flow_version_id='fv-001' and is_deleted=0 order by display_order;
```

## 25. パフォーマンス
FlowVersion一括取得とNode別接続検索のため、version/from/toのIndexを持つ。

## 26. 将来拡張
通信プロトコル、API仕様生成、PLCタグ生成に必要な属性を追加する。

## 27. テスト観点
Link作成、ループ作成、from/to整合、Node削除時整合、Template適用時張替えを確認する。

## 28. 完了条件
Linkがデータ送受信と条件を表現でき、AIが通信方向を誤解しないこと。
