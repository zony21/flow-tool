# 04_12_USER詳細設計

## 1. 本書の目的

USERテーブルは、GitHub OAuthでログインした利用者をアプリ内ユーザーとして管理する。

## 2. テーブル概要

GitHubユーザーID、ログイン名、表示名、メール、最終ログイン日時を保持する。プロジェクト権限、監査カラム、設定の参照元となる。

## 3. 採用理由

監査カラム、権限管理、共同編集、通知、ユーザー設定を実現するため、GitHub情報とは別にアプリ内USERを保持する。

## 4. 利用機能

認証、プロジェクトメンバー、監査、設定、通知、EditorLock。

## 5. 関連画面

ログイン画面、ユーザー設定画面、プロジェクト一覧、フローエディタ。

## 6. 関連API

GET /api/auth/me、POST /api/auth/logout、GET/PUT /api/users/me/settings。

## 7. ER上の位置

USER 1:N PROJECT_MEMBER、USER 1:N USER_SETTING、USER 1:N NOTIFICATION、USER 1:N EDITOR_LOCK。

## 8. テーブル定義

| 論理名 | 物理名 |
| --- | --- |
| ユーザー | USER |

## 9. カラム一覧

| カラム | 型 | NULL | 説明 |
| --- | --- | --- | --- |
| user_id | TEXT | NO | PK、GUIDまたはGitHub由来ID |
| github_id | TEXT | NO | GitHubユーザーID |
| github_login | TEXT | NO | GitHubログイン名 |
| display_name | TEXT | YES | 表示名 |
| email | TEXT | YES | メール |
| avatar_url | TEXT | YES | アイコンURL |
| last_login_at | TEXT | YES | 最終ログイン日時 |
| is_active | INTEGER | NO | 有効フラグ |
| created_at | TEXT | NO | 作成日時 |
| updated_at | TEXT | NO | 更新日時 |

## 10. PK
user_idをPKとする。

## 11. FK
USER自身は主要親テーブルでありFKは持たない。

## 12. Unique
github_id、github_loginはUniqueとする。

## 13. Index
idx_user_github_login、idx_user_active。

## 14. Default値
is_active=1。

## 15. NULL可否
github_id、github_loginはNOT NULL。

## 16. CHECK制約
is_activeは0/1。

## 17. 論理削除
退職・利用停止はis_active=0で表現する。

## 18. 監査カラム
USERは監査の起点であるため、created_at、updated_atを保持する。

## 19. 更新タイミング
初回ログイン、ログイン情報更新、利用停止時。

## 20. 削除ルール
物理削除しない。監査履歴保護のため無効化する。

## 21. Versionとの関係
FlowVersionのcreated_by/updated_byとして参照される。

## 22. Templateとの関係
Template作成者、更新者として利用する。

## 23. サンプルデータ
```sql
insert into USER(user_id, github_id, github_login, display_name, is_active, created_at, updated_at)
values('u-001','38487257','zony21','zony21',1,'2026-07-03T00:00:00','2026-07-03T00:00:00');
```

## 24. SQL例
```sql
select * from USER where github_login='zony21' and is_active=1;
```

## 25. パフォーマンス
OAuth認証時にgithub_idで検索するためUnique Indexを利用する。

## 26. 将来拡張
ロール、所属、権限グループ、監査ログ連携を追加可能。

## 27. テスト観点
初回ログイン作成、再ログイン更新、無効ユーザー拒否、監査参照を確認する。

## 28. 完了条件
GitHub OAuthユーザーをアプリ内ユーザーとして一意管理できること。
