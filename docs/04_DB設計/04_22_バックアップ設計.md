# 04_22_バックアップ設計

## 1. 本書の目的

本書は、AI Flow Designer のDBバックアップ設計を定義する。
設計データの消失を防ぎ、復旧可能な状態を維持することを目的とする。

## 2. 基本方針

Project、Flow、FlowVersion、Node、Linkなどは設計資産であり、バックアップ対象とする。
初期SQLiteではDBファイル単位バックアップを行う。
SQL Server移行後はDBバックアップ機能を利用する。

## 3. バックアップ対象

- PROJECT
- FLOW
- FLOW_VERSION
- LANE
- STAGE
- NODE
- LINK
- COMMENT
- TEMPLATE
- USER_SETTING
- EXPORT_HISTORY

## 4. バックアップ頻度

初期運用では日次バックアップを想定する。
重要更新前には手動バックアップを行えるようにする。

## 5. 保持期間

初期案として直近7世代を保持する。
本番運用では社内ルールに合わせて変更する。

## 6. 復旧方針

SQLiteではDBファイルを停止状態で復元する。
SQL Serverではフルバックアップと差分バックアップの利用を検討する。

## 7. 注意点

バックアップ中に書き込みが発生すると不整合が起きる可能性があるため、SQLiteではアプリ停止または安全なコピー方式を採用する。

## 8. テスト観点

- バックアップファイルが作成されること
- 復旧後にProject/Flow/Versionが参照できること
- 復旧後にLink参照が壊れていないこと
- 世代管理が機能すること

## 9. 完了条件

設計データを定期バックアップし、復旧手順が明確であること。
