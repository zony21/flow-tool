# 04_25_DB設計まとめ

## 1. 本書の目的

本書は、`04_DB設計` 配下の設計内容を総括し、DB実装時に参照すべき全体方針、対象テーブル、リレーション、今後の詳細化対象を整理する。

## 2. DB設計の到達点

04_DB設計では、AI Flow Designer のSingle Source of TruthをDBで表現するため、以下を定義した。

- DB設計方針
- ER図
- Project / Flow / Version構造
- Lane / Stage / Node / Link / Comment構造
- Template構造
- User / ProjectMember / UserSetting
- EditorLock / ExportHistory / Notification
- Index
- Migration
- SQL Server移行
- サンプルデータ

## 3. 中核構造

中核構造は以下である。

```text
PROJECT
  └ FLOW
      └ FLOW_VERSION
          ├ LANE
          ├ STAGE
          ├ NODE
          ├ LINK
          └ COMMENT
```

この構造が、AI Flow Designerの正となる。
