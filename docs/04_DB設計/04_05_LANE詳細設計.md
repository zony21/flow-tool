# 04_05_LANE詳細設計

## 1. 本書の目的
LANEテーブルは、FlowVersion内の担当レーンを保持する。担当レーンは包装PLC、統括PLC、WCS、RCS、AGF、作業者など、処理責務を持つ主体を表す。

## 2. テーブル概要
LANEはFLOW_VERSIONに属するSnapshot要素である。FlowVersionごとに独立して保持し、過去版のレーン構成を維持する。

## 3. 採用理由
AIが「どの機器が処理するか」を推測せず理解できるよう、Nodeの担当をLaneとして構造化する。

## 4. 利用機能
フローエディタ、レーン編集、Node配置、Mermaid sequenceDiagram、AI DSL出力で利用する。

## 5. 関連画面
フローエディタ画面、プロパティパネル、共通ダイアログ。

## 6. 関連API
GET/PUT /api/flow-versions/{flowVersionId}、Lane個別APIを将来追加可能。

## 7. ER上の位置
FLOW_VERSION 1:N LANE、LANE 1:N NODE。

## 8. テーブル定義
| 論理名 | 物理名 |
| --- | --- |
| 担当レーン | LANE |

## 9. カラム一覧
| カラム | 型 | NULL | 説明 |
| --- | --- | --- | --- |
| lane_id | TEXT | NO | PK |
| flow_version_id | TEXT | NO | FLOW_VERSION FK |
| lane_name | TEXT | NO | 担当名 |
| category | TEXT | YES | PLC/WCS/作業者等 |
| icon_file_id | TEXT | YES | アイコン画像参照 |
| color | TEXT | YES | 表示色 |
| description | TEXT | YES | 説明 |
| display_order | INTEGER | NO | 表示順 |
| is_deleted | INTEGER | NO | 論理削除 |
| created_at | TEXT | NO | 作成日時 |
| created_by | TEXT | NO | 作成者 |
| updated_at | TEXT | NO | 更新日時 |
| updated_by | TEXT | NO | 更新者 |

## 10. PK
lane_idをPKとする。

## 11. FK
flow_version_idはFLOW_VERSION.flow_version_idを参照する。

## 12. Unique
同一flow_version_id内のlane_name重複は初期では許可しない方針を推奨する。

## 13. Index
idx_lane_version_order: flow_version_id, is_deleted, display_order。

## 14. Default値
is_deleted=0、display_order=0。

## 15. NULL可否
lane_name、flow_version_idはNOT NULL。説明、色、アイコンはNULL可。

## 16. CHECK制約
is_deletedは0/1。colorは将来HEX形式チェックを検討する。

## 17. 論理削除
Lane削除時にNodeがある場合、削除・移動・キャンセルを画面で選択する。自動削除は禁止する。

## 18. 監査カラム
共通監査カラムを保持する。

## 19. 更新タイミング
レーン追加、名称変更、順序変更、色変更、削除時。

## 20. 削除ルール
Nodeが存在する場合はアプリ側で選択処理後に削除する。

## 21. Versionとの関係
LaneはFlowVersion Snapshotの一部であり、Version間で共有しない。

## 22. Templateとの関係
Template適用時はTemplate Laneから新規Lane IDを採番する。

## 23. サンプルデータ
```sql
insert into LANE(lane_id, flow_version_id, lane_name, category, display_order, is_deleted, created_at, created_by, updated_at, updated_by)
values('l-001','fv-001','包装PLC','PLC',1,0,'2026-07-03T00:00:00','u-001','2026-07-03T00:00:00','u-001');
```

## 24. SQL例
```sql
select * from LANE where flow_version_id = 'fv-001' and is_deleted = 0 order by display_order;
```

## 25. パフォーマンス
FlowVersion単位で一括取得するため、flow_version_idを先頭にしたIndexを必須とする。

## 26. 将来拡張
Lane種別マスタ、アイコン管理、責務分類、AI推奨レーンを追加可能。

## 27. テスト観点
Lane追加、表示順、Node紐付き削除制御、Version複製時のID再採番を確認する。

## 28. 完了条件
Laneが担当責務として保存され、Node、Version、Templateと整合すること。
