# 06_01_フローエディタ設計方針

## 1. 本書の目的

本書は、AI Flow Designer の中核機能であるフローエディタの設計方針を定義する。
フローエディタは単なる作図画面ではなく、AIが理解できる構造化データを作成・編集・保存する画面である。

## 2. 基本思想

正となるのは描画結果ではなく、FlowVersion配下のLane、Stage、Node、Link、Commentである。
画面は構造化データを編集するためのUIであり、見た目は構造データの投影結果として扱う。

## 3. 採用技術

FrontendはVue3、TypeScript、PrimeVue、Vue Flow、Pinia、Viteを使用する。
Vue FlowはCanvas、Node、Edge、Zoom、Pan、Selectionの基盤として利用する。

## 4. UI方針

古いExcel風・Visio風の固定表レイアウトにはしない。
現代的なWebアプリとして、上部Toolbar、左NodePalette、中央Canvas、右PropertyPanelを基本構成とする。

## 5. 編集対象

- Lane
- Stage
- Node
- Link
- Comment
- FlowVersion
- Template
- Export

## 6. 保存方針

編集内容はPiniaで保持し、保存時にFlowVersion APIへ一括送信する。
保存途中で不整合が残らないようAPI側でトランザクション保存する。

## 7. Version方針

VersionはSnapshotであり、Undo/Redoとは分離する。
エディタは現在開いているFlowVersionを編集対象とする。

## 8. テスト観点

- 構造化データを正として編集できること
- 画面表示と保存データが一致すること
- 保存後に再読込して同じ構造が復元できること

## 9. 完了条件

フローエディタが構造化データを作成・編集・保存する中核画面として設計されていること。
