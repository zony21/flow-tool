# 06_21_Node拡張仕様

## 1. 目的

AI Flow DesignerのNode拡張方式を定義する。

Nodeは固定種別だけでなく、将来的に業務、DB、API、PLC、外部システムなどを表す拡張Nodeを追加できる構造にする。

## 2. 基本方針

- Node Typeを文字列だけで分岐しない
- Node Type Registryで定義を管理する
- Nodeの表示、Property、Validation、Export変換を分離する
- 初期Nodeは固定提供する
- Custom Nodeは将来追加可能な構造にする

## 3. 初期Node Type

| Node Type | 用途 |
| --- | --- |
| start | 開始 |
| end | 終了 |
| process | 処理 |
| decision | 条件分岐 |
| hexagon | 外部接続・特殊処理 |
| image | 画像 |
| comment | コメント |

## 4. Node Type Registry

Node TypeはRegistryで管理する。

```ts
interface NodeTypeDefinition {
  type: string
  displayName: string
  category: string
  icon: string
  defaultLabel: string
  defaultProperties: Record<string, unknown>
  propertySchema: NodePropertySchema[]
  validationRules: NodeValidationRule[]
  rendererKey: string
  exporterKey: string
}
```

## 5. Property Schema

Nodeごとの入力項目はpropertySchemaで定義する。

```ts
interface NodePropertySchema {
  key: string
  label: string
  type: 'text' | 'textarea' | 'select' | 'number' | 'boolean' | 'json'
  required: boolean
  options?: string[]
  defaultValue?: unknown
}
```

例: Decision

```ts
{
  key: 'condition',
  label: '条件',
  type: 'textarea',
  required: true
}
```

## 6. Validation Rule

Node TypeごとのValidationを定義する。

| Node Type | Validation |
| --- | --- |
| start | incoming linkは原則0 |
| end | outgoing linkは原則0 |
| decision | conditionまたはbranch condition必須 |
| image | imageId必須 |
| hexagon | externalTypeまたはtargetSystem推奨 |

Validation結果は保存時とAI DSL Export時に利用する。

## 7. Renderer分離

Node表示はrendererKeyで切り替える。

```text
start -> StartNodeRenderer
process -> ProcessNodeRenderer
decision -> DecisionNodeRenderer
image -> ImageNodeRenderer
```

Vue Component内で大量のif文を作らない。

## 8. Exporter分離

Export変換はexporterKeyで切り替える。

| Export | 役割 |
| --- | --- |
| JSON Exporter | SSOTをそのまま出力 |
| Mermaid Exporter | 図表現へ変換 |
| AI DSL Exporter | AI理解形式へ変換 |
| PDF Exporter | 人間向け表示へ変換 |

## 9. Custom Node追加手順

将来Custom Nodeを追加する場合の手順:

1. NodeTypeDefinitionを追加
2. Property Schemaを定義
3. Validation Ruleを定義
4. Rendererを追加
5. Exporter変換を追加
6. AI DSL Schemaへの対応を追加
7. Testを追加

## 10. Backend保存方針

DBではNode TypeとProperties JSONを保持する。

| カラム | 説明 |
| --- | --- |
| NODE_TYPE | Node Type |
| PROPERTIES_JSON | Node固有Property |

Propertyの意味はNode Type Registryで解釈する。

## 11. 完了条件

- Node Type Registry方針が定義されている
- Property Schema方針が定義されている
- Validation Rule方針が定義されている
- RendererとExporterが分離されている
- Custom Node追加手順が定義されている
