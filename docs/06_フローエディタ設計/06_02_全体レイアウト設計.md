# 06_02_全体レイアウト設計

## 1. 本書の目的

フローエディタ画面の全体レイアウトを定義する。
利用者が構造化フローを効率よく編集できるよう、主要領域の役割と配置を明確にする。

## 2. レイアウト構成

```text
[Toolbar]
[NodePalette] [Canvas] [PropertyPanel]
[Status / Toast / Dialog Layer]
```

## 3. Toolbar

保存、Undo、Redo、Zoom、Export、AIレビュー、Version作成などの主要操作を配置する。

## 4. NodePalette

左側に配置し、NodeTypeを検索・選択・ドラッグできるようにする。

## 5. Canvas

中央に配置し、Vue FlowでNode、Link、Commentを表示・編集する。
CanvasはZoom、Pan、Selection、Grid、Snapに対応する。

## 6. PropertyPanel

右側に配置し、選択中のLane、Stage、Node、Link、Commentの詳細を編集する。

## 7. Status表示

未保存、保存中、保存済み、Lock状態、エラー状態を表示する。

## 8. レスポンシブ

PC大画面を主対象とする。
画面幅が狭い場合、NodePaletteとPropertyPanelは折りたたみ可能とする。

## 9. テスト観点

- 各領域が役割通りに表示されること
- Canvasが中央の主作業領域として表示されること
- 左右パネルを折りたためること

## 10. 完了条件

フローエディタの基本レイアウトが実装者に明確であること。
