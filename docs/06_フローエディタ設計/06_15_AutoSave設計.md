# 06_15_AutoSave設計

## 1. 本書の目的

AutoSaveの設計仕様を定義する。
利用者の編集内容消失を防ぎつつ、保存競合や不完全な構造保存を避ける。

## 2. 基本方針

初期実装では明示保存を基本とする。
AutoSaveは将来対応とし、Draftまたは一時保存として正式保存と分離する。

## 3. 保存対象

- Lane
- Stage
- Node
- Link
- Comment
- Canvas座標

## 4. 保存タイミング

将来対応では、一定秒数の無操作後または画面離脱前に一時保存する。

## 5. 競合制御

EditorLockと組み合わせ、他ユーザー編集中の上書きを防止する。

## 6. エラー処理

AutoSave失敗時は明示保存を促す。
編集内容は破棄しない。

## 7. テスト観点

- 未保存状態が検知されること
- AutoSave失敗時に編集内容が残ること
- Lock競合時に上書きしないこと

## 8. 完了条件

AutoSave方針が明示保存、EditorLock、Version管理と矛盾しないこと。
