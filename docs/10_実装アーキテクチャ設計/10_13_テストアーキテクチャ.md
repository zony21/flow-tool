# 10_13_テストアーキテクチャ

## 1. 目的

本書は、AI Flow Designerにおけるテストアーキテクチャを定義する。

本システムは、SSOTを中心に画面表示、保存、Snapshot、Export、AI DSL、PDF、Mermaidを生成する。
そのため、UIだけ、APIだけ、DBだけの確認では品質を担保できない。

## 2. テスト分類

| 分類 | 対象 |
| --- | --- |
| Unit Test | Domain Rule、変換処理、Validator |
| Application Test | UseCase、Service単位 |
| Repository Test | EF Core / DBアクセス |
| API Test | Controller、認証、権限、Response |
| Frontend Unit Test | Store、Adapter、Component補助処理 |
| E2E Test | Flow編集から保存・Exportまで |
| Export Test | Mermaid / PDF / JSON / AI DSL |

## 3. Backend Unit Test

対象:

- FlowDomainValidator
- NodeTypeRule
- LinkReferenceRule
- LaneStageReferenceRule
- SnapshotSerializer
- ExportInputBuilder
- AiDslConverter

確認観点:

- ID重複を検出できる。
- 存在しないNode参照を検出できる。
- Loopを許可できる。
- 判定Nodeの条件Link不足を検出できる。
- AI専用メモをExport対象に含められる。

## 4. Application Test

対象:

- SaveFlowUseCase
- CreateSnapshotUseCase
- ExportPdfUseCase
- ExportMermaidUseCase
- ImportFlowUseCase
- ApplyTemplateUseCase

確認観点:

- Save時にDomain検証が呼ばれる。
- Snapshotは保存済みFlowから作成される。
- ExportはSnapshot指定時にSnapshotを優先する。
- Template適用時にID再採番される。

## 5. Repository Test

対象:

- FlowRepository
- ProjectRepository
- SnapshotRepository
- TemplateRepository

確認観点:

- Flow構造を保存・取得できる。
- Node / Link / Commentが正しく紐付く。
- Transaction失敗時にロールバックされる。
- Snapshot作成後に元Flowを変更してもSnapshotが変わらない。

## 6. API Test

対象:

- Project API
- Flow API
- Snapshot API
- Export API
- Auth API

確認観点:

- 未ログインは401になる。
- 権限なしは403になる。
- Validation Errorは標準形式で返る。
- 存在しないFlowは404になる。
- Version競合は409になる。

## 7. Frontend Test

対象:

- Pinia Store
- Adapter
- Property Panel
- Node Palette
- Flow Canvas補助処理

確認観点:

- Flow構造からVue Flow表示モデルへ変換できる。
- Vue Flow操作結果が編集コマンドへ変換される。
- Undo / RedoがFrontend一時操作履歴として動作する。
- 保存成功後にBackend Responseで再同期される。

## 8. E2E Test

代表シナリオ:

1. GitHub OAuthでログインする。
2. Projectを作成する。
3. Flowを作成する。
4. Lane / Stageを追加する。
5. Nodeを追加する。
6. Linkを接続する。
7. Commentを追加する。
8. 保存する。
9. Snapshotを作成する。
10. Mermaid / PDF / AI DSLをExportする。

## 9. Export Test

ExportはSSOTから生成されることを確認する。

確認観点:

- MermaidにNode / Link / 条件が出る。
- PDFにLane / Stage / Node / Commentが出る。
- JSONにSSOT構造が出る。
- AI DSLに責務、工程、条件、AIメモが出る。
- 未保存Frontend状態がExportへ混入しない。

## 10. 完了条件

- Domain、Application、Repository、API、Frontend、E2Eのテスト境界が定義されている。
- SSOTから派生出力されることをテストできる。
- ExportとSnapshotの再現性をテストできる。
- AIがテスト実装の対象を判断できる。
