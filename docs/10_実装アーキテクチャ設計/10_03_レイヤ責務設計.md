# 10_03_レイヤ責務設計

## 1. 目的

本書は、AI Flow Designerの実装レイヤごとの責務を定義する。

本システムでは、画面表示、編集状態、保存データ、出力データ、AI向けデータが混在しやすい。
そのため、各レイヤの責務を明確化し、実装時にVue Flow表示モデルやPDF出力モデルがSSOTへ混入しないようにする。

## 2. レイヤ一覧

```text
Frontend
  ├─ View Layer
  ├─ Component Layer
  ├─ Store Layer
  ├─ Client Adapter Layer
  └─ API Client Layer

Backend
  ├─ API Layer
  ├─ Application Layer
  ├─ Domain Layer
  ├─ Export Layer
  └─ Infrastructure Layer

Database
  ├─ Master / Transaction Tables
  ├─ Flow Structure Tables
  ├─ Snapshot Tables
  └─ Setting / Auth Tables
```

## 3. Frontend責務

### 3.1 View Layer

画面表示を担当する。

対象:

- Header
- Toolbar
- Node Palette
- Flow Canvas
- Property Panel
- Status Bar
- Dialog

禁止事項:

- DB保存形式を直接組み立てない。
- BackendのDomain Entityを前提にしない。
- Export処理を完結させない。

### 3.2 Component Layer

UI部品単位の入力・表示制御を担当する。

対象:

- Node部品
- Lane部品
- Stage部品
- Property入力部品
- Exportボタン
- AIレビューボタン

禁止事項:

- Component内に保存APIの詳細を埋め込まない。
- Component内でFlow全体の整合性検証を完結させない。

### 3.3 Store Layer

Piniaで編集中状態を管理する。

保持対象:

- 選択中Project
- 選択中Flow
- 編集中Flow構造
- Canvas表示状態
- 選択中Node / Link / Comment
- Undo / Redo履歴
- 保存状態

注意点:

- Storeの状態は編集中の一時状態であり、DBの正ではない。
- 保存成功後にBackend Responseを元に再同期する。

### 3.4 Client Adapter Layer

SSOT構造とVue Flow表示モデルの変換を担当する。

例:

- Flow構造 → Vue Flow Nodes / Edges
- Vue Flow操作結果 → Flow編集コマンド
- Node Property → 表示用フォームモデル

禁止事項:

- Vue Flow NodeをDB保存モデルとして扱う。
- 表示専用座標から業務責務を推定する。

## 4. Backend責務

### 4.1 API Layer

HTTP Request / Responseを担当する。

対象:

- Controller
- Request DTO
- Response DTO
- 認証フィルタ
- 入力形式チェック

禁止事項:

- Controllerに業務ロジックを置かない。
- Controllerから直接DbContextを操作しない。

### 4.2 Application Layer

ユースケースを担当する。

対象:

- Project作成
- Flow保存
- Flow取得
- Snapshot作成
- Export実行
- Import実行
- Template適用

Application Serviceは、Domain検証とRepository呼び出しを組み合わせる。

### 4.3 Domain Layer

SSOTの意味と整合性を担当する。

対象:

- Project
- Flow
- Lane
- Stage
- Node
- Link
- Comment
- Version Snapshot

Domain Layerでは、構造として成立しない状態を検出する。

例:

- 存在しないNodeをLinkが参照している。
- Nodeが存在しないLaneを参照している。
- Flow内でIDが重複している。
- Node種別に必要なPropertyが不足している。

### 4.4 Export Layer

Mermaid、PDF、JSON、AI DSLの生成を担当する。

Export Layerは、保存済みFlowまたはVersion Snapshotを入力とする。
画面DOMやVue Flow内部状態を入力にしない。

### 4.5 Infrastructure Layer

外部技術依存を担当する。

対象:

- EF Core
- SQLite
- SQL Server
- File Storage
- GitHub OAuth
- PDF生成ライブラリ
- ログ出力

## 5. DB責務

DBはSSOTと履歴を保持する。

DBが保持するもの:

- 現在のFlow構造
- Version Snapshot
- Template
- Image Metadata
- User / Auth
- Setting

DBが保持しないもの:

- Undo / Redoの一時操作履歴
- Canvasの一時選択状態
- 一時的な検索条件
- Exportプレビューの一時表示状態

## 6. 完了条件

- Controllerに業務ロジックが集中しない。
- Vue ComponentがDB保存形式を直接扱わない。
- Export処理が画面DOMに依存しない。
- SSOTの検証責務がBackend側に存在する。
- DB固有処理がInfrastructure層へ閉じている。
