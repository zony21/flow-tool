# 10_05_SSOT境界設計

## 1. 目的

本書は、AI Flow DesignerにおけるSSOTと派生データの境界を定義する。

AI Flow Designerでは、Flow Canvas、Vue Flow Node、Mermaid、PDF、JSON、AI DSL、AIレビュー入力など、多数の形式で同じFlowを扱う。
これらを混同すると、AIが理解すべき責務・工程・条件・通信・DB更新情報が欠落する。

そのため、何を正とし、何を派生物とするかを固定する。

## 2. SSOTとするデータ

SSOTは以下の構造化データである。

```text
Project
 └ Flow
     ├ Lane
     ├ Stage
     ├ Node
     ├ Link
     └ Comment
```

加えて、保存時点を固定するVersion Snapshotを正式履歴として扱う。

## 3. SSOTに含める情報

### 3.1 Project

- projectId
- projectName
- description
- ownerUserId
- settings
- createdAt
- updatedAt

### 3.2 Flow

- flowId
- projectId
- flowName
- purpose
- version
- lanes
- stages
- nodes
- links
- comments
- metadata

### 3.3 Lane

- laneId
- flowId
- name
- responsibility
- orderNo
- style

Laneは、AIが「どこがその工程を担うか」を理解するための重要情報である。
単なる見た目の帯ではない。

### 3.4 Stage

- stageId
- flowId
- name
- purpose
- orderNo
- style

Stageは工程区切りであり、処理の時間的・業務的なまとまりを表す。

### 3.5 Node

- nodeId
- flowId
- laneId
- stageId
- nodeType
- title
- description
- properties
- position
- size
- aiNotes

Nodeは処理の意味を持つ。
positionやsizeは表示用だが、Nodeの一部として保存する。
ただし、positionから業務意味を推定しない。

### 3.6 Link

- linkId
- flowId
- sourceNodeId
- targetNodeId
- label
- condition
- dataName
- communicationType
- style

Linkは単なる線ではなく、処理の接続、条件、データ受け渡しを表す。

### 3.7 Comment

- commentId
- flowId
- targetType
- targetId
- commentType
- body
- position
- aiVisible

Commentは独立コメントとNode紐付けコメントの両方を許可する。
AI専用メモも保持する。

## 4. 派生データ

以下はSSOTではない。

| データ | 扱い |
| --- | --- |
| Vue Flow Nodes / Edges | Frontend表示用派生モデル |
| Canvas DOM | 表示結果 |
| Mermaid Text | Export結果 |
| PDF | Export結果 |
| JSON File | Export結果。ただしImport元にはなり得る |
| AI DSL | AI読解用Export結果 |
| AIレビュー結果 | 分析結果 |
| Undo / Redo履歴 | Frontend一時操作履歴 |

## 5. 境界ルール

### 5.1 Vue Flowを正にしない

Vue FlowのNode / Edgeは、Canvas表示のための形式である。
DB保存時は必ずFlow構造へ変換する。

### 5.2 PDFを正にしない

PDFは設計書として利用できる成果物だが、編集元にはしない。
PDFから逆変換してFlowを復元する仕様は初期対象外とする。

### 5.3 Mermaidを正にしない

Mermaidは説明・共有用の成果物である。
MermaidからFlowを完全復元することは初期対象外とする。

### 5.4 JSONとDBの違い

JSON ExportはSSOTを外部共有するためのファイルである。
DBはアプリケーション内の永続化形式である。

JSON Import時は、Backendで検証し、DB保存形式へ変換する。

### 5.5 AI DSLを正にしない

AI DSLはAIが読みやすい形式であり、保存の正ではない。
AI DSLはJSONとは別の独自仕様として管理する。

## 6. 実装上の禁止事項

- PDF出力結果を編集元として扱う。
- Mermaid文字列からDB更新する。
- Vue Flow NodeをそのままDB Entityにする。
- AI DSLをDBの正とする。
- Commentを表示注釈だけとして扱い、AI専用メモを欠落させる。
- Linkを線としてだけ扱い、条件・データ・通信情報を欠落させる。

## 7. 完了条件

- SSOTと派生データが明確に分離されている。
- Frontend表示モデルがDBへ混入しない。
- Export成果物が保存の正として扱われない。
- AIが責務・工程・接続条件を理解するための情報がSSOTに含まれている。
