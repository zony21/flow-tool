# 10_実装アーキテクチャ設計

## 1. 本書の位置付け

本フォルダは、AI Flow Designerを実装するための全体アーキテクチャ設計を管理する。

AI Flow Designerは、単なるフロー図作成ツールではなく、SSOTである構造化データを中心に、画面表示、編集、保存、Version Snapshot、Mermaid出力、PDF出力、JSON出力、AI DSL出力、将来のAPI仕様生成・DB更新一覧生成・コード生成へ展開する設計ツールである。

そのため、本章ではフロントエンド、バックエンド、DB、出力処理、認証、将来拡張を分断せず、実装時に責務境界がぶれないようにする。

## 2. 設計思想

- 図ではなく構造化データを正とする。
- Frontendは編集体験と一時操作を担当する。
- Backendは永続化、検証、Version Snapshot、Export、認証・権限を担当する。
- DBはSSOTの永続化と履歴管理を担当する。
- 出力機能は保存済み構造化データまたはSnapshotから生成する。
- Mermaid、PDF、AI DSL、JSONは個別仕様を持つが、入力元は共通化する。
- 将来SQL Serverへ移行できるよう、SQLite固有実装を閉じ込める。
