# 10_12_エラー処理ログ設計

## 1. 目的

本書は、AI Flow Designerにおけるエラー処理・ログ設計を定義する。

AI Flow Designerでは、Flow編集、保存、Export、Import、AIレビュー、認証など、多くの処理が構造化データを扱う。
エラー発生時に原因を追跡できないと、設計書品質やAIによる実装精度が低下する。

## 2. 基本方針

- Backendで標準エラーレスポンス形式を定義する。
- Frontendはエラー種別に応じて表示を切り替える。
- Validation Errorはユーザーが修正できる粒度で返す。
- System Errorは詳細を画面へ出しすぎない。
- ログには追跡IDを付与する。

## 3. エラー分類

| 分類 | HTTP | 内容 |
| --- | --- | --- |
| Validation | 400 | 入力・構造検証エラー |
| AuthRequired | 401 | 未ログイン |
| PermissionDenied | 403 | 権限なし |
| NotFound | 404 | 対象データなし |
| Conflict | 409 | 更新競合、Version競合 |
| Unsupported | 422 | 未対応形式、未対応Node |
| System | 500 | 予期しない障害 |

## 4. 標準エラーレスポンス

```json
{
  "errorCode": "FLOW_VALIDATION_FAILED",
  "message": "Flowの構造に不整合があります。",
  "traceId": "xxxxxxxx",
  "details": [
    {
      "field": "links[0].sourceNodeId",
      "message": "参照先Nodeが存在しません。"
    }
  ]
}
```

## 5. Validation Error方針

Validation Errorは、FrontendがProperty PanelやStatus Barで表示できる粒度にする。

例:

- Node名が未入力
- LinkのsourceNodeIdが存在しない
- LinkのtargetNodeIdが存在しない
- Lane参照が存在しない
- Stage参照が存在しない
- 判定Nodeの条件Linkが未設定
- 画像Nodeの画像Metadataが存在しない

## 6. Frontend表示方針

| エラー | 表示 |
| --- | --- |
| Validation | 対象項目、Status Bar、一覧表示 |
| 401 | ログイン画面へ誘導 |
| 403 | 権限不足メッセージ |
| 404 | 対象が存在しないメッセージ |
| 409 | 再読み込み・上書き確認 |
| 500 | 汎用エラー、traceId表示 |

## 7. ログ出力方針

Backendは以下をログに出力する。

- API開始・終了
- 認証ユーザー
- Project / Flow ID
- Snapshot ID
- Export Type
- エラー種別
- traceId
- 処理時間

System ErrorではStack Traceをログに残す。
ただし、FrontendへStack Traceを返さない。

## 8. traceId方針

すべてのAPIレスポンスにtraceIdを含めるか、少なくともエラーレスポンスには必ず含める。

目的:

- ユーザー問い合わせ時の調査
- Export失敗原因の追跡
- AIレビュー失敗原因の追跡
- DB保存失敗原因の追跡

## 9. 禁止事項

- Controllerごとにバラバラのエラー形式を返す。
- System ErrorのStack Traceを画面へ表示する。
- Validation Errorを単一メッセージだけに潰す。
- Export失敗時に対象Flow / Snapshotをログに残さない。
- 権限エラーとデータ不存在を不用意に区別し、情報漏えいさせる。

## 10. テスト観点

- Validation Errorが標準形式で返る。
- 401 / 403 / 404 / 409 / 500の表示分岐ができる。
- System Error時にtraceIdが返る。
- ログにProject ID / Flow ID / traceIdが出力される。
- Stack TraceがFrontendへ返らない。

## 11. 完了条件

- 標準エラーレスポンス形式が定義されている。
- Frontend表示方針が定義されている。
- traceIdでログ追跡できる。
- Validation Errorが項目単位で扱える。
