# 10_04_データフロー設計

## 1. 目的

本書は、AI Flow Designerにおける主要なデータフローを定義する。

本システムでは、ユーザー操作、Canvas表示、SSOT保存、Version Snapshot、Export、AIレビューが同じFlowを扱う。
そのため、どの処理がどのデータを入力とし、どのデータを出力するかを明確化する。

## 2. 基本データフロー

```text
User Operation
  ↓
Frontend Editing State
  ↓
Flow Structure Request DTO
  ↓
Backend Validation
  ↓
Domain Model
  ↓
Database Save
  ↓
Saved Flow Response
  ↓
Frontend Re-Sync
```

## 3. Flow編集データフロー

### 3.1 Node追加

```text
Node Palette Drag
  ↓
Canvas Drop
  ↓
Frontend creates temporary Node
  ↓
Pinia Store updates editing Flow
  ↓
Vue Flow Adapter reflects display Node
  ↓
User edits Property
  ↓
Save Request
  ↓
Backend validates Node
  ↓
DB save
```

検証観点:

- Node IDが一意である。
- Node Typeが定義済み、または拡張Nodeとして登録されている。
- Lane / Stage参照が存在する。
- 必須Propertyが入力されている。
- 画像Nodeの場合、画像Metadataが存在する。

### 3.2 Link追加

```text
User connects source Node to target Node
  ↓
Frontend creates temporary Link
  ↓
Pinia Store updates editing Flow
  ↓
Property Panel edits condition / label / data
  ↓
Save Request
  ↓
Backend validates Link
  ↓
DB save
```

検証観点:

- sourceNodeIdが存在する。
- targetNodeIdが存在する。
- Link IDが一意である。
- Loopは許可する。
- 条件分岐NodeからのLinkには条件名を設定する。

### 3.3 Lane / Stage変更

Lane / StageはNodeの責務や工程理解に直結する。
そのため、削除時は明示的な方針をユーザーに選ばせる。

選択肢:

- 内包Nodeを別Lane / Stageへ移動する。
- 内包Node / Link / Commentをまとめて削除する。

Backendでは、削除後に孤立参照が残らないことを検証する。

## 4. 保存データフロー

保存時は、Frontend編集状態から保存Request DTOを作成する。

```text
Pinia Editing Flow
  ↓ Convert
SaveFlowRequest
  ↓ API
FlowController
  ↓
FlowApplicationService
  ↓
FlowDomainValidator
  ↓
FlowRepository
  ↓
Database
```

保存成功時は、Backendが保存済みFlowを返却する。
Frontendは一時IDや更新日時をBackend Responseで再同期する。

## 5. Version Snapshotデータフロー

Version SnapshotはUndo / Redoとは異なる。

```text
Saved Flow
  ↓
Create Snapshot Request
  ↓
Backend loads current Flow
  ↓
Backend serializes immutable snapshot
  ↓
Snapshot saved
  ↓
Snapshot version returned
```

Snapshotの入力は保存済みFlowとする。
未保存のFrontend一時状態から直接Snapshotを作成しない。

## 7. AIレビュー要求データフロー

AIレビューは将来機能とするが、入力境界は先に固定する。

```text
Saved Flow or Snapshot
  ↓
AI Review Input Builder
  ↓
AI DSL / Structured JSON
  ↓
AI Review Service
  ↓
Review Result
  ↓
Frontend Review Panel
```

AIレビューには画面DOMを渡さない。
責務、工程、通信、条件、DB更新、PLC連携などの意味情報を含む構造化データを渡す。

## 8. 完了条件

- 編集、保存、Snapshot、Export、AIレビューの入力元が明確である。
- 未保存のFrontend一時状態から正式出力を作らない。
- Exportが画面DOMやVue Flow内部状態に依存しない。
- 保存成功後にFrontendがBackend Responseで再同期する設計になっている。
