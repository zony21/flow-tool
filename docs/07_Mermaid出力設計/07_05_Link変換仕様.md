# 07_05_Link変換仕様

## 1. 本書の目的

LinkをMermaid記法へ変換する仕様を定義する。
LinkはNode間の方向、データ名、通信種別、条件を表す重要な構造化要素である。

## 2. 入力項目

- linkId
- fromNodeId
- toNodeId
- dataName
- communicationType
- conditionText
- description
- lineType

## 3. flowchart変換

基本形は以下とする。

```mermaid
N001 -->|RFID番号| N002
```

## 4. ラベル生成

ラベルは以下の優先順位で組み立てる。

1. dataName
2. communicationType
3. conditionText

例:

```text
RFID番号 / PLC通信 / 読取成功時
```

## 5. sequenceDiagram変換

fromNodeのLaneからtoNodeのLaneへメッセージを生成する。

```mermaid
PLC->>WCS: RFID番号
```

## 6. 条件表現

初期はラベル内に条件を出力する。
将来alt/opt表現へ拡張可能とする。

## 7. ループ

fromNodeIdとtoNodeIdが同じ場合も有効とする。
循環処理として出力する。

## 8. Validation

fromNodeId、toNodeIdが存在しない場合は出力エラーとする。

## 9. テスト観点

- Link方向が保持されること
- dataNameがラベルへ出ること
- 条件が欠落しないこと
- ループ接続が出力できること

## 10. 完了条件

Linkの意味情報がMermaid上に失われず表現されること。
