# 07_15_識別子設計

## 1. 本書の目的

Mermaid出力で使用する識別子の生成仕様を定義する。
Mermaid識別子は表示名ではなく、内部IDを元に生成する。

## 2. 基本方針

Node、Lane、Stage、Linkの表示名は変更される可能性がある。
そのため、Mermaid上の参照識別子には内部IDを使用する。

## 3. Node識別子

nodeIdを元に `N_` プレフィックス付きの安全な識別子を生成する。

例:

```text
nodeId: n-001
Mermaid ID: N_n_001
```

## 4. Lane識別子

laneIdを元に `LANE_` プレフィックスを付ける。

## 5. Stage識別子

stageIdを元に `STAGE_` プレフィックスを付ける。

## 6. 変換ルール

- ハイフンはアンダースコアへ変換する
- 空白はアンダースコアへ変換する
- Mermaidで扱いにくい記号は除去または置換する
- 先頭が数字になる場合はプレフィックスを付ける

## 7. ラベルとの分離

識別子は内部参照用、ラベルは表示用とする。
表示名変更でLink参照が壊れてはならない。

## 8. テスト観点

- 表示名変更でMermaid識別子が変わらないこと
- 特殊文字を含むIDでも安全な識別子になること
- NodeとLaneの識別子が衝突しないこと

## 9. 完了条件

Mermaid内の参照が安定し、表示名変更に影響されないこと。
