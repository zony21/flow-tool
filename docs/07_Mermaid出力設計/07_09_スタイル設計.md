# 07_09_スタイル設計

## 1. 本書の目的

Mermaid出力時のスタイル設計を定義する。
色や形状は補助情報であり、構造化データの意味を壊さない範囲で使用する。

## 2. 基本方針

MermaidはGitHubやMarkdownで表示されるため、過度な装飾に依存しない。
読みやすさと互換性を優先する。

## 3. Nodeスタイル

NodeTypeに応じて形状を変える。
色指定は初期実装では最小限とする。

## 4. Laneスタイル

LaneのcolorをMermaid styleへ反映するかは出力オプションで制御する。
互換性問題がある場合はラベルで担当を表現する。

## 5. Linkスタイル

条件付きLinkやエラー系Linkは将来スタイル変更可能とする。
初期実装では通常矢印を使用する。

## 6. テスト観点

- 装飾なしでも意味が伝わること
- NodeTypeごとの形状が維持されること
- GitHub上で表示可能であること

## 7. 完了条件

Mermaid出力が視覚的に読みやすく、互換性を損なわないこと。
