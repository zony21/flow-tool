# 07_13_ループ表現設計

## 1. 本書の目的

ループ構造をMermaidで表現する仕様を定義する。
AI Flow Designerではフローの循環を許可するため、Mermaid出力でもループを正しく扱う。

## 2. ループ許可理由

PLC、WCS、RCSでは、待機、受信、判定、待機へ戻るような処理が多い。
そのため、ループは異常ではなく正式な構造として扱う。

## 3. flowchart表現

Linkが前工程または自己Nodeへ戻る場合も通常の矢印として出力する。

```mermaid
N001 --> N002
N002 -->|再待機| N001
```

## 4. 自己ループ

fromNodeIdとtoNodeIdが同じ場合も出力可能とする。
Mermaid表示が読みにくくなる場合はコメントで補足する。

## 5. sequenceDiagram表現

同一Laneへ戻る処理は自己メッセージとして表現する。

## 6. 無限ループとの違い

Mermaid出力は構造表現であり、実行制御ではない。
無限ループ判定はAIレビューまたは将来の検証機能で扱う。

## 7. テスト観点

- 戻り線が出力できること
- 自己ループが出力できること
- ループを不正扱いしないこと

## 8. 完了条件

循環構造をMermaidで表現でき、AIが意図的なループとして解釈できること。
