# 20_01_実装方針

## 1. 目的

AI Flow Designerの実装方針を定義する。

本ファイルは、各実装タスクを進める際の共通ルール、粒度、完了条件を定める。

## 2. 基本方針

- mainブランチを唯一の正とする
- ブランチは作成しない
- 仕様はdocsを正とする
- UI変更は設計に従う
- SSOTを崩さない
- 先に構造化データ、後から表示・出力を作る
- 実装とテスト観点を必ず紐付ける

## 3. 実装粒度

1タスクは、AIが1回の実装作業で完了できる程度にする。

悪い例:

```text
Flow Editorを実装する
```

良い例:

```text
FlowCanvas.vueを作成し、固定Nodeを1件表示する
```

## 4. タスク完了条件

各タスクは以下を満たす。

- 対象ファイルが明確である
- 入力と出力が明確である
- 関連設計が明記されている
- テスト観点が明記されている
- 完了判定が明確である

## 5. 実装優先順位

1. DB / Entity
2. Backend API
3. Auth / Permission
4. Frontend基盤
5. Flow Editor
6. Export / AI DSL
7. Settings
8. Test / QA

## 6. 禁止事項

- docsにない独自仕様を勝手に追加しない
- 構造化データを無視して図形だけ保存しない
- Frontendだけで権限制御を完結しない
- テスト観点なしでタスク完了にしない
- 不要ファイル、temp、draft、oldを残さない

## 7. 完了条件

- 実装ルールが明確である
- タスク粒度が明確である
- Phase別タスクへ展開できる
