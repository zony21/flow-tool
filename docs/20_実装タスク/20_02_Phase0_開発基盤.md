# 20_02_Phase0_開発基盤

## 1. 目的

開発を開始するためのRepository、Frontend、Backend、DB、共通設定の初期構成を作成する。

## 2. Task一覧

| Task ID | Task Name | 優先度 |
| --- | --- | --- |
| P0-001 | Repository構成確認 | A |
| P0-002 | Frontend初期構成 | A |
| P0-003 | Backend初期構成 | A |
| P0-004 | SQLite接続設定 | A |
| P0-005 | 共通README整備 | B |

## 3. P0-001 Repository構成確認

目的:

mainブランチ上で実装対象構成を確認する。

対象:

- repository root
- docs
- frontend
- backend

実装内容:

- 不要ファイルの有無を確認
- frontend / backend構成の有無を確認
- 存在しない場合は作成予定として整理

関連設計:

- 00_全体方針
- 10_実装アーキテクチャ設計

テスト観点:

- docs以外に不要なtemp/draft/oldがない

完了条件:

- main上の構成が確認されている
- 実装開始に必要なroot構成が明確である

## 4. P0-002 Frontend初期構成

目的:

Vue3 / TypeScript / PrimeVue / Vue Flow / Pinia / ViteのFrontend基盤を作成する。

対象:

- frontend

実装内容:

- Vite + Vue3 + TypeScript構成
- PrimeVue導入
- Pinia導入
- Vue Router導入
- Vue Flow導入
- src配下の基本構成作成

関連設計:

- 11_フロントエンド設計

テスト観点:

- npm installが成功する
- npm run devが起動する
- 初期画面が表示される

完了条件:

- Frontendが起動できる
- Vue / Pinia / Router / PrimeVueの最小構成が動作する

## 5. P0-003 Backend初期構成

目的:

ASP.NET Core .NET 8 / EF CoreのBackend基盤を作成する。

対象:

- backend

実装内容:

- ASP.NET Core Web API作成
- Controller構成作成
- Service / Repository / Entity用フォルダ作成
- appsettings作成
- CORS設定

関連設計:

- 12_バックエンド設計

テスト観点:

- dotnet buildが成功する
- HealthCheck APIが200を返す

完了条件:

- Backendが起動できる
- /api/health が200を返す

## 6. P0-004 SQLite接続設定

目的:

初期DBとしてSQLiteへ接続できる状態を作る。

対象:

- backend
- appsettings
- DbContext

実装内容:

- EF Core SQLite導入
- AppDbContext作成
- 接続文字列設定
- 初期Migration準備

関連設計:

- 04_DB設計
- 12_バックエンド設計

テスト観点:

- DB接続できる
- Migrationが作成できる

完了条件:

- SQLite接続が成功する
- AppDbContextがDI登録されている

## 7. P0-005 共通README整備

目的:

開発者が起動手順を理解できるようにREADMEを整備する。

対象:

実装内容:

- 起動手順
- 技術スタック

- 00_全体方針

完了条件:
