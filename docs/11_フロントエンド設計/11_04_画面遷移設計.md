# 11_04_画面遷移設計

## 1. 目的

本書は、AI Flow DesignerのFrontend画面遷移を定義する。

本システムは、Projectを選び、Flowを作成・編集し、保存・Snapshot・Export・AIレビューへ進む導線を持つ。
そのため、各画面の役割と遷移条件を明確にする。

## 2. 画面一覧

| 画面 | 役割 |
| --- | --- |
| LoginPage | GitHub OAuthログイン入口 |
| ProjectListPage | Project一覧・作成 |
| ProjectDetailPage | Project詳細・Flow一覧 |
| FlowEditorPage | Flow編集メイン画面 |
| SettingsPage | 設定画面 |
| NotFoundPage | 存在しないURL |

## 3. 基本遷移

```text
LoginPage
  ↓ login success
ProjectListPage
  ↓ select project
ProjectDetailPage
  ↓ select flow / create flow
FlowEditorPage
  ├─ save
  ├─ create snapshot
  ├─ export
  └─ ai review
```

## 4. LoginPage

### 4.1 目的

GitHub OAuthによるログイン導線を提供する。

### 4.2 表示項目

- アプリ概要
- GitHubでログインボタン
- ログイン失敗メッセージ

### 4.3 遷移

| 操作 | 遷移先 |
| --- | --- |
| GitHubログイン成功 | ProjectListPage |
| GitHubログイン失敗 | LoginPage |
| 既にログイン済み | ProjectListPage |

## 5. ProjectListPage

### 5.1 目的

ユーザーがアクセス可能なProject一覧を表示し、新規Project作成導線を提供する。

### 5.2 表示項目

- Project一覧
- Project名
- 更新日時
- 権限
- 新規作成ボタン
- 検索欄

### 5.3 遷移

| 操作 | 遷移先 |
| --- | --- |
| Project選択 | ProjectDetailPage |
| 新規Project作成 | ProjectDetailPageまたはFlowEditorPage |
| ログアウト | LoginPage |

## 6. ProjectDetailPage

### 6.1 目的

Project配下のFlow一覧とProject情報を表示する。

### 6.2 表示項目

- Project名
- Project説明
- Flow一覧
- Flow作成ボタン
- 最終更新日時
- 権限表示

### 6.3 遷移

| 操作 | 遷移先 |
| --- | --- |
| Flow選択 | FlowEditorPage |
| Flow作成 | FlowEditorPage |
| Project一覧へ戻る | ProjectListPage |

## 7. FlowEditorPage

### 7.1 目的

AI Flow Designerの中核画面である。
Lane / Stage / Node / Link / Commentを編集する。

### 7.2 画面構成

- Header
- Toolbar
- NodePalette
- FlowCanvas
- PropertyPanel
- StatusBar

### 7.3 主要操作

- Node追加
- Node移動
- Link接続
- Comment追加
- Lane / Stage編集
- 保存
- Snapshot作成
- Export
- AIレビュー

### 7.4 遷移・状態変更

| 操作 | 結果 |
| --- | --- |
| 保存 | Backend保存後、編集状態を再同期 |
| Snapshot作成 | Snapshot一覧または通知表示 |
| Export | Export Dialog表示 |
| AIレビュー | AI Review Panel表示 |
| Flow切替 | 未保存変更があれば確認 |
| Project切替 | 未保存変更があれば確認 |

## 8. SettingsPage

### 8.1 目的

ユーザー設定、表示設定、将来のNode拡張設定を扱う。

### 8.2 初期対象

- 表示テーマ
- Grid初期表示
- Snap初期状態
- Export初期オプション

### 8.3 将来対象

- Node Type拡張
- Template管理
- AIレビュー設定

## 9. 未保存変更時の遷移制御

未保存変更がある場合、以下の操作で確認Dialogを表示する。

- Project切替
- Flow切替
- 画面離脱
- ブラウザ戻る
- ログアウト

Dialog選択肢:

- 保存して移動
- 保存せず移動
- キャンセル

## 10. 権限による遷移制御

| 権限 | 制御 |
| --- | --- |
| 未ログイン | LoginPageへ遷移 |
| Viewer | 編集操作を非活性 |
| Editor | Flow編集可能 |
| Owner | Project設定・権限変更可能 |

Frontendは表示制御を行うが、最終権限判定はBackendで行う。

## 11. ルーティング例

```text
/login
/projects
/projects/:projectId
/projects/:projectId/flows/:flowId
/settings
```

## 12. 完了条件

- ProjectからFlow編集までの導線が明確である。
- 未保存変更時の制御が定義されている。
- 権限ごとの表示・遷移制御が定義されている。
- FlowEditorPageの主要操作と状態変化が明確である。
