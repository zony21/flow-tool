# 11_20_フロントエンド設計まとめ

## 1. 目的

本書は、11_フロントエンド設計の総まとめである。

AI Flow DesignerのFrontendは、SSOTを直感的に編集するためのUIであり、単なる作図画面ではない。
Vue3、TypeScript、PrimeVue、Vue Flow、Pinia、Viteを用いて、モダンで拡張可能な設計ツールを実装する。

## 2. Frontendの責務

Frontendが担当するもの:

- Project / Flow選択
- Flow Canvas表示
- Lane / Stage / Node / Link / Comment編集
- Property Panel編集
- Node Palette操作
- Undo / Redo
- Dialog表示
- Export要求
- AIレビュー要求
- Error表示
- 保存状態表示

Frontendが担当しないもの:

- DB永続化
- 保存時の最終Validation
- 権限判定の最終決定
- PDF / Mermaid / AI DSLの正式生成
- Version Snapshotの正式作成

## 4. 中核設計

### 4.1 SSOTと表示モデルの分離

正はBackendで検証・保存されたFlow構造である。
Vue Flow Node / Edgeは表示用派生モデルである。

### 4.2 Store

Pinia Storeは編集中状態を管理する。
保存成功後はBackend Responseで再同期する。

### 4.3 Adapter

Adapterは以下を変換する。

- Flow構造 → Vue Flow表示モデル
- Vue Flow Event → Editing Command
- editingFlow → SaveFlowRequest
- API Response → Store Model

### 4.4 Component

Componentは表示と操作受付を中心にする。
API通信、変換、保存処理を直接抱え込まない。

## 9. 品質基準

- Vue Flow表示モデルをDB保存形式にしない。
- 未保存変更を失わない。
- 保存失敗時もeditingFlowを保持する。
- Exportは保存済みFlowまたはSnapshotを対象にする。
- AIレビュー指摘は自動反映しない。
- Dialog、Error、Status表示を統一する。
- 大規模Flowを妨げない構成にする。
