# 11_18_アクセシビリティ設計

## 1. 目的

本書は、AI Flow DesignerのFrontendアクセシビリティ設計を定義する。

AI Flow DesignerはCanvas中心の設計ツールであり、マウス操作に依存しやすい。
しかし、業務設計ツールとして利用する以上、キーボード操作、フォーカス制御、色だけに依存しない状態表示、読み上げ補助を考慮する必要がある。

## 2. 基本方針

- 主要操作はキーボードでも実行できるようにする。
- Dialogはフォーカス制御を行う。
- 状態表示は色だけに依存しない。
- Tooltipやaria-labelでアイコン操作を補足する。
- Canvas操作は完全対応が難しいため、代替操作をProperty Panelや一覧で提供する。

## 3. 対象範囲

対象:

- Header
- Toolbar
- Node Palette
- Flow Canvas
- Property Panel
- Dialog
- Status Bar
- Error表示
- Export UI
- AIレビューUI

## 4. キーボード操作

### 4.1 共通操作

| 操作 | 動作 |
| --- | --- |
| Tab | 次の操作可能要素へ移動 |
| Shift + Tab | 前の操作可能要素へ移動 |
| Enter | Primary操作 |
| Escape | Dialog閉じる / 選択解除 |
| Delete | 選択要素削除確認 |
| Ctrl + S | 保存 |
| Ctrl + Z | Undo |
| Ctrl + Shift + Z | Redo |

### 4.2 Canvas操作

初期実装では、Canvas上の完全キーボード編集は必須としない。

代替として、以下を提供する。

- Property PanelでLane / Stage変更
- Node一覧から選択
- Link一覧から選択
- 数値入力によるPosition編集は将来対応

## 5. フォーカス制御

### 5.1 Dialog

Dialogを開いた時:

- Dialog内の最初の操作可能要素へFocusする。
- Tab移動をDialog内に閉じる。
- Dialogを閉じた時、元の操作要素へFocusを戻す。

### 5.2 Property Panel

選択対象が変わった時:

- Panel先頭へ無理にFocusを移さない。
- キーボード操作で選択した場合は、対象Formの先頭へFocusできるようにする。

## 6. 色覚多様性対応

状態表示は色だけに依存しない。

例:

- Errorは赤色 + Error Icon + テキスト
- Warningは黄色 + Warning Icon + テキスト
- Successは緑色 + Check Icon + テキスト
- 選択中は色 + Border + Badgeで示す

## 7. aria属性

アイコンボタンにはaria-labelを設定する。

例:

- 保存
- Undo
- Redo
- Export
- AIレビュー
- Grid表示切替
- Snap切替

Dialogにはroleやaria-modalを適切に設定する。
PrimeVue部品を利用する場合も、必要に応じて補足する。

## 8. スクリーンリーダー対応

Canvas内の全要素を完全に読み上げ対象にすることは初期必須ではない。

代替として、以下を用意する。

- Flow構造一覧
- Node一覧
- Link一覧
- Validation Error一覧
- AIレビュー指摘一覧

これにより、視覚Canvasを直接読めなくても構造情報へアクセスできる。

## 9. ショートカットキー

初期ショートカット:

| キー | 操作 |
| --- | --- |
| Ctrl + S | 保存 |
| Ctrl + Z | Undo |
| Ctrl + Shift + Z | Redo |
| Delete | 削除確認 |
| Esc | 選択解除 / Dialog閉じる |
| Ctrl + F | 検索 |

ショートカットはToolbarやHelpに表示する。

## 10. Error表示

Errorは以下を満たす。

- 項目直下に表示する。
- Summaryにも表示する。
- 色だけでなくIconとテキストで示す。
- traceIdがある場合はコピーできるようにする。

## 11. 禁止事項

- アイコンだけで意味を伝える。
- 色だけで状態を示す。
- Dialog表示後にFocus位置を失う。
- キーボードでDialogから抜け出せなくなる。
- Canvas操作しか代替手段がない状態にする。

## 12. テスト観点

- Tabで主要操作を移動できる。
- Dialogを開くとFocusがDialog内へ移る。
- Dialogを閉じると元のFocus位置へ戻る。
- Ctrl + Sで保存操作が呼ばれる。
- Errorが色以外でも判別できる。
- Node一覧またはError一覧から対象要素を選択できる。

## 13. 完了条件

- キーボード操作方針が定義されている。
- Dialogのフォーカス制御方針が定義されている。
- 色だけに依存しない状態表示が定義されている。
- Canvasの代替アクセス方針が定義されている。
