# 11_19_将来拡張設計

## 1. 目的

本書は、AI Flow DesignerのFrontend将来拡張設計を定義する。

AI Flow Designerは、初期Nodeや初期Export形式だけで完結するツールではない。
将来的に、拡張Node、AIレビュー強化、コード生成、共同編集、多言語対応、カスタムProperty、追加Export形式などを扱う可能性がある。

そのため、初期実装時点から拡張を妨げないFrontend構成にする。

## 2. 基本方針

- Node種別を拡張可能にする。
- Property FormをNode種別ごとに差し替え可能にする。
- Export形式を追加可能にする。
- AIレビューUIを拡張可能にする。
- Feature単位で分離し、巨大Component化を避ける。
- SSOTと表示モデルの境界を維持する。

## 3. 拡張対象

| 対象 | 拡張例 |
| --- | --- |
| Node | API Node / DB Node / PLC Node |
| Property | カスタム項目 |
| Export | Excel / Markdown / OpenAPI / SQL |
| AIレビュー | 追加レビュー観点 |
| Theme | Dark Theme / Accent Color |
| Auth | Team / Role拡張 |
| Editor | 共同編集 |
| i18n | 多言語対応 |

## 4. Node拡張

Node拡張では以下を追加できる構成にする。

- nodeType
- Palette表示
- Canvas表示Component
- Property Form
- Validation Rule
- Mermaid変換
- PDF変換
- AI DSL変換

FrontendではNodeRegistryを用意する。

```ts
interface NodeRegistryItem {
  nodeType: string;
  label: string;
  paletteItem: NodePaletteItem;
  canvasComponent: Component;
  propertyComponent: Component;
}
```

## 5. Property拡張

Node種別ごとのPropertyは固定項目だけでなく、将来カスタムPropertyへ拡張する。

方針:

- 共通Propertyと種別Propertyを分ける。
- FormはSchema Drivenに拡張できる余地を残す。
- Backend Validationとの整合を保つ。

## 6. Export拡張

将来追加候補:

- Markdown設計書
- Excel一覧
- OpenAPI定義
- SQL変更一覧
- PLC一覧
- 通信一覧
- コード生成結果

FrontendではExportTypeRegistryを用意し、ExportDialogへ形式を追加できる構成にする。

## 7. AIレビュー拡張

将来追加候補:

- 責務レビュー
- 通信レビュー
- DB更新レビュー
- API生成レビュー
- コード生成レビュー
- セキュリティレビュー
- パフォーマンスレビュー

ReviewIssueのcategoryを拡張可能にし、UIは未知カテゴリでも最低限表示できるようにする。

## 8. 共同編集

共同編集は将来機能とする。

想定課題:

- 同時編集競合
- Cursor表示
- 選択中要素表示
- Lock制御
- 操作履歴共有

初期実装では、Version競合と再読み込み導線まで対応する。

## 9. 多言語対応

初期は日本語UIを主対象とする。

将来i18nを導入できるよう、以下を避ける。

- Component内に大量の固定文言を直書きする。
- Error文言を各所に散らす。
- Node種別表示名を固定文字列で埋め込む。

## 10. Theme拡張

Design Tokenにより以下を拡張可能にする。

- Dark Theme
- Accent Color
- Project別Theme
- 高コントラストTheme

## 11. Plugin方針

完全なPlugin機構は初期対象外とする。

ただし、以下のRegistry構成により、将来Plugin化しやすくする。

- NodeRegistry
- ExportTypeRegistry
- ReviewCategoryRegistry
- PropertyFormRegistry

## 12. 禁止事項

- Node種別をif文だけで増やし続ける。
- Export形式をDialogへ直書きし続ける。
- AIレビューカテゴリを固定Enumに閉じすぎる。
- 多言語対応を妨げる文言散在を行う。
- Vue Flow内部Modelに拡張情報を閉じ込める。

## 13. テスト観点

- 新しいNode定義を追加してPaletteに表示できる。
- 新しいNode用Property Formを表示できる。
- 新しいExport形式をExportDialogに表示できる。
- 未知のAIレビューカテゴリでも一覧表示できる。
- Theme Token変更でUIが反映される。

## 14. 完了条件

- Node / Property / Export / AIレビューの拡張方針が定義されている。
- 初期実装が将来拡張を妨げない。
- Registry構成の方向性が定義されている。
- AIが本書を読んで拡張可能なFrontendを実装できる。
