# 11_16_AIレビューUI設計

## 1. 目的

本書は、AI Flow DesignerのAIレビューUI設計を定義する。

AIレビューは、保存済みFlowまたはVersion Snapshotを対象に、責務の抜け、通信の抜け、条件分岐の不備、DB更新・API連携・PLC連携などの設計上の問題を検出するための機能である。
FrontendはAIレビューの実行条件、結果表示、問題箇所への誘導、修正支援を担当する。

## 2. 基本方針

- AIレビュー対象は保存済みFlowまたはSnapshotとする。
- 未保存変更がある場合は保存確認を表示する。
- AIレビュー結果はFlow Canvas、Property Panel、専用Panelで確認できるようにする。
- 指摘はNode / Link / Lane / Stage / Commentへ紐付けられる設計にする。
- AIの指摘は自動反映せず、ユーザー確認後に反映する。

## 3. UI構成

```text
EditorHeader
  └─ AIレビュー Button

AIReviewStartDialog
AIReviewPanel
  ├─ ReviewSummary
  ├─ ReviewIssueList
  ├─ ReviewIssueDetail
  └─ ReviewActionArea

FlowCanvas
  └─ Issue Highlight

PropertyPanel
  └─ Related Review Issues
```

## 4. AIReviewStartDialog

### 4.1 入力項目

- 対象Flow
- Snapshot指定
- レビュー観点
- コメントを含めるか
- AI専用メモを含めるか
- 重要度フィルタ

### 4.2 レビュー観点

- 責務抜け
- 通信抜け
- 条件分岐不足
- DB更新不足
- API連携不足
- PLC連携不足
- AI理解困難箇所

### 4.3 実行条件

- ログイン済みである。
- 対象Projectへの閲覧権限がある。
- 対象Flowが保存済みである。
- 未保存変更がある場合は保存後に実行する。

## 5. AIReviewPanel

### 5.1 表示位置

初期実装では右側Property Panel内のTab、または下部Drawerとして表示する。

推奨:

- 通常時はProperty Panel内Tab
- 指摘一覧を広く見たい場合は下部Drawer

### 5.2 表示内容

- 総指摘数
- 重要度別件数
- カテゴリ別件数
- 指摘一覧
- 指摘詳細
- 対象要素へのジャンプ
- 修正提案

## 6. ReviewIssue Model

```ts
export interface ReviewIssue {
  issueId: string;
  severity: 'info' | 'warning' | 'error';
  category: 'responsibility' | 'communication' | 'condition' | 'db' | 'api' | 'plc' | 'aiReadability';
  title: string;
  description: string;
  targetType?: 'flow' | 'lane' | 'stage' | 'node' | 'link' | 'comment';
  targetId?: string;
  suggestion?: string;
  createdAt: string;
}
```

## 7. 指摘一覧

表示項目:

- 重要度
- カテゴリ
- タイトル
- 対象要素
- 状態

状態:

- 未確認
- 確認済み
- 修正済み
- 保留
- 無視

## 8. Canvasハイライト

AIレビュー指摘がNode / Linkへ紐付く場合、Canvas上で対象をハイライトする。

例:

- Node外枠に警告表示
- Linkに警告Style
- 対象要素のBadge表示

ハイライトはReviewIssueを確認中のみ強調し、通常編集の邪魔にならないようにする。

## 9. 対象要素ジャンプ

指摘一覧から対象要素へジャンプできる。

処理:

```text
Issue Click
  ↓
selectionStore.selectElement(targetType, targetId)
  ↓
FlowCanvas fit / center target
  ↓
PropertyPanel displays target
```

## 10. 修正提案UI

AIの修正提案は、直接自動反映しない。

表示:

- 修正提案本文
- 影響対象
- 適用可能な操作
- 手動修正導線

将来、自動修正候補を提示する場合も、ユーザー確認後にCommandとして適用する。

## 11. 履歴

AIレビュー履歴は将来機能として扱う。

初期では、最新レビュー結果をFrontend Storeで保持し、画面表示する。
将来、Backend保存により過去レビュー比較を可能にする。

## 12. Store連携

aiReviewStoreを将来追加する。

State:

- reviewDialogVisible
- isReviewing
- reviewIssues
- selectedIssueId
- reviewError
- lastReviewedAt

Actions:

- openReviewDialog
- executeReview
- selectIssue
- markIssueStatus
- clearReview

## 13. Error表示

- AIレビュー失敗時はtraceIdを表示する。
- 対象Flowが未保存の場合は保存確認を表示する。
- 権限不足の場合は403表示へつなげる。

## 14. テスト観点

- 未保存状態でAIレビュー実行時に保存確認が出る。
- レビュー結果一覧が表示される。
- 指摘クリックで対象Nodeへジャンプする。
- Link指摘がCanvas上でハイライトされる。
- 修正提案が自動反映されない。

## 15. 完了条件

- AIレビュー開始UIが定義されている。
- レビュー結果一覧・詳細・対象ジャンプが定義されている。
- Canvas / Property Panelとの連携が定義されている。
- AI指摘を安全に扱う方針が定義されている。
