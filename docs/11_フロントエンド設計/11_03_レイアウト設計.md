# 11_03_レイアウト設計

## 1. 目的

本書は、AI Flow DesignerのFrontendレイアウト設計を定義する。

本システムの編集画面は、単なるフォーム画面ではない。
Project / Flowを選択し、Node PaletteからNodeを追加し、Flow CanvasでLane / Stage / Node / Link / Commentを編集し、Property Panelで構造化情報を入力する設計ツールである。

そのため、画面領域ごとの責務を明確に定義する。

## 2. 全体レイアウト

```text
┌──────────────────────────────────────────────┐
│ Header                                       │
├──────────────────────────────────────────────┤
│ Toolbar                                      │
├───────┬───────────────────────────────┬──────┤
│       │                               │      │
│ Node  │         Flow Canvas           │Property
│Palette│                               │Panel │
│       │                               │      │
├───────┴───────────────────────────────┴──────┤
│ Status Bar                                   │
└──────────────────────────────────────────────┘
```

## 3. Header

### 3.1 目的

Headerは、Project / Flowの文脈、保存、Version、Export、AIレビューへの入口を提供する。

### 3.2 表示項目

- ロゴまたはアプリ名
- Project選択
- Flow選択
- 保存ボタン
- 保存状態
- Version表示
- Exportボタン
- AIレビューボタン
- ユーザー表示

### 3.3 操作

| 操作 | 内容 |
| --- | --- |
| Project選択 | 編集対象Projectを切り替える |
| Flow選択 | 編集対象Flowを切り替える |
| 保存 | BackendへSaveFlowRequestを送信する |
| Version | Snapshot一覧または作成導線を開く |
| Export | Exportメニューを開く |
| AIレビュー | AIレビュー要求を開始する |

### 3.4 注意点

- 未保存変更がある状態でProject / Flowを切り替える場合は確認する。
- 保存ボタンは保存中、保存済み、エラーを状態表示する。
- AIレビューは保存済みFlowまたはSnapshotを対象とするため、未保存状態では警告する。

## 4. Toolbar

### 4.1 目的

Toolbarは、Canvas編集を補助する操作を提供する。

### 4.2 表示項目

- Undo
- Redo
- Zoom In
- Zoom Out
- Fit View
- Grid表示切替
- Snap切替
- Template適用
- 検索

### 4.3 操作方針

- Undo / RedoはFrontend一時操作履歴を対象とする。
- Version Snapshotの復元とは別機能である。
- Grid / Snapは表示・編集補助であり、SSOTの業務意味ではない。
- Template適用時はID再採番後に編集状態へ反映する。

## 5. Node Palette

### 5.1 目的

Node Paletteは、Flowへ追加可能なNode種別を表示する。

### 5.2 初期Node

- 開始
- 終了
- 処理
- 判定
- 六角形
- 吹き出し
- 画像

### 5.3 将来拡張

Nodeは拡張可能とする。

将来、以下を追加できる設計にする。

- API呼び出しNode
- DB更新Node
- PLC通信Node
- 外部システム連携Node
- AIメモNode

### 5.4 操作

- PaletteからCanvasへDrag & Dropする。
- Drop位置を元にNode positionを設定する。
- Drop時に選択状態へする。
- Property Panelで詳細入力を促す。

## 6. Flow Canvas

### 6.1 目的

Flow Canvasは、Lane / Stage / Node / Link / Commentを視覚的に編集する中央領域である。

### 6.2 表示対象

- Lane
- Stage
- Node
- Link
- Comment
- Grid
- Selection
- Drag Preview

### 6.3 操作

- Node移動
- Node選択
- Link接続
- Link選択
- Comment配置
- Lane / Stage表示
- Zoom / Pan
- 範囲選択

### 6.4 注意点

Flow Canvasは編集体験を提供するが、SSOTそのものではない。
Vue FlowのNode / Edgeは表示用派生モデルであり、保存時はFlow構造へ変換する。

## 7. Property Panel

### 7.1 目的

Property Panelは、選択中要素の構造化情報を編集する。

### 7.2 対象

- Project
- Flow
- Lane
- Stage
- Node
- Link
- Comment

### 7.3 Node編集項目例

- Node名
- Node種別
- 説明
- Lane
- Stage
- 詳細Property
- AI専用メモ

### 7.4 Link編集項目例

- ラベル
- 条件
- データ名
- 通信方式
- 補足説明

### 7.5 Comment編集項目例

- コメント本文
- 紐付け対象
- AIへ渡すか
- コメント種別

## 8. Status Bar

### 8.1 目的

Status Barは、編集状態や検証状態を常時表示する。

### 8.2 表示項目

- 保存状態
- 未保存変更有無
- Validation Error数
- 選択中要素
- Zoom倍率
- Backend同期状態

### 8.3 表示例

```text
未保存の変更あり / エラー2件 / 選択: NODE-001 処理 / Zoom 100%
```

## 9. レスポンシブ方針

初期実装はPCブラウザを主対象とする。

ただし、将来のタブレット利用を妨げないよう、以下を考慮する。

- 左右Panelは折りたたみ可能にする。
- Property PanelはDrawer化できる構成にする。
- Toolbarはアイコン化できる構成にする。

## 10. 完了条件

- Header / Toolbar / NodePalette / FlowCanvas / PropertyPanel / StatusBarの責務が明確である。
- Canvas表示とSSOTの境界が明確である。
- 未保存変更、保存状態、検証状態を画面上で把握できる。
- AIが本書を読んでEditorLayoutの実装に着手できる。
