# 09_03_Flow構造

## 1. 目的

Flowは、Project内の1つの業務・処理・システム連携・搬送工程を表す単位である。
FlowはLane、Stage、Node、Link、Commentを保持し、AI構造化データの中心となる。

## 2. 必須項目

| 項目 | 型 | 必須 | 内容 |
| --- | --- | --- | --- |
| flowId | string | yes | Flow識別子 |
| projectId | string | yes | 所属Project |
| name | string | yes | Flow名 |
| description | string | no | Flow説明 |
| flowType | enum | yes | business / system / sequence / communication / custom |
| lanes | Lane[] | yes | Lane一覧 |
| stages | Stage[] | yes | Stage一覧 |
| nodes | Node[] | yes | Node一覧 |
| links | Link[] | yes | Link一覧 |
| comments | Comment[] | yes | Comment一覧 |
| currentVersionId | string | no | 最新Version |

## 3. JSON例

```json
{
  "flowId": "FLOW-0001",
  "projectId": "PRJ-0001",
  "name": "RFID読取から搬送開始まで",
  "flowType": "system",
  "lanes": [],
  "stages": [],
  "nodes": [],
  "links": [],
  "comments": []
}
```

## 4. 設計ルール

- Flowは空の状態で作成可能とする。
- Node、Link、CommentはFlowに直接属する。
- LaneとStageはNode分類・背景表示・AI責務理解に利用する。
- Flow内のIDは一意にする。

## 5. AI向け意味

AIはFlowを解析対象の最小単位として扱う。
複数Flowをまたぐ解析はProject全体レビュー機能の責務とする。

## 6. 完了条件

Flow単体を読み込めば、AIが工程、責務、接続、条件、補足を理解できる。
