# 09_07_Link構造

## 1. 目的

Linkは、Node間の接続、処理順、分岐条件、データ受け渡し、通信関係を表す要素である。
AI構造化データでは、図上の線ではなくLinkデータが接続関係の正となる。

## 2. 必須項目

| 項目 | 型 | 必須 | 内容 |
| --- | --- | --- | --- |
| linkId | string | yes | Link識別子 |
| flowId | string | yes | 所属Flow |
| fromNodeId | string | yes | 接続元Node |
| toNodeId | string | yes | 接続先Node |
| label | string | no | 表示ラベル |
| condition | string | no | 分岐条件 |
| dataName | string | no | 受け渡しデータ名 |
| protocol | string | no | 通信方式 |
| linkType | enum | yes | normal / condition / data / communication / loop |
| lineStyle | enum | yes | bezier / straight / step / smoothStep |
| properties | object | no | 拡張Property |

## 3. JSON例

```json
{
  "linkId": "LINK-001",
  "flowId": "FLOW-0001",
  "fromNodeId": "NODE-001",
  "toNodeId": "NODE-002",
  "label": "読取成功",
  "condition": "RFID番号を取得できた場合",
  "dataName": "rfidNo",
  "protocol": "internal",
  "linkType": "condition",
  "lineStyle": "bezier"
}
```

## 4. 設計ルール

- Loopは許可する。
- fromNodeIdとtoNodeIdが同一の自己参照Linkも保存可能とする。
- 存在しないNodeへのLinkは保存時バリデーションで禁止する。
- 処理順は座標ではなくLinkを正とする。
- 通信一覧生成ではdataName、protocol、fromNodeId、toNodeIdを利用する。

## 5. AI向け意味

AIはLinkを処理順序と依存関係として扱う。
Decision Nodeから出るLinkではconditionが重要であり、条件未設定はAIレビューで警告する。

## 6. 完了条件

Link情報だけでNode間の接続、条件、データ受け渡し、通信関係を復元できる。
