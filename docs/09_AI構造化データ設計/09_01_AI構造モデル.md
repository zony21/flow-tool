# AI構造化データ設計

## 基本思想
フロー図は表示結果であり、構造化データを唯一の正とする。

## モデル
Project
 └ Flow
    ├ Lane
    ├ Stage
    ├ Node
    ├ Link
    └ Comment

## Node例
```json
{
  "id":"NODE-001",
  "type":"Process",
  "title":"RFID読取",
  "laneId":"LANE-PLC",
  "stageId":"STAGE-READ",
  "processType":"Read",
  "description":"RFIDタグを読取"
}
```

## Link例
```json
{
  "from":"NODE-001",
  "to":"NODE-002",
  "data":"RFID番号",
  "protocol":"TCP/IP",
  "condition":"読取成功"
}
```

## AI DSL方針
独自DSLを正式仕様とする。
- FLOW
- LANE
- STAGE
- NODE
- LINK
- COMMENT
