# 09_01_AI構造化データ概要

## 1. 目的

AI構造化データは、AI Flow DesignerにおけるSingle Source of Truthである。
画面上の図形配置、Mermaid、PDF、JSON、AI DSL、将来のAPI仕様生成、DB更新一覧生成、PLC一覧生成、通信一覧生成、コード生成はすべて本データから生成する。

## 2. 採用理由

フロー図を正とすると、AIが読み取る際に線の重なり、図形の近接、レーン境界、コメント位置の曖昧さが残る。
そのため本システムでは、視覚表現を結果とし、構造化データを正とする。

## 3. 基本構造

```text
Project
 └ Flow
     ├ Lane
     ├ Stage
     ├ Node
     ├ Link
     └ Comment
```

## 4. 設計原則

- IDで参照関係を表現する。
- 表示座標と意味情報を分離する。
- Node種別は拡張可能にする。
- LinkはLoopを許可する。
- Commentは独立配置とNode紐付けの両方を許可する。
- AI専用メモは通常説明と分離して保持する。
- Version Snapshotで過去時点を再現できるようにする。

## 5. 生成対象

| 出力 | 用途 |
| --- | --- |
| Flow図 | 人が視覚的に確認する |
| Mermaid | Markdown・GitHub・AI入力で利用する |
| PDF | レビュー・承認・共有で利用する |
| JSON | Import / Export / API連携で利用する |
| AI DSL | AI解析・設計生成で利用する |
| API仕様 | 将来の自動生成対象 |
| DB更新一覧 | 将来の自動生成対象 |
| 通信一覧 | WCS / PLC / 外部連携整理で利用する |

## 6. 完了条件

AI構造化データだけを読めば、AIがフローの工程、責務、接続、条件、コメント、バージョンを理解できること。
