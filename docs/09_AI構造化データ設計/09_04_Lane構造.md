# 09_04_Lane構造

## 1. 目的

Laneは、Flow内の責務・担当・システム・設備・組織を表す分類単位である。
AIが「どの処理を誰・どのシステム・どの設備が担うか」を理解するための重要情報である。

## 2. 必須項目

| 項目 | 型 | 必須 | 内容 |
| --- | --- | --- | --- |
| laneId | string | yes | Lane識別子 |
| flowId | string | yes | 所属Flow |
| name | string | yes | Lane名 |
| responsibility | string | no | 責務説明 |
| laneType | enum | yes | human / system / plc / agf / external / other |
| orderNo | number | yes | 表示順 |
| style | object | no | 表示色、折りたたみ等 |

## 3. JSON例

```json
{
  "laneId": "LANE-WCS",
  "flowId": "FLOW-0001",
  "name": "WCS",
  "responsibility": "搬送指示と状態管理を担当する",
  "laneType": "system",
  "orderNo": 10
}
```

## 4. 設計ルール

- LaneはNodeの分類に利用する。
- NodeはlaneIdを持たないことも許可するが、AIレビューでは警告対象にする。
- Lane削除時はNodeを移動するか、Lane配下Nodeを削除するかをユーザーが選択する。

## 5. AI向け意味

Laneは責務境界である。
AIはLaneをもとに、責務漏れ、通信元・通信先、担当不明Nodeを検出する。

## 6. 完了条件

Lane情報により、AIが各Nodeの担当主体を判定できる。
