# 09_18_互換性設計

## 1. 目的

AI構造化データのSchema変更、項目追加、Node種別追加に対応し、過去Version SnapshotやExport JSONを読み続けられるようにする。

## 2. Schema Version方針

| 区分 | 例 | 方針 |
| --- | --- | --- |
| Major | 2.0.0 | 破壊的変更。明示Migrationが必要 |
| Minor | 1.1.0 | 後方互換あり。項目追加中心 |
| Patch | 1.0.1 | 説明修正、軽微な定義修正 |

## 3. 互換ルール

- 不明なpropertiesは削除せず保持する。
- 不明なnodeTypeはcustomとして扱う。
- 必須項目が欠落した旧データはMigrationで補完する。
- Link参照不整合は自動補完せずエラーにする。

## 4. Migration

MigrationはBackendで実行する。
Import時、Snapshot読込時、Export再生成時にschemaVersionを確認し、必要に応じて変換する。

## 5. 完了条件

過去に保存したFlowVersion Snapshotを、Schema更新後も読み込み、必要に応じてMigrationできる。
