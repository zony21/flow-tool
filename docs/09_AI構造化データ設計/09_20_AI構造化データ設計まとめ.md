# 09_20_AI構造化データ設計まとめ

## 1. 目的

09_AI構造化データ設計の内容を総括し、AI Flow DesignerにおけるSSOTの実装判断基準を整理する。

## 2. 設計思想

AI Flow Designerでは図を正としない。
正となるのは Project / Flow / Lane / Stage / Node / Link / Comment を中心とするAI構造化データである。

## 3. 実装の中核

- Projectは設計対象全体を表す。
- Flowは解析・出力の中心単位である。
- Laneは責務境界を表す。
- Stageは工程境界を表す。
- Nodeは処理単位を表す。
- Linkは接続、順序、条件、通信を表す。
- Commentは補足、レビュー、AI専用メモを表す。
- Version Snapshotは再現性を担保する。

## 5. 完了条件

AI構造化データだけを入力として、Flow図、Mermaid、PDF、JSON、AI DSLを生成できる。
AIはこのデータから工程、責務、処理順、条件、通信、補足を理解できる。
