# 09_13_バリデーション

## 1. 目的

AI構造化データの整合性を検証し、AI、Mermaid、PDF、AI DSLが誤った入力を受け取らないようにする。

## 2. 検証分類

| 分類 | 内容 | エラー区分 |
| --- | --- | --- |
| 必須項目 | ID、名称、型の不足 | error |
| 参照整合性 | Link先Node、NodeのLane / Stage存在 | error |
| AI品質 | Lane未設定、Stage未設定、条件未設定 | warning |
| 表示品質 | Node重なり、極端な座標 | warning |
| 拡張Property | 型不正、予約語利用 | error/warning |

## 3. Link検証

- fromNodeIdが存在すること。
- toNodeIdが存在すること。
- Loopは許可するため、循環はエラーにしない。
- Decision Nodeから出るLinkでconditionが空の場合はwarningにする。

## 4. Node検証

- nodeIdがFlow内で一意であること。
- nodeTypeが既知またはcustomであること。
- positionとsizeが数値であること。
- titleが空でないこと。

## 5. AIレビュー連携

バリデーションのwarningはAIレビューの入力に含める。
AIはwarningをもとに、責務不明、工程不明、通信条件不足を指摘する。

## 6. 完了条件

構造破綻は保存・Import前に止め、品質上の不足はAIレビューで指摘できる。
