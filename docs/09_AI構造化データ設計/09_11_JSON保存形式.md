# 09_11_JSON保存形式

## 1. 目的

AI構造化データをJSONとして保存・Export・Importできる形式を定義する。
JSONは外部連携、バックアップ、AI入力、将来のDSL変換元として利用する。

## 2. 基本方針

- JSONのルートはProjectまたはFlowVersion Snapshotとする。
- API通信ではcamelCaseを使用する。
- ID参照はオブジェクト埋め込みではなくIDで表現する。
- 循環参照をJSON構造として持たせない。
- 表示情報と意味情報を分ける。

## 3. Flow Snapshot JSON例

```json
{
  "schemaVersion": "1.0.0",
  "project": {
    "projectId": "PRJ-0001",
    "name": "倉庫制御システム"
  },
  "flow": {
    "flowId": "FLOW-0001",
    "name": "搬送開始フロー",
    "lanes": [],
    "stages": [],
    "nodes": [],
    "links": [],
    "comments": []
  },
  "snapshot": {
    "versionId": "VER-0001",
    "versionNo": 1,
    "createdAt": "2026-07-03T00:00:00+09:00"
  }
}
```

## 4. 保存単位

通常保存ではDB正規化テーブルを利用する。
Version Snapshot、Export、ImportではJSONとして一括保持・受け渡しする。

## 5. 完了条件

JSONだけでFlowVersionを復元でき、Mermaid、PDF、AI DSL生成の入力にできる。
