# 09_19_テスト仕様

## 1. 目的

AI構造化データがSSOTとして正しく機能し、各出力機能の共通入力として利用できることを確認する。

## 2. 単体テスト

- Project構造検証
- Flow構造検証
- Lane / Stage参照検証
- Node必須項目検証
- Link参照整合性検証
- Comment紐付け検証
- JSON Schema検証
- Version Snapshot復元
- AI DSL変換

## 3. 結合テスト

- Flow保存後にMermaid生成できる。
- Flow保存後にPDF生成できる。
- Export JSONをImportできる。
- Version Snapshotから同一構造を復元できる。
- Loopを含むFlowをエラーにしない。

## 4. AI品質テスト

- Lane未設定Nodeをwarningにできる。
- Stage未設定Nodeをwarningにできる。
- Decision Nodeの条件未設定をwarningにできる。
- CommentとAI専用メモを区別できる。

## 5. 受入条件

AI構造化データを入力として、Flow図、Mermaid、PDF、JSON、AI DSLを生成でき、各出力の要素数が一致する。
