# 09_06_Node構造

## 1. 目的

Nodeは、Flow内の処理、開始、終了、判断、画像、吹き出し、将来拡張要素を表す中心要素である。
AIはNodeを工程内の意味単位として解析する。

## 2. 必須項目

| 項目 | 型 | 必須 | 内容 |
| --- | --- | --- | --- |
| nodeId | string | yes | Node識別子 |
| flowId | string | yes | 所属Flow |
| nodeType | enum/string | yes | start / end / process / decision / hexagon / callout / image / custom |
| title | string | yes | 表示名 |
| description | string | no | 人向け説明 |
| aiMemo | string | no | AI専用メモ |
| laneId | string | no | 所属Lane |
| stageId | string | no | 所属Stage |
| position | object | yes | x, y |
| size | object | yes | width, height |
| properties | object | no | 拡張Property |
| orderHint | number | no | 同階層内の補助順序 |

## 3. JSON例

```json
{
  "nodeId": "NODE-001",
  "flowId": "FLOW-0001",
  "nodeType": "process",
  "title": "RFID読取",
  "description": "RFIDタグを読み取る",
  "aiMemo": "読取失敗時は再試行分岐へ進む",
  "laneId": "LANE-WCS",
  "stageId": "STAGE-READ",
  "position": { "x": 240, "y": 160 },
  "size": { "width": 160, "height": 80 },
  "properties": {
    "processType": "read",
    "target": "rfidTag"
  }
}
```

## 4. 設計ルール

- nodeTypeは固定列挙だけでなくcustomを許可する。
- Nodeの意味情報はtitle、description、properties、aiMemoで表現する。
- 座標は表示用であり、処理順はLinkを正とする。
- Lane / Stage未設定Nodeは保存可能だが、AIレビューで警告対象にする。

## 5. AI向け意味

AIはNodeを処理単位として扱い、Linkで順序、Laneで責務、Stageで工程を判断する。
Decision Nodeでは分岐条件をLink側のconditionと合わせて解釈する。

## 6. 完了条件

Node単体から処理内容、責務候補、工程候補、AI解析用補足を読み取れる。
