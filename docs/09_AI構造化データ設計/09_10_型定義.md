# 09_10_型定義

## 1. 目的

AI構造化データをFrontend、Backend、JSON、AI DSLで同じ意味として扱うため、TypeScriptとC#の型定義方針を定める。

## 2. TypeScript Interface方針

FrontendではVue3 / TypeScript / Piniaで利用するため、AI構造化データはinterfaceまたはtypeで定義する。

```ts
export interface FlowModel {
  flowId: string;
  projectId: string;
  name: string;
  description?: string;
  lanes: LaneModel[];
  stages: StageModel[];
  nodes: NodeModel[];
  links: LinkModel[];
  comments: CommentModel[];
}
```

## 3. C# DTO方針

BackendではAPI Request / Response / Export Serviceで利用するDTOを定義する。
Entity FrameworkのEntityとAPI DTOは分離する。

```csharp
public sealed class FlowModelDto
{
    public string FlowId { get; set; } = string.Empty;
    public string ProjectId { get; set; } = string.Empty;
    public string Name { get; set; } = string.Empty;
    public List<NodeModelDto> Nodes { get; set; } = new();
    public List<LinkModelDto> Links { get; set; } = new();
}
```

## 4. 命名規則

| 領域 | 規則 |
| --- | --- |
| JSON | camelCase |
| TypeScript | camelCase |
| C# DTO | PascalCase |
| DB | UPPER_SNAKE_CASE |

## 5. 完了条件

Frontend、Backend、JSON、DBで項目名の対応関係が明確で、変換時に意味が変わらない。
