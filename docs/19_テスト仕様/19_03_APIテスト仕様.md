# 19_03_APIテスト仕様

## 1. 目的

Backend APIの入出力、認証、認可、Validation、DB反映を確認するテスト仕様を定義する。

## 2. 基本方針

- API単位で正常系・異常系を確認する
- 401 / 403 / 404 / 400を明確に確認する
- DB保存結果まで確認する
- Project単位の認可を必ず確認する
- Export APIは出力内容も検証する

## 3. 共通確認項目

全APIで以下を確認する。

| 観点 | 内容 |
| --- | --- |
| 認証 | 未ログインは401 |
| 認可 | 権限不足は403 |
| Validation | 不正Requestは400 |
| NotFound | 対象なしは404 |
| Response | 必要項目が返る |
| DB | 保存・更新・削除結果が正しい |
