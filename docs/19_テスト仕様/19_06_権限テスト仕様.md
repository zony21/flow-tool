# 19_06_権限テスト仕様

## 1. 目的

Owner / Editor / ViewerのRole別操作可否を確認するテスト仕様を定義する。

権限制御はFrontend表示だけではなく、Backend APIで必ず確認する。

## 2. 基本方針

- Role別に許可・拒否を確認する
- Frontend制御とBackend認可を両方確認する
- 401と403を明確に分ける
- 最後のOwner保護を必ず確認する
- 設定機能の権限制御も確認する

## 3. Test User

| User | Role | 用途 |
| --- | --- | --- |
| ownerUser | Owner | 全操作確認 |
| editorUser | Editor | Flow編集確認 |
| viewerUser | Viewer | 閲覧・Export確認 |
| outsideUser | なし | Project未参加確認 |
| suspendedUser | suspended | 停止User確認 |
