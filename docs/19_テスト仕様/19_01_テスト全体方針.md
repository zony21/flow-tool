# 19_01_テスト全体方針

## 1. 目的

AI Flow Designer全体のテスト方針を定義する。

本アプリは単なる図形描画ツールではなく、構造化データを正とする設計基盤である。

そのため、テストでは見た目だけでなく、保存データ、API、出力物、AI DSL、権限制御まで確認する。

## 2. 最重要方針

Single Source of Truthである構造化データを最優先で検証する。

確認対象:

- Project
- Flow
- Lane
- Stage
- Node
- Link
- Comment
- Image
- Version
- Metadata

図形の表示は結果であり、構造化データと矛盾していないことを確認する。

## 3. テスト分類

| 分類 | 対象 | 目的 |
| --- | --- | --- |
| Unit Test | Service / Utility / Store | 小さい単位のロジック確認 |
| API Test | Controller / Service / DB | 入出力、認可、Validation確認 |
| Frontend Test | Component / Store | 表示、操作、状態管理確認 |
| E2E Test | 画面操作全体 | ユーザー操作から保存・出力まで確認 |
| Permission Test | Owner / Editor / Viewer | Role別操作可否確認 |
| Export Test | Mermaid / PDF / JSON / AI DSL | 出力品質確認 |
| AI DSL Test | DSL構造 | AIが曖昧なく解析できるか確認 |
| Regression Test | 既存仕様 | 退行防止 |

## 4. テスト優先順位

優先度A:

- 構造化データ保存・取得
- Flow Editor基本操作
- API認可
- Role別権限
- JSON / AI DSL出力

優先度B:

- Mermaid出力
- PDF出力
- Undo / Redo
- Version Snapshot
- Template適用
- Image管理

優先度C:

- UI細部
- 表示密度
- 補助設定
- Export見た目微調整

## 5. テスト命名規則

テスト名は以下の形式を推奨する。

```text
対象_条件_期待結果
```

例:

```text
FlowUpdate_ViewerRole_Returns403
NodeCreate_ValidRequest_CreatesNode
AiDslExport_DecisionNode_IncludesCondition
```

## 6. テストデータ方針

最低限以下のProjectを用意する。

| データ | 用途 |
| --- | --- |
| EmptyProject | 新規Project確認 |
| BasicFlowProject | Start / Process / End確認 |
| DecisionFlowProject | Decision / 条件分岐確認 |
| LoopFlowProject | Loop確認 |
| ImageFlowProject | Image Node確認 |
| PermissionProject | Owner / Editor / Viewer確認 |
| ExportProject | Mermaid / PDF / JSON / AI DSL確認 |

## 7. 合格基準

- 主要APIが期待Statusを返す
- 構造化データがDBとAPIで一致する
- Frontend操作後の保存結果が正しい
- Role別に操作可否が正しい
- Export出力に必要情報が欠落しない
- AI DSLが仕様どおりの構造を持つ
- 既存の重要テストが退行しない

## 8. 完了条件

- テスト分類が明確である
- 優先順位が定義されている
- テストデータ方針が定義されている
- 20_実装タスクの完了条件に転用できる
