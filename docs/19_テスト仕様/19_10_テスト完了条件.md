# 19_10_テスト完了条件

## 1. 目的

AI Flow Designerのテスト完了条件を定義する。

本ファイルは、実装タスクごとの完了判定、リリース前確認、AI実装後レビューの基準として利用する。

## 2. 全体完了条件

以下を満たすこと。

- Unit Testが主要ServiceとStoreを確認している
- API Testが主要APIの正常系・異常系を確認している
- Frontend Testが主要Componentと権限制御を確認している
- E2E Testが主要導線を確認している
- Permission TestがOwner / Editor / Viewerを確認している
- Export TestがMermaid / PDF / JSON / AI DSLを確認している
- AI DSL TestがAI解析に必要な構造を確認している

## 3. 必須合格項目

優先度Aのテストはすべて合格必須とする。

| 分類 | 必須項目 |
| --- | --- |
| SSOT | 構造化データ保存・取得 |
| Editor | Node / Link / Lane / Stage編集 |
| Permission | Owner / Editor / Viewer制御 |
| API | 401 / 403 / 400 / 404 |
| Settings | Default値 / 更新 / Validation |
| Export | JSON / AI DSL |
| AI DSL | Node / Link / Decision / Loop / Metadata |

## 4. 実装タスク完了条件への接続

20_実装タスクでは、各タスクに最低限以下を持たせる。

```text
対象:
実装内容:
関連設計:
テスト観点:
完了条件:
```

完了条件には19章のテスト観点を紐付ける。

例:

```text
Task: PermissionService実装
関連設計: 17_06_API認可設計
テスト観点: 19_02 PermissionService Unit Test / 19_06 権限テスト
完了条件: Owner/Editor/ViewerのPermission判定が期待通りである
```

## 5. リリース前確認

リリース前には以下を確認する。

- 既存テストがすべて成功している
- 重要E2Eが成功している
- Viewerで編集できない
- OwnerでMember管理できる
- Exportが成功する
- AI DSLが出力できる
- 設定Default値が返る
- 主要Errorが画面表示される

## 6. Regression観点

改修時には以下の退行を必ず確認する。

- Flow保存後にNode/Linkが欠落しない
- Version Snapshotが壊れない
- Viewer権限が編集可能にならない
- Export出力からMetadataが意図せず消えない
- AI DSLのVersionが消えない
- 設定変更でProject共通設定とUser個人設定が混ざらない

## 7. 品質ゲート

| Gate | 条件 |
| --- | --- |
| Gate 1 | Unit / APIの優先度A成功 |
| Gate 2 | Frontend / E2Eの優先度A成功 |
| Gate 3 | Export / AI DSLの優先度A成功 |
| Gate 4 | Role別権限テスト成功 |
| Gate 5 | READMEとdocs状態が一致 |

## 8. 完了条件

19章は以下を満たしたため詳細化済みとする。

- テスト分類が定義されている
- Unit / API / Frontend / E2Eの観点が定義されている
- 権限テストが定義されている
- Export / AI DSLテストが定義されている
- テストデータが定義されている
- 実装タスクの完了条件へ接続できる
